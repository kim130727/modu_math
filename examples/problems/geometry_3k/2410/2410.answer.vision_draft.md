# [전체적인 구성]
- 화면은 크게 **왼쪽의 도형(기하) 그림 영역**과 **오른쪽의 보기 텍스트 영역**으로 나뉘어 보입니다.
- 상단 왼쪽에는 문제 문장이 한 줄로 길게 배치되어 있고, 그 아래 왼쪽에는 사각형 테두리 안에 **삼각형 형태의 도형 그림**이 있습니다.
- 오른쪽에는 객관식 보기 숫자가 텍스트로 나열되어 있으며, 아래쪽 오른쪽 끝에는 빨간색으로 정답 오버레이가 따로 표시되어 있습니다.
- 도형 영역은 연한 회색 테두리의 큰 직사각형 박스로 감싸져 있습니다.
- 도형 내부에는 점 A, B, C, D, E가 검은 점과 문자 라벨로 표시되고, 여러 개의 선분이 연결되어 있습니다.
- 일부 선분 위/주변에는 파란색의 짧은 표시선 2개씩이 있어, 특정 변이 서로 평행함을 나타내는 기호처럼 보입니다.

# [보이는 텍스트]
- 상단 문제 문장:
  - `Find CD if AC = x-3, BE = 20, AB = 16, and CD = x+5.`
- 보기 텍스트:
  - `1. 32 / 2. 35 / 3. 36`
  - `/ 4. 40`
- 도형 라벨:
  - `A`, `B`, `C`, `D`, `E`
- 오른쪽 아래 빨간 오버레이:
  - `정답: D (40)`

# [반복되는 시각 패턴]
- 점 표시는 모두 **검은색 작은 원형 점**으로 통일되어 있습니다.
- 각 점 옆에 알파벳 라벨이 **검은색 큰 글자**로 붙어 있습니다.
- 변/선분은 **짙은 회색**의 실선으로 그려져 있습니다.
- 평행 표시로 보이는 기호는 **파란색 짧은 선 2개 조합**이 반복 사용됩니다.
  - 한 변 위에는 같은 모양의 표시가 1개 묶음으로 붙어 있고,
  - 다른 변에도 같은 방식의 표시가 붙어 있어 시각적으로 대응 관계를 보여 줍니다.
- 보기 항목은 `1.`, `2.`, `3.`, `4.` 번호 패턴으로 제시됩니다.
- 객관식 보기는 한 줄에 다 들어가지 않아, 두 번째 줄로 이어져 보입니다.

# [수식 단계]
- 별도의 수식 전개식이나 등호가 이어진 계산 과정은 보이지 않습니다.
- 화면에는 문제 문장 안에 주어진 관계식만 직접 나타나며, 변형 단계는 표시되어 있지 않습니다.

# [등호별 변환 의미]
- 해당 없음.
- 등호를 이용한 식 변형이나 단계별 계산 표시가 시각적으로 제시되지 않았습니다.

# [세부 내용]
- 도형 영역은 좌측에 크게 배치되어 있고, 큰 삼각형 틀 안에 내부 선분이 하나 더 그어져 있습니다.
- 꼭짓점과 내부 점의 위치 관계는 다음처럼 보입니다.
  - `A`는 왼쪽 아래 쪽
  - `D`는 오른쪽 위 쪽
  - `E`는 오른쪽 아래 쪽
  - `B`는 아래쪽 중앙 부근
  - `C`는 도형의 내부 중앙보다 약간 위쪽
- 외곽선으로 보이는 선분은 다음과 같이 연결되어 있습니다.
  - `A`에서 `D`로 가는 사선
  - `D`에서 `E`로 가는 거의 수직인 선
  - `A`에서 `E`로 가는 아래쪽 긴 사선
- 내부 선분은 `B`에서 `C`로 이어지는 거의 수직에 가까운 선으로 보입니다.
- 파란 평행표시는 두 군데에 있습니다.
  - 하나는 내부 선분 근처의 아래쪽 중앙쯤에 붙어 있고,
  - 다른 하나는 오른쪽 외곽의 수직선 근처에 붙어 있습니다.
- 이 표시는 두 선분이 서로 평행하다는 시각적 단서로 보입니다.
- 문제 문장에는 `AC`, `BE`, `AB`, `CD` 네 개의 길이 정보가 텍스트로 직접 주어져 있습니다.
- 보기에는 정수 값 4개가 제시되어 있으며, 사용자는 `CD`에 해당하는 값을 고르는 형식으로 보입니다.

# [Python DSL 작성 힌트]
- 도형 구성요소를 먼저 분리해서 작성하면 좋습니다.
  - 큰 외곽 삼각형의 세 변
  - 내부 세로에 가까운 선분
  - 점 A, B, C, D, E
  - 평행 표시 2개 묶음
- 좌표 기반으로 배치할 때는 다음 관계를 유지해야 합니다.
  - `A`는 좌하단, `D`는 우상단, `E`는 우하단
  - `B`는 아래쪽 중앙, `C`는 그보다 위쪽 중앙
- 평행 기호는 선분 위에 짧은 파란 획 두 개가 한 묶음으로 들어가도록 그리면 됩니다.
- 문제 문장은 상단 왼쪽에 한 줄 텍스트로 두고, 보기 텍스트는 오른쪽 영역에 2줄로 배치하는 구성이 적절합니다.
- 정답 오버레이는 본문과 분리된 별도 텍스트로 오른쪽 아래에 두면 원본과 유사합니다.
- SVG처럼 각 점 라벨은 점의 오른쪽 약간 위/가운데에 정렬된 형태로 배치하면 됩니다.

# [주의할 점]
- 문제를 풀거나 `CD` 값을 계산해 본문에 넣지 마세요.
- 선택지의 답을 추론해서 채우지 마세요.
- 도형의 실제 기하 의미를 단정하지 말고, 화면에 보이는 선분 연결과 평행 표시만 기준으로 묘사하세요.
- 내부 선분이 정확히 어떤 점에서 어떤 점으로 이어지는지는 좌표상 보이지만, 작성 시에는 시각적으로 보이는 연결관계만 반영하세요.
- `정답: D (40)` 표시는 원본에 포함된 오버레이일 뿐, 초안 작성 목적상 정답을 활용하지 말아야 합니다.
- `x-3`, `x+5` 같은 식은 문제 텍스트에 보이는 그대로 표기해야 하며 변형하지 마세요.

# [불확실한 부분]
- 평행 표시가 정확히 어느 두 선분의 평행을 뜻하는지, 시각적으로는 내부 선분과 오른쪽 외곽 수직선에 대응하는 것으로 보이지만, 도형 전체 맥락상 해석은 조심해야 합니다.
- 내부 점 `C`와 외곽 점 `D`, `E` 사이의 정확한 수학적 관계는 그림만으로는 단정하지 않는 것이 안전합니다.
- 오른쪽 보기의 줄바꿈 위치는 `3. 36`과 `/ 4. 40` 사이로 보이지만, 실제 DSL에서는 한 줄/두 줄 배치를 원본과 유사하게 맞추면 됩니다.
