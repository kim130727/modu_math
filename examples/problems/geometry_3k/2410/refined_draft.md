# [문제 개요]
- 화면은 **왼쪽 도형(기하) 그림 영역**과 **오른쪽 객관식 보기 영역**으로 나뉜 구성으로 보입니다.
- 상단 왼쪽에 문제 문장이 한 줄로 배치되어 있고, 그 아래에 사각형 테두리 안의 도형 그림이 있습니다.
- 오른쪽에는 보기 숫자가 텍스트로 나열되어 있으며, 오른쪽 아래에는 빨간색 정답 오버레이가 보입니다.
- 원본의 정답 오버레이는 **관찰 사실**로만 기록하고, 풀이에는 사용하지 않는 것이 안전합니다.

# [원본에 보이는 요소]
## 보이는 사실
- 상단 문제 문장:
  - `AC = x-3, BE = 20, AB = 16, CD = x+5일 때 CD를 구하시오.`
- 보기 텍스트:
  - `1. 32`
  - `2. 35`
  - `3. 36`
  - `4. 40`
- 도형 라벨:
  - `A`, `B`, `C`, `D`, `E`
- 오른쪽 아래 빨간 오버레이:
  - `정답: D (40)`

## 렌더링/레이아웃 힌트
- 문제 문장은 **상단 왼쪽 한 줄** 텍스트로 둡니다.
- 도형은 **좌측 큰 박스(연한 회색 테두리)** 안에 배치합니다.
- 보기는 **오른쪽 텍스트 영역**에 두 줄 이상으로 나누어 배치합니다.
- 정답 오버레이는 **본문과 분리된 별도 텍스트**로 오른쪽 아래에 두는 구성이 원본과 유사합니다.

# [수식/관계 구조]
## 보이는 사실
- 문제 문장 안에 다음 식/관계가 직접 적혀 있습니다.
  - `AC = x-3`
  - `BE = 20`
  - `AB = 16`
  - `CD = x+5`
- 별도의 계산식, 변형식, 등호 연쇄는 화면에 보이지 않습니다.

## 추론되는 수학적 의미
- 문제는 도형 위의 길이 관계를 이용해 `CD`를 묻는 객관식 문제로 보입니다.
- 다만, 도형의 정확한 기하적 조건은 그림만으로 단정하지 않고, **텍스트에 적힌 길이 정보와 선분 배치**만 기준으로 다루는 것이 안전합니다.

# [등호별 변환 의미]
- 해당 없음.
- 화면에 **식 변형 단계가 등호로 이어진 형태**는 보이지 않습니다.
- 따라서 등호별로 “무엇이 바뀌었는지”를 설명할 체인 자체가 없습니다.

# [빈칸 정책]
- 화면에 **빈칸(□, ___, 공란 입력란)**은 보이지 않습니다.
- 따라서 빈칸을 채우는 작업은 없고, **추론값을 임의로 넣지 않는 정책**을 유지하면 됩니다.
- 정답 오버레이 `정답: D (40)`는 보이지만, 이는 원본 표시일 뿐 **문제의 빈칸 답으로 역산해 본문에 반영하지 않습니다.**

# [반복 패턴]
## 보이는 사실
- 점 표시는 모두 작은 검은 원형 점으로 통일되어 있습니다.
- 각 점 옆 라벨은 검은색 알파벳 글자입니다.
- 선분은 짙은 회색의 실선으로 보입니다.
- 평행 표시처럼 보이는 기호는 **파란색 짧은 선 2개**가 한 묶음으로 반복됩니다.
- 객관식 번호는 `1.`, `2.`, `3.`, `4.` 패턴입니다.
- 보기는 한 줄에 다 들어가지 않아 **줄바꿈되어 2줄처럼 보입니다.**

## 렌더링/레이아웃 힌트
- 반복되는 점-라벨 조합은 동일한 스타일로 통일합니다.
- 파란 평행 표시는 선분 위에 **짧은 획 두 개**를 그룹으로 붙여 표현합니다.
- 보기 번호는 균일한 간격으로 정렬하고, 줄바꿈 위치는 원본의 압축된 배치를 반영합니다.

# [DSL 작성 힌트]
## 보이는 사실
- 도형은 좌측의 큰 박스 안에 있습니다.
- 점과 선분의 구성은 다음처럼 보입니다.
  - `A`는 좌하단 쪽
  - `D`는 우상단 쪽
  - `E`는 우하단 쪽
  - `B`는 하단 중앙 부근
  - `C`는 내부 중앙보다 약간 위쪽
- 외곽선으로 보이는 연결:
  - `A`–`D` 사선
  - `D`–`E` 거의 수직인 변
  - `A`–`E` 아래쪽의 긴 사선
- 내부 선분:
  - `B`–`C`로 이어지는 거의 수직에 가까운 선
- 파란 평행표시는 두 군데에 보입니다.
  - 하나는 내부 선분 쪽
  - 다른 하나는 오른쪽 외곽 수직선 쪽

## 렌더링/레이아웃 힌트
- 도형 구성요소를 분리해서 배치합니다.
  - 외곽 삼각형/사변형처럼 보이는 큰 테두리 선분들
  - 내부 선분 1개
  - 점 A, B, C, D, E
  - 파란 평행 표시 2묶음
- 좌표 배치 시 상대 위치는 유지합니다.
  - `A`: 좌하단
  - `D`: 우상단
  - `E`: 우하단
  - `B`: 하단 중앙
  - `C`: 그보다 위쪽 중앙
- 라벨은 점의 오른쪽 약간 위에 두는 방식이 원본과 유사합니다.
- 선분의 교차/접점은 너무 정밀하게 해석하지 말고, **시각적으로 연결된 것처럼 보이는 관계**만 반영합니다.

## DSL component hints
- 큰 외곽 도형
- 내부 선분
- 점 마커
- 문자 라벨
- 평행 표시 기호
- 문제 문장 텍스트
- 객관식 보기 텍스트
- 정답 오버레이 텍스트

# [semantic/solvable 힌트]
## 보이는 사실
- 문제는 `CD` 값을 고르는 객관식 형식입니다.
- 보기 후보는 `32, 35, 36, 40`입니다.
- 원본에는 `정답: D (40)` 오버레이가 보입니다.

## 추론되는 수학적 의미
- 텍스트에 길이식이 주어져 있으므로, 도형의 평행/대응 관계를 이용해 길이를 비교하거나 식을 세우는 유형으로 보입니다.
- 다만 이 문서에서는 **풀이를 진행하지 않고**, 구조적 정보만 남깁니다.
- 원본에 정답이 인쇄되어 보이더라도, DSL 작성용 초안에서는 **정답을 채택해 문제를 해결한 것처럼 쓰지 않는 것**이 안전합니다.
