# [전체적인 구성]
- 화면은 **좌측에 도형/각도 표시가 있는 기하 그림**, **우측 상단에 선택지 텍스트**, **좌측 상단에 문제 문장**이 배치된 형태입니다.
- 전체 배경은 흰색이며, 좌측 큰 영역에는 연한 회색 테두리의 사각형 박스가 하나 있습니다. 이 박스는 기하 도형이 들어 있는 작업 영역처럼 보입니다.
- 기하 영역 안에는 검은 점 10개(A~J)와 파란 선분들이 연결되어 있고, 일부 선분 끝에는 화살표 모양이 있어 **방향이 있는 선**처럼 표현됩니다.
- 각 점 주변에는 각도 값 텍스트가 배치되어 있으며, 중심에 해당하는 점이나 각도 표시가 여러 개 겹쳐 보입니다.
- 우측에는 보기(1~4)가 두 줄로 나뉘어 표시되어 있습니다.
- 우하단에는 빨간색으로 `정답: C (74)`라는 오버레이가 보이지만, 이는 입력 SVG에 포함된 표시이며 문제 풀이 과정의 일부로 보이지는 않습니다.

# [보이는 텍스트]
- 상단 문제 문장: `Find the value of x.`
- 우측 선택지:
  - `1. 56 / 2. 68 / 3. 74 /`
  - `4. 84`
- 도형 주변 각도 표기:
  - `X°`
  - `(X-6)°`
  - `(X+10)°`
  - `56°`
  - `(X+4)°`
- 우하단 빨간 오버레이:
  - `정답: C (74)`

# [반복되는 시각 패턴]
- **점과 선분의 반복**: A~J로 보이는 점들이 여러 선분으로 연결되어 있어 다각형/연결 도형 구조를 이룹니다.
- **파란색 선분 통일**: 모든 선분이 동일한 파란색 계열로 그려져 있어 하나의 연속된 기하 도형 묶음처럼 보입니다.
- **화살표 끝단 반복**: 일부 선분의 끝에는 두 개의 짧은 선으로 이루어진 화살표 머리가 반복적으로 붙어 있습니다.
- **각도 라벨 반복**: 각도는 `X°`, `(X-6)°`, `(X+10)°`, `(X+4)°`, `56°`처럼 텍스트 형태로 분산 배치되어 있습니다.
- **정답 오버레이 형식**: 우하단에 붉은 글씨로 정답 안내가 겹쳐져 있는 형식이 보입니다.

# [세부 내용]
- 좌측 기하 도형 영역에는 점들이 다음처럼 배치되어 있습니다:
  - 상단 쪽: `F`, `J`, `D` 부근
  - 중간 쪽: `B`, `C`
  - 하단 쪽: `H`, `G`, `E`, `A`, `I`
- 파란 선분 연결 관계가 복잡하게 얽혀 있으며, 몇몇 선분은 한 점에서 다른 점으로 뻗고, 또 다른 선분과 이어져 있습니다.
- 각도 라벨의 위치를 보면:
  - `X°`는 왼쪽 중단 부근, `C` 근처 아래쪽에 배치
  - `(X-6)°`는 하단 중앙 왼쪽 쪽, `H` 근처
  - `(X+10)°`는 하단 중앙 오른쪽 쪽, `G` 근처
  - `56°`는 오른쪽 상단 쪽, `J` 근처
  - `(X+4)°`는 상단 중앙 왼쪽 쪽, `B` 근처
- 문제 문장은 좌측 상단에 단독으로 놓여 있고, 선택지는 우측 상단에 두 줄 구성으로 배치되어 있습니다.
- 선택지 텍스트는 숫자와 슬래시(`/`)로 이어진 형태이며, 줄바꿈이 한 번 들어가 있습니다.
- 빨간 `정답: C (74)`는 화면의 가장 오른쪽 아래에 붙어 있어, 실제 문제의 본문과는 분리된 별도 표시처럼 보입니다.

# [Python DSL 작성 힌트]
- 기하 도형은 **점 10개를 먼저 좌표로 배치한 뒤**, 선분을 순서대로 연결하는 방식이 적절해 보입니다.
- 선분은 모두 동일한 파란색, 비교적 두꺼운 선폭으로 표현되어 있으므로 스타일을 통일하는 편이 좋습니다.
- 화살표가 있는 선분은 일반 선분과 구분하여, **끝점에 화살표 머리 2개 선**을 별도로 추가하는 구조가 맞아 보입니다.
- 각도 텍스트는 도형 안에 직접 얹히므로, 점/선분보다 위 레이어에 배치해야 합니다.
- 문제 문장과 선택지는 별도의 텍스트 블록으로 두어야 하며, 선택지는 두 줄로 나뉘어 있으므로 줄간격을 고려해야 합니다.
- 우하단의 빨간 정답 오버레이는 필요에 따라 제외하거나, 참고용으로만 별도 레이어 처리하는 것이 좋습니다.

# [주의할 점]
- **정답은 추론해서 쓰지 마세요.** 화면에 `정답: C (74)`가 보이더라도, DSL 작성용 초안에서는 문제 풀이 결과로 간주하지 않는 편이 안전합니다.
- `X`는 대문자 형태로 보이며, 일부 각도 표기에서는 소문자 x가 아니라 **대문자 X**로 읽힙니다.
- 기하 도형의 실제 연결 구조는 선분이 겹치거나 교차해 보여서, 단순한 다각형 하나로 단정하면 안 됩니다.
- 각도 텍스트의 정확한 기준 각(내각/외각)은 그림만으로 확정하기 어려우므로, 위치 관계만 서술하는 것이 좋습니다.
- 선택지 텍스트는 원문 SVG상 슬래시로 구분되어 있으므로, 실제 렌더링 시 줄바꿈과 간격을 주의해야 합니다.

# [불확실한 부분]
- 점 A~J의 정확한 도형 의미(어떤 점이 어떤 변의 꼭짓점인지)는 그림만으로 완전히 확정하기 어렵습니다.
- 화살표가 있는 선분이 정확히 어떤 방향성을 의미하는지, 혹은 단순 장식인지 SVG만으로는 단정하기 어렵습니다.
- `X°`와 다른 각도 표기들이 각각 어떤 각의 측정값인지, 도형의 내부각인지 외부각인지는 명확히 판별되지 않습니다.
- 우하단의 `정답: C (74)`는 자동 오버레이로 보이지만, 작업 목적상 포함 여부를 별도로 판단해야 합니다.
