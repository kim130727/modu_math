[전체적인 구성]
- 화면 왼쪽에는 도형 중심의 기하 그림이 크게 배치되어 있고, 오른쪽에는 선택지 텍스트가 세로로 정렬되어 있습니다.
- 상단 왼쪽에는 영어 문장 형태의 문제 설명이 한 줄로 표시됩니다.
- 기하 도형은 큰 삼각형 1개와 그 내부/아래쪽의 추가 선분 1개로 구성됩니다.
- 도형 영역은 연한 회색 테두리의 직사각형 박스 안에 들어 있습니다.
- 화면 오른쪽 하단에는 정답 오버레이처럼 보이는 붉은 글씨가 있으나, 이는 문제 본문이 아니라 시스템 표시로 보입니다.

[보이는 텍스트]
- 상단 문제 문장:
  - `If X N = 6, X M = 2, and X Y = 10, find N Z.`
- 오른쪽 선택지:
  - `1. 12 / 2. 16 / 3. 24`
  - `4. 32`
- 도형 위 점 이름:
  - `X`, `Y`, `Z`, `N`, `M`
- 우하단 붉은 글씨:
  - `정답: C (24)`

[반복되는 시각 패턴]
- 삼각형의 각 꼭짓점에 검은 점이 찍혀 있고, 점 옆에 큰 글자 라벨이 붙어 있습니다.
- 바닥 변과 내부 선분 일부에는 파란색의 짧은 표시가 두 줄씩 반복되어 있어, 평행 표식 또는 같은 성질을 암시하는 기호처럼 보입니다.
- 선택지는 숫자 앞에 `1.`, `2.`, `3.`, `4.`가 붙는 객관식 형식입니다.
- 텍스트는 왼쪽 문제문과 오른쪽 선택지로 공간이 명확히 분리되어 있습니다.

[수식 단계]
- 별도의 등호 변형식이나 계산 과정은 보이지 않습니다.
- 문제 문장 안에 점 사이의 거리 관계가 직접 주어지는 형태입니다.
- 보이는 관계는 다음과 같은 입력 정보 수준의 문장형 수식입니다:
  - `XN = 6`
  - `XM = 2`
  - `XY = 10`
  - `find NZ`
- 단, 이는 문제에 적힌 관계를 옮겨 적은 것이며 풀이 전개는 아닙니다.

[등호별 변환 의미]
- 해당 없음.
- 등호가 연속된 변형 과정이 표시되지 않았습니다.

[세부 내용]
- 도형은 삼각형 `XZY`로 보입니다.
  - 밑변은 `X`에서 `Z`로 이어지는 수평선입니다.
  - 왼쪽 변은 `X`에서 위쪽 꼭짓점 `Y`로 이어지는 대각선입니다.
  - 오른쪽 변은 `Y`에서 `Z`로 이어지는 대각선입니다.
- 점 `N`은 밑변 `XZ` 위에 있으며, `X`와 `N` 사이에 놓여 있습니다.
- 점 `M`은 선분 `XY` 위에 있으며, `X`와 `Y` 사이의 중간보다 아래쪽에 위치한 점으로 보입니다.
- 점 `M`에서 `N`으로 이어지는 내부 선분이 하나 그려져 있습니다.
- 파란색 짧은 표식이 두 군데에 있습니다.
  - 하나는 내부 선분 `MN` 근처의 오른쪽 위쪽에,
  - 다른 하나는 밑변 쪽의 왼쪽 아래 구간 근처에 있습니다.
- 이 표식들은 해당 두 선분 또는 두 구간이 같은 종류의 표시를 공유함을 시각적으로 암시합니다.
- 오른쪽 선택지는 한 줄과 다음 줄에 걸쳐 표시되며, 마지막 선택지 `4. 32`는 아래 줄로 내려가 있습니다.
- 화면 맨 아래 오른쪽의 붉은 `정답: C (24)`는 시험지 본문과 구분되는 별도 오버레이입니다.

[Python DSL 작성 힌트]
- 도형 영역을 먼저 큰 박스로 잡고, 그 안에 삼각형 외곽선 3개를 그리는 구조가 적절합니다.
- 꼭짓점 좌표는 다음 관계를 유지하면 됩니다:
  - `X`는 좌하단,
  - `Z`는 우하단,
  - `Y`는 상단,
  - `N`은 `XZ` 위,
  - `M`은 `XY` 위.
- 점 라벨은 각 검은 점의 바로 옆에 붙이고, 글자 크기는 문제문보다 훨씬 작지 않게 설정하는 편이 좋습니다.
- `MN` 내부선분은 대각선으로 그려지고, 바닥의 `XZ` 선분은 수평입니다.
- 파란 표식은 선분에 짧은 평행 표시처럼 2개의 짧은 사선을 묶어 배치하면 유사하게 표현할 수 있습니다.
- 문제문은 좌상단, 선택지는 우상단~중단에 배치하는 레이아웃이 안전합니다.
- 선택지는 `1~4`를 두 줄에 나누어 배치하는 형식도 반영할 수 있습니다.

[주의할 점]
- `X N`, `X M`, `X Y`는 텍스트에서 띄어쓰기로 분리되어 보이지만, 의미상 점 사이 거리 표기로 읽히는 형태입니다. 다만 풀이나 값 계산은 하지 마세요.
- `정답: C (24)`는 시각 자료에 포함되어 있지만, DSL 작성 시에는 일반적으로 제외하는 편이 자연스럽습니다.
- 파란 표식이 정확히 어떤 성질을 뜻하는지(평행, 합동 표시 등)는 그림만으로 확정하지 말고, 표시 모양만 기술하세요.
- 내부 점 `M`과 `N`의 정확한 수치 좌표는 SVG에 있으나, DSL에서는 상대적 배치 관계를 우선적으로 반영하면 됩니다.

[불확실한 부분]
- 파란 표시의 수학적 의미는 명시되지 않아 확정할 수 없습니다.
- `MN`이 실제로 어떤 성질을 나타내는 선분인지, 또는 추가 조건선인지 그림만으로는 단정하기 어렵습니다.
- 선택지의 줄바꿈은 원본 배치상 두 줄이지만, 화면폭에 따라 한 줄처럼도 보일 수 있습니다.
- 우하단의 붉은 정답 표시는 원문 문제의 일부가 아니라 시스템 오버레이일 가능성이 높습니다.
