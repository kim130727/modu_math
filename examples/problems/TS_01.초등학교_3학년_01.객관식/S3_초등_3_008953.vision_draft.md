[전체적인 구성]

- 화면 상단에는 문제 문장이 왼쪽부터 가로로 길게 배치되어 있습니다.
- 그 아래 중앙에는 연한 연두색 계열의 표가 있고, 표 위쪽에 제목 상자가 따로 떠 있는 형태입니다.
- 표는 2열 구조이며, 왼쪽 열은 목장 이름, 오른쪽 열은 우유 생산량을 그림 아이콘으로 나타냅니다.
- 표 아래쪽 오른편에는 아이콘 범례가 있어, 큰 우유병 아이콘과 작은 우유병 아이콘이 각각 몇 kg를 뜻하는지 보여줍니다.
- 화면 하단에는 `(정답)`, `(해설)` 텍스트와 함께 설명 문장이 이어집니다.
- 전체적으로 “문장 → 그림 표 → 범례 → 정답/해설” 순서로 읽히는 구성입니다.

[보이는 텍스트]

- `23.`
- `목장별 우유 생산량을 조사하여 그림 그래프로 나타내었습니다. 소목장의 우유 생산량은 430kg이라고 합니다. 우유 생산량이 두 번째로 많은 목장은 어디일까요?`
- 표 제목: `목장별 우유 생산량`
- 표의 열 제목: `목장`, `우유 생산량`
- 행 이름: `신선`, `청정`, `초록`, `푸른`
- 범례: 큰 우유병 그림 옆에 `100kg`, 작은 우유병 그림 옆에 `10kg`
- 아래쪽 문구: `( 신선, 청정, 초록, 푸른 ) 목장`
- `(정답)초록`
- `(해설)각 목장별로 [작은 우유병 그림]의 수를 비교하면 신선 목장이 3개, 청정 목장이 5개, 초록 목장이 4개, 푸른 목장이 2개입니다.`
- `따라서 5>4>3>2이므로 우유 생산량이 두 번째로 많은 목장은 초록 목장입니다.`

[반복되는 시각 패턴]

- 각 목장 행마다 우유병 아이콘이 여러 개 반복되어 있어 생산량을 시각적으로 비교하게 되어 있습니다.
- 큰 우유병 아이콘과 작은 우유병 아이콘이 섞여 있으며, 같은 종류의 아이콘이 연속으로 나열된 형태입니다.
- 행마다 아이콘의 개수가 다르며, 이 차이를 통해 상대적인 크기를 비교하도록 구성되어 있습니다.
- 표의 배경색과 테두리는 연한 초록색 계열로 통일되어 있고, 제목 상자도 같은 계열로 맞춰져 있습니다.
- 범례는 본문 표와 분리되어 있지만 같은 아이콘을 사용해 의미를 연결합니다.

[수식 단계]

- 화면에 계산식처럼 이어진 등식 변형은 보이지 않습니다.
- 다만 해설 부분에 `5>4>3>2`와 같이 크기 비교 표현이 한 줄로 제시되어 있습니다.
- 이 표기는 순서 비교를 나타내는 것으로 보이며, 단계별 계산식 변형은 아닙니다.

[등호별 변환 의미]

- 해당 없음.
- 등호 `=`를 사용한 연속 변형식은 화면에 나타나지 않습니다.

[세부 내용]

- 표의 구조:
  - 왼쪽 좁은 열: 목장 이름
  - 오른쪽 넓은 열: 각 목장의 생산량을 아이콘으로 표시
- 행 구성:
  - `신선` 행: 큰 우유병 여러 개와 작은 우유병 여러 개가 오른쪽 칸에 왼쪽부터 순서대로 배치됨
  - `청정` 행: 큰 우유병 여러 개와 작은 우유병 몇 개가 이어서 배치됨
  - `초록` 행: 표 안에서는 아이콘이 보이지 않는 빈 행으로 보임
  - `푸른` 행: 큰 우유병 2개와 작은 우유병 여러 개가 이어서 배치됨
- 범례:
  - 큰 우유병 그림 1개가 `100kg`을 뜻하는 것으로 표시됨
  - 작은 우유병 그림 1개가 `10kg`을 뜻하는 것으로 표시됨
- 해설 텍스트는 각 목장의 “작은 우유병 개수”를 비교하는 방식으로 설명하고 있습니다.
- 문제의 질문은 “두 번째로 많은 목장”을 묻고 있으므로, 단순히 가장 많은 곳이 아니라 순위를 따지는 구조입니다.
- 아래 `( 신선, 청정, 초록, 푸른 ) 목장` 문구는 보기처럼 제시된 목장 이름 나열로 보입니다.
- `(정답)초록`은 화면에 직접 인쇄되어 있지만, 작성 단계에서는 정답을 그대로 옮겨 적는 용도보다 시각 구조 파악에만 참고해야 합니다.

[Python DSL 작성 힌트]

- 표를 먼저 큰 사각형으로 만들고, 그 안에 2열 5행 형태로 나누는 구성이 필요해 보입니다.
- 상단에는 표 제목용 작은 연두색 박스를 별도로 배치해야 합니다.
- 왼쪽 열은 좁고, 오른쪽 열은 넓게 설정하는 것이 자연스럽습니다.
- 각 행의 왼쪽 셀에는 목장 이름 텍스트를 가운데 정렬에 가깝게 배치합니다.
- 오른쪽 셀에는 우유병 아이콘을 가로로 반복 배치합니다.
- 큰 아이콘과 작은 아이콘의 크기 차이가 분명하므로, 서로 다른 이미지 소스를 써서 구분해야 합니다.
- 범례는 표 아래 오른쪽에 따로 두고, 큰 아이콘 옆에 `100kg`, 작은 아이콘 옆에 `10kg` 텍스트를 나란히 배치합니다.
- 문제 문장은 화면 상단 왼쪽에서 시작해 여러 줄로 줄바꿈되어 있으므로, 텍스트 박스 폭을 적절히 제한해야 합니다.
- 해설 문구는 본문과 분리된 아래쪽 텍스트 영역으로 처리하는 것이 좋습니다.

[주의할 점]

- `초록`을 정답으로 단정해 먼저 채우지 않도록 주의해야 합니다.
- 화면에 보이는 아이콘 개수와 위치를 그대로 옮기는 데 집중하고, 값 자체를 재계산해서 표시하지 않는 것이 안전합니다.
- 표 안의 `초록` 행은 아이콘이 비어 있는 것처럼 보이므로, 빈 공간을 임의로 채우지 마세요.
- 범례의 숫자 `100kg`, `10kg`은 아이콘의 의미를 설명하는 표시이므로, 표 안의 개수와 혼동하지 않도록 해야 합니다.
- 해설에 있는 비교식 `5>4>3>2`는 시각적 설명의 일부로 보이지만, 이를 별도의 계산 단계로 과도하게 확대 해석하지 마세요.

[불확실한 부분]

- `신선`, `청정`, `푸른` 행의 아이콘 조합은 큰 병과 작은 병이 정확히 몇 개씩인지 화면 해상도상 다소 겹쳐 보여 세부 개수 판독에 약간의 불확실성이 있습니다.
- `초록` 행이 완전히 빈칸인지, 아이콘이 매우 작게 표시되어 잘 안 보이는 것인지는 이미지상 다소 अस्प명합니다.
- 표 아래의 `( 신선, 청정, 초록, 푸른 ) 목장` 문구는 보기 목록인지, 해설용 표기인지 배치상 약간 애매합니다.
- 그러나 전체 구조와 범례, 문제 의도는 비교적 명확하게 읽힙니다.
