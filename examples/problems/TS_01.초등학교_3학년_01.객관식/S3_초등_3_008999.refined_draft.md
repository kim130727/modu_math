[문제 개요]
- 문제 번호로 보이는 `37.`이 상단 왼쪽에 있음.
- 문제 지시문은 한 줄 문장으로 보이며, 내용은 `네 각이 모두 직각인 사각형을 모두 찾아 기호를 선택하세요.`임.
- 가운데에는 연한 파란색 격자 배경 위에 도형 4개가 가로로 배치되어 있음.
- 도형 아래에는 괄호로 묶인 표기 `（ 가, 나, 다, 라 ）`가 가운데 정렬에 가깝게 있음.
- 하단에는 `(정답)`, `(해설)`이 이미 인쇄된 상태로 보이며, 그 뒤에 텍스트가 이어짐.

[원본에 보이는 요소]
- 상단 문장: `37. 네 각이 모두 직각인 사각형을 모두 찾아 기호를 선택하세요.`
- 그림 속 도형 내부 라벨: `가`, `나`, `다`, `라`
- 그림 아래 표기: `（ 가, 나, 다, 라 ）`
- 하단 정답 표기: `(정답)가, 나`
- 하단 해설 표기: `(해설)네 각이 모두 직각인 사각형은 직사각형과 정사각형입니다. 따라서 가와 나입니다.`

[수식/관계 구조]
- 등식, 분수, 계산식은 보이지 않음.
- 문제의 관계 구조는 “조건을 만족하는 도형을 고르기” 형태임.
- 비교 대상은 4개의 도형이며, 각 도형에 대응하는 기호 `가/나/다/라`를 선택하는 구조로 보임.
- 선택지처럼 보이는 `（ 가, 나, 다, 라 ）`는 라벨 나열 또는 선택 대상 목록 역할을 함.

[등호별 변환 의미]
- 해당 없음.
- 등호 연쇄나 식 변형이 보이지 않음.

[빈칸 정책]
- 이 원본에는 빈칸이 별도로 보이지 않음.
- 만약 DSL에서 선택 결과를 입력해야 하는 칸이 생기더라도, 원본에 정답이 인쇄되어 있지 않다면 추론값을 넣지 말고 빈칸으로 유지해야 함.
- 원본에 `(정답)`, `(해설)`이 인쇄되어 보이는 경우에는 그 텍스트는 “보이는 사실”로만 기록하고, 문제 풀이를 통해 새로 채워 넣지 않음.

[반복 패턴]
- 도형 4개가 모두 한 줄로 배열되어 있으며, 각 도형 내부에 한 글자 라벨이 1개씩 들어 있음.
- 격자 배경이 전체 그림 영역에 반복적으로 깔려 있어, 변의 직각 여부나 기울기를 비교하기 쉽게 만든 구성으로 보임.
- 아래의 `（ 가, 나, 다, 라 ）` 표기는 도형 라벨을 다시 한 번 순서대로 반복하는 형태임.
- 하단의 `(정답)`, `(해설)` 형식은 다른 문제에서도 자주 쓰이는 고정 패턴으로 보임.

[DSL 작성 힌트]
- 문제 상단 텍스트와 그림 영역, 하단 정답/해설 영역을 세 구역으로 분리해 배치하는 것이 적절함.
- 그림 영역은 연한 파란색 격자 사각형 배경 위에 도형 4개를 좌→우 순서로 놓는 구조가 자연스러움.
- 각 도형은 윤곽선만 있는 도형으로 표현하고, 내부 중앙에 각각 `가`, `나`, `다`, `라` 라벨을 넣는 방식이 적합함.
- `가`는 가로로 긴 직사각형처럼 보이고, `나`는 더 큰 직사각형처럼 보이며, `다`는 윗변이 기울어진 좁은 사각형 계열로 보이고, `라`는 사다리꼴처럼 보임.
- 아래의 `（ 가, 나, 다, 라 ）`는 별도 텍스트 객체로 중앙 하단에 배치하는 것이 좋음.
- `(정답)`, `(해설)`은 그림과 분리된 일반 텍스트 블록으로 하단에 배치해야 함.
- 격자선은 옅은 파란색의 반복 선으로 표현하고, 도형 테두리는 더 진한 회색 또는 검정 계열 선으로 표현하면 원본에 가깝게 보임.
- 도형들의 정확한 기울기나 비율은 원근/해상도 영향이 있어, 대략적인 상대 위치와 형태를 우선 반영하는 편이 좋음.

[semantic/solvable 힌트]
- 문제는 “네 각이 모두 직각인 사각형”을 찾는 판별형 문항으로 보임.
- 보이는 해설 문구에 따르면 해당 조건은 직사각형과 정사각형에 해당한다고 적혀 있음.
- 다만 이 항목은 원본에 인쇄된 해설을 관찰한 사실로 적는 것이며, 별도 풀이로 새 결론을 만드는 것은 아님.
- 선택 대상은 `가, 나, 다, 라` 네 개이며, 실제 선택 결과는 원본 하단의 `(정답)가, 나`로 인쇄되어 보임.

[주의 및 TODO]
- `다` 도형의 윗변 기울기와 전체 사각형 성질은 이미지 해상도상 다소 흐릴 수 있으므로, 세부 기울기는 과도하게 단정하지 말 것.
- `라` 도형은 사다리꼴처럼 보이지만, 정확한 평행 관계는 확대 없이는 완전히 확정하기 어려울 수 있음.
- `（ 가, 나, 다, 라 ）`는 선택지인지 단순 열거인지 문맥상 선택지처럼 보이지만, 구성상 표기 자체를 그대로 옮기는 것이 안전함.
- 정답인 `가, 나`는 이미지에 실제로 보이므로 관찰 사실로만 기록하고, 풀이 과정에서 새로 추론해 채워 넣지 말 것.
- 빈칸이 없는 문제로 보이므로, DSL 변환 시에도 추정값 삽입 없이 원본 텍스트와 레이아웃을 우선 반영해야 함.
