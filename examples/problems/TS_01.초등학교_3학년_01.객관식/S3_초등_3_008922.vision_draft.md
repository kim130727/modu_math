- 본문 아래에는 **“월별 휴대전화 판매량”**이라는 제목의 그래프 표가 중앙에 배치되어 있습니다.
- 표는 왼쪽에 `월`, 오른쪽에 `판매량` 열로 나뉘며, 각 행에 `9월`, `10월`, `11월`, `12월`이 적혀 있습니다.
- 각 월의 판매량은 숫자 대신 **휴대전화 그림 아이콘**으로 표시되어 있습니다.
- 표 오른쪽에는 아이콘 범례가 따로 있어, 큰 휴대전화 그림 옆에 `10대`, 작은 휴대전화 그림 옆에 `1대`라고 적혀 있습니다.
- 표 아래쪽에는 보기 ①~④가 가로로 배치되어 있고, 각 보기는 월의 순서를 나열한 문장입니다.
- 왼쪽 아래에는 `(정답 ④)`와 `(해설)`이 따로 표시되어 있습니다.
- 해설 문장 옆에도 큰 휴대전화 그림 아이콘이 함께 있습니다.

[반복되는 시각 패턴]
- 각 월의 판매량은 **휴대전화 그림 여러 개의 반복**으로 나타납니다.
- 그림 크기가 두 종류로 반복되며, 범례를 통해 의미가 구분됩니다.
  - 큰 그림: `10대`
  - 작은 그림: `1대`
- 각 월 행마다 큰 그림과 작은 그림이 조합되어 있고, 그 조합 방식이 월별로 다릅니다.
- 보기 ①~④는 모두 같은 형식으로, `월 이름 4개를 쉼표로 나열`한 패턴입니다.
- 정답 표시와 해설은 문제 본문과 분리되어 하단에 반복적으로 배치되어 있습니다.

[세부 내용]
- 표는 **테두리가 있는 직사각형 구조**이며, 왼쪽 열은 월 이름, 오른쪽 넓은 열은 판매량 그림이 들어갑니다.
- 각 행은 가로줄로 구분되어 있고, 첫 행은 제목 행, 그 아래로 월별 행이 이어집니다.
- `9월` 행:
  - 큰 휴대전화 그림과 작은 휴대전화 그림이 보입니다.
  - 그림 개수는 월별 판매량의 조합을 나타냅니다.
- `10월` 행:
  - 큰 그림이 여러 개 연속으로 있고, 뒤쪽에 작은 그림들이 이어집니다.
- `11월` 행:
  - 큰 그림이 가장 많이 보이는 편이며, 뒤에 작은 그림이 붙어 있습니다.
- `12월` 행:
  - 큰 그림 몇 개 뒤에 작은 그림들이 연속으로 이어집니다.
- 오른쪽 범례는 표와 분리되어 있으며, 위아래로 두 줄 배치입니다.
  - 위쪽: 큰 휴대전화 아이콘 + `10대`
  - 아래쪽: 작은 휴대전화 아이콘 + `1대`
- 보기 번호 `①, ②, ③, ④`는 파란색 동그라미 숫자처럼 표시되어 있고, 각 선택지는 검은 글씨로 월 순서를 적었습니다.
- 하단 해설에서는 큰 휴대전화 그림 아이콘이 문장 앞에 놓여 있어, 특정 그림의 값을 설명하려는 의도로 보입니다.

[Python DSL 작성 힌트]
- 문제는 **그림그래프 읽기 문제**이므로, DSL에서는 다음 요소를 분리해서 기록하면 좋습니다.
  - 문제 지문 텍스트
  - 표 제목
  - 표의 열 이름
  - 월별 행 라벨
  - 각 행에 들어간 아이콘의 종류와 배열
  - 범례(큰 아이콘=10대, 작은 아이콘=1대)
  - 보기 ①~④의 텍스트
  - 정답 및 해설 영역
- 표 영역은 `좌측 라벨 열 + 우측 그림 열` 구조로 처리하면 좋습니다.
- 각 월별 판매량은 **아이콘 개수와 크기 구분**으로 표현되어 있으므로, 아이콘을 단순 반복 요소로 모델링하면 좋습니다.
- 보기 영역은 4개 선택지를 별도 그룹으로 묶어 배치 정보까지 기록하는 것이 좋습니다.
- 해설 문장에는 빈칸이나 수식이 없고, 서술형 문장과 아이콘만 있으므로 텍스트+이미지 혼합 구조로 다루면 됩니다.

[주의할 점]
- 그림 개수를 실제 숫자로 환산해 **정답 순서를 직접 계산해서 적지 마세요**.
- 범례가 존재하므로, 큰 그림과 작은 그림의 의미를 임의로 바꾸지 마세요.
- `정답 ④` 표시는 화면에 실제로 보이지만, Python DSL 작성 시에는 문제 본문과 정답 영역을 구분해 기록하는 것이 좋습니다.
- 해설 문장의 첫부분은 아이콘 때문에 일부 가려지거나 시각적으로 붙어 보일 수 있으니, 텍스트를 임의로 보완하지 마세요.
- 각 월 행의 아이콘 배치는 정확히 같지 않으므로, 월별로 따로 기록하는 것이 좋습니다.

[불확실한 부분]
- 각 월 행의 아이콘 개수는 이미지 해상도와 겹침 때문에 세밀하게 셀 때 오차가 있을 수 있습니다.
