[문제 개요]
- 문제 번호 `69.`가 보인다.
- 문장형 질문: `체리 상자와 소고기 팩에는 각각 어떤 단위가 표시되어 있는지 선택해보세요.`
- 주제는 무게의 단위 판별이다.
- 아래에 그림 2개가 좌우로 배치되어 있고, 각 그림에 단위 표기가 함께 있다.
- 하단에는 선택형 답안 문장과 `(정답)`, `(해설)` 텍스트가 인쇄되어 있다.

[원본에 보이는 요소]
- 왼쪽 그림: 체리 상자.
  - 상자 앞면에 `체리`, `1 kg` 표기가 보인다.
  - 상자 왼쪽에 원형 말풍선이 있고 `1 kg`이 적혀 있다.
  - 말풍선 꼬리가 상자를 가리킨다.
- 오른쪽 그림: 소고기 팩.
  - 팩 오른쪽 아래에 `600 g` 표기가 보인다.
  - 팩 오른쪽에 원형 말풍선이 있고 `600 g`이 적혀 있다.
  - 말풍선 꼬리가 팩을 가리킨다.
- 오른쪽 상단 쪽에 아이 그림이 있고, 말풍선 안에 단위 설명 문장이 있다.
  - `무게의 단위에는 킬로그램과 그램 등이 있어.`
- 중앙 하단 선택형 문장:
  - `체리 상자 → ( kg, g ), 소고기 팩 → ( kg, g )`
- 하단 추가 텍스트:
  - `(정답)kg, g`
  - `(해설)체리 상자에 1 kg이 쓰여 있으므로 표시된 단위는 kg, 소고기 팩에 600 g이 쓰여 있으므로 표시된 단위는 g입니다.`

[수식/관계 구조]
- 등호가 포함된 계산식은 보이지 않는다.
- 수식 변형 단계도 없다.
- 이 문제는 계산이 아니라, 각 물체에 적힌 숫자+단위를 보고 해당 단위를 고르는 구조이다.
- 관계 구조:
  - 체리 상자에 `1 kg` 표기 → 단위는 `kg`
  - 소고기 팩에 `600 g` 표기 → 단위는 `g`

[등호별 변환 의미]
- 해당 없음.
- 등호 연쇄나 변형식이 제시되지 않았다.

[빈칸 정책]
- 선택형 괄호 `( kg, g )`는 빈칸 성격의 응답 자리로 유지한다.
- 정답을 추론해서 빈칸에 미리 채워 넣지 않는다.
- DSL 작성 시에도 빈칸은 빈칸 상태로 표현하고, 관찰된 텍스트와 분리해서 다룬다.
- 원본에 인쇄된 `(정답)kg, g`는 보이는 사실로만 기록하되, 문제 본문 빈칸에 대입하는 방식으로 처리하지 않는다.

[반복 패턴]
- 대상 2개가 좌우로 나란히 배치되는 반복 구조이다.
- 각 대상마다:
  - 그림
  - 말풍선 형태의 단위 표시
  - 그림 자체의 작은 단위 라벨
  이 함께 나타난다.
- 하단 답안 문장에서도 두 대상에 대해 동일한 괄호 구조 `( kg, g )`가 반복된다.
- 무게 단위 예시가 말풍선 속 설명 문장으로 한 번 제시된다.

[DSL 작성 힌트]
- 상단 문제 문장은 하나의 텍스트 블록으로 두되, 줄바꿈 가능성을 고려해 1~2줄 레이아웃을 허용하는 편이 좋다.
- 왼쪽/오른쪽 대상은 각각 별도 그룹으로 묶는다.
- 왼쪽 그룹 요소:
  - 체리 상자 그림
  - 왼쪽 원형 말풍선 `1 kg`
  - 상자 앞면의 `체리`, `1 kg` 라벨
- 오른쪽 그룹 요소:
  - 소고기 팩 그림
  - 오른쪽 원형 말풍선 `600 g`
  - 팩의 작은 `600 g` 라벨
- 중앙 하단의 `체리 상자 → ( kg, g ), 소고기 팩 → ( kg, g )`는 한 줄 문장으로 배치한다.
- `→` 화살표와 괄호 구조를 그대로 유지하는 것이 좋다.
- `(정답)`, `(해설)`은 문제 본문과 분리된 하단 보조 텍스트 블록으로 처리한다.
- 말풍선 꼬리의 정확한 점 위치는 세부 렌더링 힌트로 남기되, 대상 그림을 향하는 연결 요소로 인식한다.

[semantic/solvable 힌트]
- 문제의 의미는 각 물체에 적힌 단위를 식별하는 것이다.
- 체리 상자와 소고기 팩은 서로 다른 단위 표기 예시를 보여 준다.
- 말풍선 문장은 무게 단위의 예시 설명 역할을 한다.
- 원본에 해설이 인쇄되어 있지만, 이는 풀이 설명이지 문제 요구 자체는 아니다.
- 정답 값이 화면에 보이더라도, 빈칸 작성용으로 재사용하지 말고 관찰 사실과 분리한다.

- 말풍선 꼬리의 연결점은 스타일상 정확 좌표를 단정하지 말고, 대상 방향 연결로만 기록한다.
