[문제 개요]
- 문제 번호는 `35.`이다.
- 주제는 **그림그래프(아이콘 그래프)** 이며, 미소네 학교 3학년 학생들의 **태어난 계절**을 조사한 결과를 묻는 문제이다.
- 질문은 `여름에 태어난 학생은 겨울에 태어난 학생보다 몇 명 더 많은가요?`이다.
- 원본 이미지 하단에는 `(정답) ④`와 `(해설)` 문장이 함께 보인다.
- 이 refined draft는 **문제 본문과 그림 구조를 DSL로 옮기기 쉽게 정리**하는 목적이다.

[원본에 보이는 요소]
- 상단 왼쪽에 `35.`가 보인다.
- 문제 지문은 상단에 2줄로 배치되어 있다.
  - `미소네 학교 3학년 학생들이 태어난 계절을 조사하여 나타낸 그림그래프입니다.`
  - `여름에 태어난 학생은 겨울에 태어난 학생보다 몇 명 더 많은가요?`
- 중앙에 큰 표 형태의 그림그래프가 있다.
- 표의 제목은 `학생들이 태어난 계절별 학생 수`이다.
- 표 머리글은 `계절`, `학생 수`이다.
- 왼쪽 행 이름은 위에서부터 `봄`, `여름`, `가을`, `겨울`이다.
- 오른쪽에는 범례가 세로로 배치되어 있다.
  - 큰 얼굴 그림 `: 10명`
  - 중간 얼굴 그림 `: 5명`
  - 작은 얼굴 그림 `: 1명`
- 표 아래에는 객관식 보기 5개가 흩어져 배치되어 있다.
  - `① 15명`
  - `② 16명`
  - `③ 17명`
  - `④ 18명`
  - `⑤ 19명`
- 하단에는 `(정답) ④`가 보인다.
- 하단 해설에는 다음 문장이 보인다.
  - `여름에 태어난 학생은 47명이고, 겨울에 태어난 학생은 29명입니다. 따라서 여름에 태어난 학생은 겨울에 태어난 학생보다 47-29=18(명) 더 많습니다.`

[수식/관계 구조]
- 문제 본문 자체에는 별도의 수식이 없다.
- 표는 **계절별 학생 수를 아이콘 개수와 크기 조합으로 나타내는 구조**이다.
- 범례에 따라 각 아이콘 크기의 의미가 대응된다.
  - 큰 얼굴 = 10명
  - 중간 얼굴 = 5명
  - 작은 얼굴 = 1명
- 해설에는 하나의 등식이 직접 제시된다.
  - `47-29=18(명)`
- 이 등식은 여름 학생 수와 겨울 학생 수의 차이를 나타낸다.
- 그림그래프의 각 행은 아이콘들의 반복 배치로 구성되어 있으며, 각 행의 총합을 읽어 비교하는 구조이다.

[등호별 변환 의미]
- `47-29=18(명)`
  - 왼쪽의 `47-29`는 여름 학생 수에서 겨울 학생 수를 뺀 식이다.
  - 오른쪽의 `18(명)`은 그 차이의 결과를 뜻한다.
  - 해설 문장에서는 이 값을 이용해 `여름에 태어난 학생은 겨울에 태어난 학생보다 18명 더 많다`는 의미를 설명한다.
- 문제 본문에는 다른 등호 연쇄나 단계적 변형은 보이지 않는다.
- 따라서 등호별 변화 설명은 해설식 1개에만 한정된다.

[빈칸 정책]
- 원본 문제에는 입력용 빈칸이 보이지 않는다.
- 보기형 문제이므로, 빈칸에 값을 채우는 형식이 아니다.
- DSL 작성 시에도 **추론값을 빈칸에 자동 삽입하지 말고**, 원본에 보이는 텍스트와 도형만 유지한다.
- 해설의 결과값 `18(명)`은 원본에 인쇄되어 보이는 사실로만 취급한다.
- 보기가 있는 경우에도 정답을 본문 DSL 요소로 강제 삽입하지 않는다.

[반복 패턴]
- 표의 각 계절 행은 동일한 구조를 반복한다.
  - 왼쪽: 계절 이름 텍스트
  - 오른쪽: 얼굴 아이콘들의 가로 나열
- 아이콘은 크기별로 반복 사용된다.
  - 큰 얼굴 아이콘
  - 중간 얼굴 아이콘
  - 작은 얼굴 아이콘
- 같은 아이콘이 여러 번 연속 배치되어 수량을 나타낸다.
- 각 행은 수평선으로 구분된 표 셀 안에 배치되어 있어, 행별 비교가 쉽다.
- 범례도 아이콘 + 설명 텍스트의 3줄 반복 구조이다.
- 객관식 보기도 `①~⑤` 번호 아이콘과 `명` 단위 텍스트가 반복된다.

[DSL 작성 힌트]
- 전체 화면 상단에 문제 번호와 지문을 먼저 배치하고, 그 아래에 그림그래프 표를 크게 두는 구성이 적절하다.
- 표는 `제목 영역 + 머리글 행 + 4개 계절 행`으로 분리해서 표현하면 좋다.
- 표 내부는 2열 구조로 이해하면 된다.
  - 왼쪽 열: `계절`
  - 오른쪽 열: `학생 수`를 아이콘으로 표현
- 표 오른쪽의 범례는 표와 같은 높이 선상에 두고, 세로로 3줄 정렬하는 것이 자연스럽다.
- 아이콘은 크기별로 별도 컴포넌트로 취급하고, 행마다 반복 배치로 표현하는 것이 좋다.
- 보기 영역은 표 아래쪽에 2~3열 정도로 분산 배치된 형태를 반영한다.
- `(정답) ④`와 해설은 본문과 분리된 하단 편집 요소로 두는 것이 안전하다.
- 정렬 관점에서는 표 제목이 표 상단 중앙에 있고, 범례는 표 오른쪽 바깥에 위치한다.
- 셀 경계선, 행 높이, 열 너비, 보기 간격을 가능한 한 유지하는 편이 좋다.

[semantic/solvable 힌트]
- 이 문제는 그림그래프에서 **두 계절의 학생 수 차이**를 묻는 문제이다.
- 풀이 관점에서는 범례를 사용해 각 행의 총합을 읽고, 여름과 겨울의 차이를 비교한다.
- 다만 이 문서는 풀이를 새로 전개하는 목적이 아니므로, 계산 과정을 확장해서 작성하지 않는다.
- 해설에 이미 `47명`, `29명`, `18명`이 인쇄되어 보인다.
- 정답 표시는 원본에 `④`로 보인다.
- 문제 유형상 표 해석과 덧셈/뺄셈 비교가 핵심이지만, 여기서는 **문항 구조 분석**에 집중한다.

[주의 및 TODO]
- 표 안의 각 행 아이콘 개수는 이미지 해상도상 세밀하게 세어야 하므로, DSL 작성 전 원본 PNG 기준 재확인이 필요하다.
- 특히 작은 아이콘은 간격이 좁아 보일 수 있어 개수 판독에 주의가 필요하다.
- 본문에 적힌 범례(10명, 5명, 1명)와 각 행의 총합이 일치하는지 별도로 검증할 수 있으나, 이 단계에서 임의 보정은 하지 않는다.
- 원본에는 `(정답) ④`가 보이지만, 본문 문제 자체에 정답을 넣는 형식으로 재작성하지는 않는다.
- 표 제목의 글꼴, 굵기, 중앙 정렬 정도는 이미지 기준으로 최대한 맞추되, 세부 폰트는 TODO로 남겨도 된다.
- 보기의 배치가 정확히 2줄인지 3분산인지 애매할 수 있으므로, 레이아웃은 원본 배치를 기준으로 재현한다.
- 필요 시 다시 확인할 항목:
  - 각 행의 아이콘 정확한 개수
  - 각 아이콘의 실제 크기 구분
  - 보기 위치의 정확한 좌표 관계
