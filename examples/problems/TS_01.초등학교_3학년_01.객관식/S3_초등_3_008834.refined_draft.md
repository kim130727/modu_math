[문제 개요]
- 초등 수학/과학형 **비교 판단 문제** 화면이다.
- 주어진 저울 그림을 보고, 아래 문장이 바르게 비교한 내용인지 판단하는 형식이다.
- 화면은 크게 **상단 지시문**, **중앙 저울 그림**, **하단 판단 문장**, **맨 아래 정답/해설**로 나뉜다.

[원본에 보이는 요소]
- 문제 번호: `72.`
- 상단 지시문:
  - `저울을 사용하여 감자와 고구마의 무게를 비교했습니다.`
  - `바르게 비교하였으면 ○표, 그렇지 않으면 ×표를 선택하세요.`
- 그림 위 라벨:
  - `감자`
  - `고구마`
- 중앙 그림:
  - 양팔저울
  - 왼쪽 접시의 `감자`
  - 오른쪽 접시의 `고구마`
- 그림 아래 판단 문장:
  - `고구마가 감자보다 더 무겁습니다. ( ○, × )`
- 하단 표기:
  - `(정답)○`
  - `(해설)고구마를 올려놓은 접시가 아래로 내려갔으므로 고구마가 감자보다 더 무겁습니다.`

[수식/관계 구조]
- 등호, 계산식, 변형식은 보이지 않는다.
- 대신 **비교 관계**가 표현되어 있다.
- 그림에서 드러나는 관계:
  - 왼쪽 접시: 감자
  - 오른쪽 접시: 고구마
  - 오른쪽 접시가 더 아래로 내려가 있음
- 문장으로 제시된 관계:
  - `고구마가 감자보다 더 무겁습니다.`
- 이 문제는 수식 변형이 아니라 **그림 관찰 → 비교 판단** 구조이다.

[등호별 변환 의미]
- 해당 없음.
- 등호 연쇄나 식 변환이 제시되지 않았다.

[빈칸 정책]
- 빈칸을 추론해서 채우지 않는다.
- ` ( ○, × ) `는 실제 선택지로 보이는 요소이므로, 임의로 특정 기호를 넣어 확정하지 않는다.
- 원본에 `(정답)○`가 인쇄되어 보이지만, DSL 작성 시에는 **문제 본문과 분리된 관찰 요소**로만 다루고, 별도 정답 생성용으로 사용하지 않는다.
- 원본에 보이는 텍스트는 그대로 유지하고, 보이지 않는 값은 비워 둔다.

[반복 패턴]
- 선택형 표기 패턴:
  - 문장 끝에 `(` `○` `,` `×` `)` 형태의 둥근 괄호 선택지가 붙는다.
- 라벨 패턴:
  - 물체 이름이 그림 위쪽에 작은 박스 형태로 각각 따로 배치되어 있다.
  - `감자`, `고구마`가 서로 분리된 라벨로 반복된다.
- 하단 안내 패턴:
  - `(정답)`과 `(해설)`이 괄호를 포함한 접두 표기로 반복된다.

[DSL 작성 힌트]
- 레이아웃은 다음 4영역으로 분리하면 좋다.
  1. 상단 문제 번호 + 지시문
  2. 중앙 저울 그림
  3. 중앙 하단의 판단 문장
  4. 최하단 정답/해설 텍스트
- 저울 그림은 구성 요소를 분해해서 표현하기 좋다.
  - 받침대
  - 중심 기둥
  - 좌우 팔
  - 좌우 접시
  - 접시 위의 물체 2개
  - 물체 위의 라벨 2개
- 렌더링 관점에서 중요한 점:
  - 오른쪽 접시가 왼쪽보다 아래로 기울어진 상태
  - 물체 라벨은 각각 해당 물체의 위쪽에 짧게 붙음
  - 하단 판단 문장은 그림 아래 중앙 정렬에 가깝게 배치됨
  - `( ○, × )`는 문장 끝에 붙는 작은 선택 영역으로 처리
- 정답/해설은 본문과 스타일을 분리해 아래쪽에 별도 텍스트 블록으로 두는 것이 좋다.

[semantic/solvable 힌트]
- 보이는 사실:
  - 저울의 오른쪽 접시가 아래로 내려가 있다.
  - 오른쪽에는 고구마, 왼쪽에는 감자가 놓여 있다.
  - 화면에는 `고구마가 감자보다 더 무겁습니다.`라는 판단 문장이 이미 인쇄되어 있다.
- 추론되는 수학적 의미:
  - 저울의 기울기 방향은 오른쪽 물체가 더 무겁다는 의미로 읽힐 수 있다.
- 그러나 DSL 작성 목적에서는 이 의미를 **설명용 메타정보**로만 남기고, 정답 자체를 새로 계산하지 않는다.
- 하단 `(정답)○`는 원본에 인쇄된 관찰 사실로만 취급한다.
