[문제 개요]
- 객관식 문제.
- 문제 번호는 `36.`.
- 문장: `그림을 보고 색칠한 부분을 분수로 바르게 나타낸 것을 고르세요.`
- 중앙에 도형 묶음 그림이 있고, 아래에 보기 5개가 배치되어 있다.
- 왼쪽 아래에 `(정답 ④)`가 인쇄되어 보인다.
- 맨 아래에 해설 문장이 인쇄되어 보인다.

[원본에 보이는 요소]
- 상단 왼쪽에 문제 번호 `36.`과 문제 문장이 한 줄로 보인다.
- 중앙에는 연한 파란색의 직사각형 테두리 안에 마름모 모양 도형들이 배열되어 있다.
- 마름모는 같은 크기이며, 일부는 주황색으로 색칠되어 있고 나머지는 윤곽선만 보인다.
- 보기 5개가 각각 동그라미 번호와 분수로 제시되어 있다.
- 보기 내용:
  - `① 1/24`
  - `② 1/12`
  - `③ 1/6`
  - `④ 1/4`
  - `⑤ 1/3`
- 왼쪽 아래에 `(정답 ④)`가 보인다.
- 해설 문장: `(해설)24를 똑같이 6씩 4묶음으로 나누면 1묶음은 전체의 1/4입니다.`
- 해설의 마지막 분수 `1/4`는 인쇄되어 보인다.

[수식/관계 구조]
- 중앙 그림 자체에는 등호로 이어지는 수식 전개식은 보이지 않는다.
- 해설 문장에는 다음 관계가 말로 제시되어 있다.
  - `24`를 `6씩 4묶음`으로 나눈다.
  - `1묶음`은 전체의 `1/4`이다.
- 이는 분할-전체 관계를 설명하는 문장형 정보이다.
- 분수 표기 `1/24`, `1/12`, `1/6`, `1/4`, `1/3`는 보기 영역에만 나타난다.

[등호별 변환 의미]
- 해당 없음.
- 이미지에 `=`를 포함한 변환 연쇄는 보이지 않는다.
- 따라서 단계별 등호 변환 설명은 필요하지 않다.

[빈칸 정책]
- 이 문항에는 직접적인 빈칸 입력란이 보이지 않는다.
- 다만 DSL 작성 시 정답을 추론해서 채워 넣지 말고, 보이는 텍스트와 도형 배치만 기록한다.
- 만약 내부 파싱에서 빈칸처럼 처리해야 하는 영역이 있더라도, 추정값은 넣지 않고 원형 그대로 유지한다.
- 보이지 않는 정보는 비워 두고 TODO로 남긴다.

[반복 패턴]
- 중앙 도형은 같은 모양의 마름모가 규칙적으로 반복된 배열이다.
- 마름모는 가로 방향으로 여러 개가 나열되고, 세로로도 여러 줄 반복된다.
- 각 줄의 왼쪽 부분에 주황색으로 칠해진 마름모가 연속으로 보이고, 나머지는 흰색 내부의 윤곽선 마름모이다.
- 반복 배치는 격자형에 가깝고, 각 행이 거의 같은 간격과 형태를 가진다.
- 보기 번호는 파란색 원형 마커 안의 숫자로 표시되어 반복된다.

[DSL 작성 힌트]
- 중앙 그림은 하나의 큰 직사각형 박스 안에 여러 개의 동일한 마름모를 반복 배치한 구조로 보면 된다.
- 박스는 연한 파란색 테두리로 표시되어 있다.
- 마름모는 `diamond` 또는 회전된 정사각형 계열 도형으로 표현하는 것이 자연스럽다.
- 색칠된 마름모와 비색칠 마름모를 서로 다른 스타일로 구분해야 한다.
- 행과 열의 반복을 살려서 좌표 기반 배열로 표현하면 좋다.
- 보기 영역은 각 항목마다
  - 파란색 원형 번호
  - 분수 텍스트
  를 한 묶음으로 다루면 편하다.
- `정답 ④` 표시는 문제 데이터의 메타 정보로 분리하는 것이 좋다.
- 해설은 본문과 분리된 텍스트 블록으로 기록한다.
- 렌더링 관점에서는 도형 박스가 보기 영역보다 위쪽 중앙에 크게 배치되고, 보기들은 아래쪽에 분산 배치된다.

[semantic/solvable 힌트]
- 보이는 사실만으로는 중앙 도형의 총 개수와 색칠 개수를 엄밀히 재단하기보다, 문제 의도상 “전체에서 색칠된 부분이 차지하는 분수”를 묻는 구조로 이해할 수 있다.
- 해설 문장은 `24를 똑같이 6씩 4묶음으로 나누면`이라는 식으로 분할을 설명한다.
- 이 해설에 따르면 전체를 같은 크기의 묶음으로 나누는 관계가 핵심이다.
- 단, 문제 풀이를 새로 계산해 답을 확정하는 것은 여기서 하지 않는다.
- 인쇄된 정답 표시는 관찰 사실로만 기록한다: `(정답 ④)`.
- 해설 끝의 `1/4`도 인쇄된 텍스트로만 기록한다.

[주의 및 TODO]
- 중앙 마름모의 정확한 총 개수는 해상도상 세부 확인이 필요하다.
- 각 행마다 정확히 몇 개가 있는지, 오른쪽 끝 도형이 온전히 보이는지 여부는 추가 확인이 필요하다.
- 색칠된 마름모의 정확한 개수는 보이는 범위에서만 기록하고, 답으로 단정하지 않는다.
- 해설 문장은 OCR 결과와 이미지가 약간 다를 수 있으므로, 최종 DSL 입력 전 원본 이미지 기준으로 재검토한다.
- `나누기 n은 분모의 ×n으로 반영된다.` 는 이 이미지에는 직접적인 분수 나눗셈 변환식이 보이지 않아 적용할 장면이 없다.
