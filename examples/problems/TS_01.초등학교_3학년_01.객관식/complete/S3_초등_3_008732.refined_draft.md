[문제 개요]
- 문제 번호는 `71.`이다.
- 지시문은 `사다리를 타고 내려가 도착한 곳이 참이면 ○표, 거짓이면 ×표 하세요.`로 보인다.
- 화면에는 분수 3개가 제시되고, 각각이 아래의 분류 라벨과 연결된 사다리형 도식이 함께 보인다.
- 상단에는 문제 본문형 도식, 하단에는 같은 내용을 다시 그린 보조 도식이 함께 배치되어 있다.
- 이 문항은 계산문이라기보다 **분수의 분류를 판단하는 표시 문제**로 보인다.

[원본에 보이는 요소]
- 상단 분수 3개:
  - `2/8`
  - `9/10`
  - `4/4`
- 상단 분수 아래 라벨 3개:
  - `가분수`
  - `가분수`
  - `진분수`
- 각 라벨 아래 선택 기호:
  - `(○, ×)`
- 본문 중간의 표기:
  - `(정답) ○, ×, ○`
- 본문 중간의 해설 표기:
  - `(해설) 2/8 → 진분수, 9/10 → 진분수, 4/4 → 가분수`
- 하단에도 동일한 분수 3개와 동일한 라벨 3개가 다시 배치되어 있다.
- 하단 도식에는 파란색 실선/점선 연결선이 보이며, 상단보다 더 강조된 경로 표시처럼 보인다.

[수식/관계 구조]
- 이 문항의 핵심 구조는 `분수 → 분류명`의 연결이다.
- 보이는 분수와 라벨의 배치는 다음처럼 읽힌다.
  - `2/8` 아래에 `가분수`
  - `9/10` 아래에 `가분수`
  - `4/4` 아래에 `진분수`
- 다만 해설 줄에는 별도의 분류 연결이 적혀 있어, 화면 내에서 **문제 배치와 해설 배치가 서로 다르게 보일 수 있다**.
- 따라서 DSL에서는 상단 문제 배치와 하단 해설 배치를 분리해서 다루는 것이 좋다.
- 등호 `=` 형태의 계산 전개는 보이지 않고, `→`를 이용한 대응 관계만 보인다.

[등호별 변환 의미]
- 화면에는 `=`로 이어진 수식 변형은 보이지 않는다.
- 대신 해설에서 `→`가 사용되어 각 분수와 판정명 사이의 대응을 나타낸다.
- 해설의 흐름은 다음과 같이 읽힌다.
  - `2/8 → 진분수`
  - `9/10 → 진분수`
  - `4/4 → 가분수`
- 여기서 `→`는 계산 변형이라기보다 **분수에서 분류명으로 이동하는 표시**로 보인다.
- 각 단계의 의미를 따로 쓰면:
  - `2/8`에서 `진분수`로 연결됨
  - `9/10`에서 `진분수`로 연결됨
  - `4/4`에서 `가분수`로 연결됨
- 주의: 위 연결은 화면에 보이는 해설 문장을 그대로 옮긴 것이며, 그 타당성을 이 답변에서 판정하지 않는다.

[빈칸 정책]
- 이 원본에는 숫자 빈칸(`__`) 같은 입력 칸은 뚜렷하게 보이지 않는다.
- 대신 `(○, ×)` 선택 영역이 반복되어 보인다.
- 선택 결과는 보이는 배치만 기록하고, 추론값으로 채워 넣지 않는다.
- 즉, **빈칸이 있다면 빈칸 그대로 유지**해야 하며, 정답을 추정해서 채우면 안 된다.
- 화면에 인쇄된 `(정답)` 표기가 있더라도, 이 답변에서는 관찰 사실만 기록하고 빈칸 정책을 바꾸지 않는다.

[반복 패턴]
- `분수 박스 3개`가 반복된다.
- `분홍색 라벨 박스 3개`가 반복된다.
- 각 라벨 아래에 `(○, ×)`가 반복된다.
- 하단에도 동일한 3개 분수와 3개 라벨이 다시 반복된다.
- 연결선은 세로선과 가로선을 조합한 사다리형 구조로 반복된다.
- 하단에서는 실선과 점선이 섞여 있어, 같은 구조를 다른 선 스타일로 재현한 것으로 보인다.

[DSL 작성 힌트]
- 상단 문제 영역은 좌→우 3열 구성으로 잡는 것이 좋다.
- 각 열은 다음 요소를 포함한다.
  - 파란 테두리의 분수 박스
  - 아래로 내려가는 세로 연결선
  - 분홍색 라벨 박스
  - 그 아래 `(○, ×)` 텍스트
- 분수 박스 내부는 위/아래 숫자와 가운데 분수선으로 구성한다.
- 연결선은 사다리처럼 중간에 가로로 이어지는 구조를 유지한다.
- 하단 도식은 상단과 같은 3열 배치이되,
  - 파란 실선
  - 파란 점선
  - 꺾임 연결
  을 구분해 표현하면 좋다.
- `(정답)`과 `(해설)`은 본문과 분리된 별도 텍스트 블록으로 배치한다.
- 해설의 `→`는 텍스트와 화살표 객체를 함께 쓰는 방식이 적절하다.
- 라벨 텍스트는 문제에 보이는 그대로 배치하고, 선택 기호는 실제 정답처럼 강조하지 않는다.

[semantic/solvable 힌트]
- 이 문항은 분수의 분류를 묻는 형식이다.
- 화면에 보이는 해설은 분수마다 판정명이 대응된다고 설명한다.
- 그러나 이 답변에서는 그 판정이 맞는지 여부를 계산으로 검증하지 않는다.
- DSL에서는 `분수`와 `분류명`의 연결 관계를 구조적으로 표현하는 데 집중하면 된다.
- 특히 상단과 하단에 같은 요소가 반복되므로, 동일한 요소군을 두 번 배치하는 구조로 이해하면 좋다.
