# [전체적인 구성]
- 화면 상단에 문제 번호 **83.** 이 있고, 그 오른쪽에 본문 지시문이 한 줄 이상에 걸쳐 배치되어 있습니다.
- 중앙에는 **두 개의 저울 그림**이 나란히 배치되어 있습니다.
  - 왼쪽 저울: **가대추**와 **바나나** 비교
  - 오른쪽 저울: **나바나나**와 **포도** 비교
- 각 저울 위쪽에는 비교 대상 이름이 **작은 상자(라벨처럼 보이는 베이지색 배경)**로 표시되어 있습니다.
- 저울 아래쪽에는 문제의 선택지 문장이 있고, 그 아래에 **(1)** 번 문항이 이어집니다.
- 맨 아래에는 **(해설)** 이라는 해설 문장이 별도로 적혀 있습니다.
- 전체적으로 **그림 + 선택형 문장 + 해설** 구조입니다.

# [보이는 텍스트]
- `83.`
- `저울로 대추, 바나나, 포도의 무게를 비교하고 있습니다. 가장 무거운 것은 무엇인지 알맞은 말을 선택해 보세요.`
- 그림 위 라벨:
  - `가`
  - `대추`
  - `바나나`
  - `나`
  - `바나나`
  - `포도`
- 문제 문장:
  - `(1) 가에서 대추와 바나나 중 더 무거운 것은 ( 대추, 바나나 )이고,`
  - `나에서 바나나와 포도 중 더 무거운 것은 ( 바나나, 포도 )입니다.`
- 해설:
  - `(해설)(1) 가에서 대추와 바나나 중 바나나를 올려놓은 접시가 내려갔으므로 더 무거운 것은 바나나이고, 나에서 바나나와 포도 중 포도를 올려놓은 접시가 내려갔으므로 더 무거운 것은 포도입니다.`
- 맨 아래 답처럼 보이는 텍스트:
  - `(1) 바나나, 포도`

# [반복되는 시각 패턴]
- 저울 그림이 **두 번 반복**됩니다.
  - 두 그림 모두 같은 형태의 저울 본체를 사용합니다.
  - 각 그림에서 양쪽 접시에 과일이 하나씩 놓여 있습니다.
  - 각 저울의 기울어진 방향을 통해 비교 결과를 나타내는 구조로 보입니다.
- 텍스트 선택지 구조도 반복됩니다.
  - `가에서 ... 중 더 무거운 것은 (   )`
  - `나에서 ... 중 더 무거운 것은 (   )`
- 각 빈칸은 **괄호 안 선택형 빈칸**처럼 보이며, 직접 쓰는 서술형 칸이 아니라 **선택지에서 고르는 자리**로 보입니다.
- 해설도 같은 판단 구조를 반복합니다.
  - 어느 쪽 접시가 내려갔는지를 근거로 더 무거운 것을 말함.

# [세부 내용]
- 왼쪽 저울:
  - 왼쪽 접시에 **대추**가 놓여 있고, 오른쪽 접시에 **바나나**가 놓여 있습니다.
  - 저울 막대가 한쪽으로 기울어져 있어, **어느 접시가 더 내려갔는지**를 시각적으로 나타냅니다.
  - 상단 라벨에서 `가`가 이 그림을 가리키는 듯합니다.
- 오른쪽 저울:
  - 왼쪽 접시에 **바나나**, 오른쪽 접시에 **포도**가 놓여 있습니다.
  - 역시 저울이 기울어진 상태로 그려져 있습니다.
  - 상단 라벨에서 `나`가 이 그림을 가리키는 듯합니다.
- 문제 본문은 두 비교 결과를 읽고,
  1. `가`에서 더 무거운 것
  2. `나`에서 더 무거운 것
  을 선택지 괄호 안에서 고르게 되어 있습니다.
- 보기의 괄호 안에는 각각 두 대상이 쉼표로 나열되어 있어, **둘 중 하나를 고르는 형식**입니다.
- 해설 문장은 그림의 **저울이 내려간 쪽**을 근거로 설명합니다.
  - 다만 실제로 어느 쪽이 더 내려갔는지는 그림 해상도와 축소 때문에 아주 세밀하게 볼 때 약간 주의가 필요합니다.

# [Python DSL 작성 힌트]
- `page` 위에 문제 번호와 본문 텍스트를 상단 정렬로 배치합니다.
- 중앙에는 `scale_left`, `scale_right` 같은 이름의 **두 개 저울 일러스트 그룹**을 좌우로 배치합니다.
- 각 저울 그룹은 다음 요소로 나눠 그리면 좋습니다.
  - 저울 받침대
  - 중앙 기둥
  - 좌우 수평 막대
  - 좌우 접시
  - 접시 위 과일 아이콘
  - 기울어짐 표현
  - 위쪽 라벨 박스(베이지색 배경)
- 라벨 박스 안 텍스트는 각각:
  - 왼쪽 그림: `가`, `대추`, `바나나`
  - 오른쪽 그림: `나`, `바나나`, `포도`
- 본문 문장은 그림 아래에 두 줄로 배치하고, 각 선택지 부분은 괄호 형태로 남겨둡니다.
- 해설은 본문 아래에 작은 글씨로 따로 배치합니다.
- 마지막 답처럼 보이는 줄은 별도 텍스트 블록으로 처리할 수 있습니다.
- 과일 아이콘은 텍스트가 아니라 **그림 요소**로 분리하는 것이 좋습니다.
- 저울의 기울기 방향은 도형 회전 또는 좌우 높이 차이로 표현합니다.

# [주의할 점]
- 답을 확정해서 설명하지 마세요. 이 문제는 시각 분석 초안이므로 **정답을 풀이 형태로 단정하지 않는 것**이 중요합니다.
- `가`, `나`는 사람 이름이 아니라 **각 저울 그림을 구분하는 기호**로 보입니다.
- `대추`, `바나나`, `포도`는 상단 라벨과 그림의 물체를 연결해서 읽어야 합니다.
- 보기 괄호 `( 대추, 바나나 )`, `( 바나나, 포도 )`는 **예시 나열이 아니라 선택지**입니다.
- 해설 문장은 화면에 실제로 적혀 있으나, DSL 작성 시에는 본문과 해설을 **구분된 블록**으로 처리하는 것이 좋습니다.
- 저울이 어느 쪽으로 더 내려갔는지는 그림에서 중요하지만, 해상도상 접시 높이 차이를 너무 단정적으로 일반화하지 않도록 주의하세요.

# [불확실한 부분]
- 저울 막대의 정확한 기울기 각도는 이미지 축소 때문에 아주 정밀하게 판독하기 어렵습니다.
- 상단의 베이지색 라벨 박스가 정확히 어떤 테두리 스타일인지(둥근 모서리인지, 사각형인지)는 세부 렌더링에 따라 다를 수 있습니다.
- 맨 아래의 `(1) 바나나, 포도`는 정답 표기처럼 보이지만, 시각 분석 단계에서는 **참고 텍스트**로만 처리하는 편이 안전합니다.
