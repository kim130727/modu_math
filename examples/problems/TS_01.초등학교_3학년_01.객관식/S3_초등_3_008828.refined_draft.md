[문제 개요]
- 1문항형 단위 선택 문제로 보인다.
- 좌상단에 문제 번호와 지시문이 있고, 중앙에는 요거트병 그림과 본문 문장이 배치되어 있다.
- 하단에는 `(정답)`과 `(해설)`이 인쇄되어 있다.
- 원본 이미지 기준으로, 정답 표시 `mL`가 **보인다**.

[원본에 보이는 요소]
- 좌상단: `□ 61. 알맞은 단위를 선택하세요.`
- 중앙 상단 쪽: 요거트병 일러스트 1개
- 중앙 본문: `요구르트병의 들이는 약 80( mL , L )입니다.`
- 하단 왼쪽: `(정답)mL`
- 하단: `(해설)요구르트병의 들이는 80 L보다 더 적으므로 80 mL가 알맞습니다.`

[수식/관계 구조]
- 수식 자체나 등호 연쇄는 보이지 않는다.
- 구조는 `숫자 80 + 괄호 안 단위 후보(mL, L)` 형태의 선택형 문장이다.
- 수학적 의미로는 “약 80인 들이의 단위를 mL 또는 L 중에서 고르는 문제”로 읽힌다.
- 다만 이는 문제 형식의 의미 해석이며, 빈칸에 값을 추론해 넣는 것은 아니다.

[등호별 변환 의미]
- 등호 기호가 없으므로 변환 단계도 없다.
- 계산식, 분수 변환, 단위 변환의 등호 연쇄는 보이지 않는다.

[빈칸 정책]
- 본문에는 정답을 써 넣는 빈칸이 아니라, 단위 후보를 고르는 괄호형 선택지가 보인다.
- 따라서 `80( mL , L )`는 빈칸으로 채우는 형식이 아니라 **선택지 표시**로 유지해야 한다.
- 이미 인쇄된 `(정답)mL`는 화면에 보이는 요소로만 기록하고, 이를 바탕으로 본문을 수정하거나 빈칸을 추론해 채우지 않는다.
- 추론으로 정답을 새로 만들거나 다른 단위를 넣지 않는다.

[반복 패턴]
- 문제 번호 앞에 작은 네모 기호 `□`가 붙는 패턴
- `숫자 + (단위 후보, 단위 후보)` 형태의 선택지 패턴
- 본문 아래에 `(정답)`과 `(해설)`이 별도 줄로 분리되는 패턴
- 그림이 본문 의미를 보조하는 단일 삽화 패턴

[DSL 작성 힌트]
- 상단 좌측의 `□ 61.`과 지시문은 한 줄 텍스트 블록으로 처리하면 좋다.
- 중앙 본문은 큰 글씨의 단일 문장으로 배치하되, `80( mL , L )` 부분의 괄호와 쉼표 간격을 원문처럼 유지하는 것이 중요하다.
- 요거트병 그림은 본문과 분리된 독립 이미지 요소로 두고, 본문 위쪽 오른편에 배치하면 유사하다.
- 하단의 `(정답)mL`와 `(해설)...`은 별도의 텍스트 블록으로 분리하는 것이 좋다.
- 문장, 그림, 정답/해설이 세로로 층분리되어 있으므로 블록 간 간격을 충분히 두는 편이 맞다.
- 선택지 `mL`, `L`는 텍스트 내부의 괄호 요소로 취급하고, 박스형 답안란으로 바꾸지 않는다.
- `□`는 체크박스처럼 보일 수 있으나, 문제 번호 앞의 기호로 취급하는 편이 안전하다.

[semantic/solvable 힌트]
- 문제의 주제는 들이의 단위 선택이다.
- 해설 문장에는 `80 L보다 더 적으므로 80 mL가 알맞습니다.`가 보인다.
- 다만 해설을 바탕으로 별도 계산을 전개하거나, 문제 본문을 재해석해 추가 값을 생성하지는 않는다.
- 원본에 이미 보이는 정답 표시는 관찰 사실로만 남기고, 풀이 결과로 확정적으로 서술하지 않는다.

[주의 및 TODO]

- 본문 선택지의 띄어쓰기 `80( mL , L )`는 인식 결과에 따라 다소 달라질 수 있으므로, 레이아웃 구현 시 괄호와 쉼표 위치를 우선 보존한다.

- 빈칸을 추론해 채우지 말고, 선택지 구조를 그대로 유지한다.
