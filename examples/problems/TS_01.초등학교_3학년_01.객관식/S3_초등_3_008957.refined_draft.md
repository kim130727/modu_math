[문제 개요]
- 문제 번호는 `27.`로 보인다.
- 문제 문장은 `가게별 일주일 동안 팔린 막대사탕 수를 조사하여 그림그래프로 나타내었습니다. 나 가게보다 막대사탕을 더 많이 판 가게를 모두 선택하세요.`로 보인다.
- 화면 중심에는 `일주일 동안 팔린 막대사탕 수`라는 제목의 그림그래프 표가 있다.
- 표를 읽고, 기준이 되는 `나` 가게보다 더 많은 막대사탕을 판 가게를 고르는 선택형 문제로 보인다.
- 하단에는 `(정답)다, 라`와 해설 문장이 인쇄되어 보인다.  
  - 정답 텍스트는 원본에 **보이는 사실**로만 기록한다.
  - 별도 계산으로 새로 확정하는 방식은 피한다.

[원본에 보이는 요소]
- 상단 왼쪽: 문제 번호 `27.`와 문제 문장.
- 중앙 상단: 연한 분홍색 배경의 제목 박스 `일주일 동안 팔린 막대사탕 수`.
- 중앙: 큰 표 1개.
  - 표는 좌우 2묶음으로 나뉜다.
  - 각 묶음은 `가게 / 막대사탕 수` 2열 구조로 보인다.
- 왼쪽 묶음의 가게 이름: `가`, `다`, `마`.
- 오른쪽 묶음의 가게 이름: `나`, `라`, `바`.
- 표 아래 오른쪽: 범례.
  - 큰 막대사탕 아이콘 옆 `100개`
  - 작은 막대사탕 아이콘 옆 `10개`
  - 아래에 `( 가, 나, 다, 라 ) 가게`
- 하단 텍스트:
  - `(정답)다, 라`
  - `(해설)나 가게에서 팔린 막대사탕 수는 240개이므로 240개보다 많은 가게를 찾습니다. 따라서 나 가게보다 막대사탕을 더 많이 판 가게는 다 가게와 라 가게입니다.`

[수식/관계 구조]
- 표의 각 행은 `가게 이름`과 `그 가게의 막대사탕 수를 그림으로 표현한 칸`으로 이루어진다.
- 수량은 두 종류의 막대사탕 그림으로 나타난다.
  - 큰 그림 1개 = `100개`
  - 작은 그림 1개 = `10개`
- 그림의 개수를 세어 가게별 수량을 비교하는 구조이다.
- 해설 문장에는 `나 가게에서 팔린 막대사탕 수는 240개`라고 적혀 있다.
  - 이 값은 원본에 인쇄된 해설 문장의 **보이는 사실**이다.
  - 별도로 재계산해서 채우지 않는다.
- 이 문제에서 핵심 관계는 `나` 가게보다 큰 수량인지 여부의 비교이다.

[등호별 변환 의미]
- 화면에 `=` 형태의 등호 연쇄 계산식은 보이지 않는다.
- 따라서 단계별 등호 변환 설명은 해당 없음.
- 다만 해설 문장에는 `240개`가 결과값처럼 제시되어 있으므로, 이는 계산 전개가 아니라 문장형 설명으로 취급한다.

[빈칸 정책]
- 원본에는 풀이 빈칸을 채우는 형식이 아니라 선택형 구조가 보인다.
- 빈칸을 추론해서 채우지 않는다.
- 이미지에서 비어 있거나 확인되지 않는 부분은 그대로 비워 두어야 한다.
- 정답 텍스트 `(정답)다, 라`는 원본에 보이지만, DSL 작성용으로는 **관찰 사실**로만 다루고, 빈칸을 이용해 새 값을 생성하지 않는다.

[반복 패턴]
- 표는 좌우 대칭에 가까운 반복 구조를 가진다.
  - 왼쪽 묶음: 3행
  - 오른쪽 묶음: 3행
- 각 행은 동일한 형식으로 반복된다.
  - 가게 이름 1개
  - 그 옆에 막대사탕 그림들
- 범례의 아이콘도 2종 반복이다.
  - 큰 아이콘
  - 작은 아이콘
- 그림 아이콘은 가로 방향으로 나열되어 수량을 나타낸다.
- 같은 종류의 사탕 그림이 연속 배치되어 있어, 반복 카운트형 시각 표현으로 보인다.

[DSL 작성 힌트]
- 큰 바깥 표를 먼저 구성하는 것이 좋다.
  - 상단 제목 박스 1개
  - 내부 표 1개
  - 좌우 2묶음 구획
- 각 묶음은 `가게` 열과 `막대사탕 수` 열로 나눈다.
- 행마다 가게 이름 텍스트와 그림 아이콘 반복을 함께 배치한다.
- 그림 아이콘은 2종으로 분리해 표현한다.
  - 큰 막대사탕 아이콘
  - 작은 막대사탕 아이콘
- 범례는 표의 오른쪽 아래에 별도 요소로 둔다.
  - 아이콘 2개와 각각의 수량 텍스트를 옆에 붙인다.
  - `( 가, 나, 다, 라 ) 가게` 문구도 별도 텍스트로 배치한다.
- 하단의 `(정답)`, `(해설)`은 본문과 분리된 텍스트 박스로 처리하는 것이 자연스럽다.
- 표 테두리와 제목 박스는 연한 분홍색 계열로 보인다.
- 셀 내부 아이콘은 왼쪽에서 오른쪽으로 정렬되어 있다.
- 좌우 묶음의 간격과 정렬을 맞춰 반복 구조가 선명하게 보이도록 배치한다.

[semantic/solvable 힌트]
- 문제의 의미는 그림그래프를 보고 특정 기준 가게 `나`보다 큰 값을 가진 가게를 고르는 것이다.
- 범례가 `100개`, `10개`로 주어져 있으므로, 각 행의 그림 개수를 세면 수량 비교가 가능하다.
- 해설 문장에 `나 가게에서 팔린 막대사탕 수는 240개`라고 보인다.
  - 이 문장은 원본에 인쇄된 설명으로만 기록한다.
  - 이 값으로 다른 수량을 새로 계산해 채우지 않는다.
- 선택 대상은 `( 가, 나, 다, 라 ) 가게` 중에서 비교되는 것으로 보인다.
- 정답 텍스트가 보이지만, DSL용 구조화에서는 정답을 새로 유도한 결과처럼 단정하지 않고 관찰 정보로만 유지하는 편이 안전하다.
