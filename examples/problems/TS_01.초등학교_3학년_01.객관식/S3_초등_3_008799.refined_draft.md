[문제 개요]
- 한 장짜리 그림 선택형 문제이다.
- 문제 번호 `28.`이 좌측 상단에 있고, 바로 뒤에 지시문 `무게가 더 무거운 것을 선택하세요.`가 이어진다.
- 본문에는 비교 대상 그림 3개가 보인다: 가위 1개, 책가방 2개.
- 좌측 아래 책가방 옆에 `(정답)` 표시가 인쇄되어 있다.
- 하단에는 `(해설) 양손으로 들었을 때 힘이 더 드는 책가방이 더 무겁습니다.`라는 문장이 보인다.
- 수식 계산 문제는 아니고, 그림을 보고 더 무거운 대상을 고르는 생활수학형 문항으로 보인다.

[원본에 보이는 요소]
- 좌측 상단:
  - 작은 빈 사각형 표시 1개
  - 문제 번호 `28.`
  - 지시문 `무게가 더 무거운 것을 선택하세요.`
- 그림 영역:
  - 화면 중앙 위쪽에 가위 그림 1개
  - 우측 상단에 책가방 그림 1개
  - 좌측 아래에 책가방 그림 1개
- 라벨:
  - 좌측 아래 책가방 왼쪽에 `(정답)` 텍스트
- 하단 텍스트:
  - `(해설)양손으로 들었을 때 힘이 더 드는 책가방이 더 무겁습니다.`
- 배경은 흰색이며, 표나 분수틀, 등호, 화살표, 계산식은 보이지 않는다.

[수식/관계 구조]
- 보이는 수식 없음.
- 관계는 다음과 같이 해석될 수 있으나, 이는 문장 의미 수준의 해석이다.
  - `더 무거운 것`을 그림들 중 하나에서 고르는 비교 관계
  - 해설 문장은 `힘이 더 드는 책가방`과 `더 무거운 책가방`을 연결하는 설명 역할
- 비교 대상은 그림 기반이며, 수치 비교나 등식 변형 구조는 없다.

[등호별 변환 의미]
- 등호 연쇄나 식 변형이 보이지 않는다.
- 따라서 단계별 변환 설명이 필요하지 않다.
- `해설`은 계산식의 변형이 아니라, 선택 기준을 설명하는 자연어 문장이다.

[빈칸 정책]
- 문제 풀이값을 추론해서 채우지 않는다.
- 빈칸, 정답란, 선택 결과가 따로 있는 형태로 보이더라도, 원본에 숫자/문자가 인쇄되어 보이지 않으면 임의로 넣지 않는다.
- 이 문항에서는 `(정답)` 라벨이 보이지만, 이것은 값이 아니라 표시 라벨로 취급한다.
- 따라서 DSL 작성 시에도 빈칸은 빈칸으로 유지하고, 보이는 라벨만 별도 텍스트 요소로 다룬다.

[반복 패턴]
- 같은 유형의 물체가 2개 반복된다:
  - 책가방 1개는 좌측 아래
  - 책가방 1개는 우측 상단
- 두 책가방은 형태가 거의 동일하게 보이며, 위치만 다르다.
- 가위는 가운데 위쪽에 단독으로 배치되어 비교 대상과 분리된다.
- `(정답)` 라벨은 특정 책가방 그림 옆에 붙는 강조 패턴으로 보인다.
- 반복 배열이라기보다, 서로 다른 위치의 독립 객체 3개가 화면에 흩어져 있는 구성이다.

[DSL 작성 힌트]
- 상단 문제문과 그림 영역, 하단 해설 영역을 분리해서 배치하는 구성이 적절하다.
- 좌표 기반 배치가 중요하다:
  - 가위는 화면 중앙 상단 쪽
  - 우측 책가방은 우측 상단 쪽
  - 좌측 책가방은 좌측 하단 쪽
- `(정답)` 텍스트는 좌측 아래 책가방 그림의 왼쪽 근처에 작은 라벨처럼 붙여 배치한다.
- 하단의 `(해설)` 문장은 별도 텍스트 블록으로 처리한다.
- 보기 박스, 선택지 번호, 선 연결, 표 구조는 보이지 않으므로 만들지 않는 것이 자연스럽다.
- 작은 빈 사각형은 체크박스처럼 보일 수 있으나 확정은 어렵다. 필요하면 단순 장식 요소로만 취급한다.

[semantic/solvable 힌트]
- 이 문제는 그림 비교형으로, 의미상 `무게가 더 무거운 것`을 선택하라는 지시문이다.
- 해설 문장은 `양손으로 들었을 때 힘이 더 드는 책가방`이 더 무겁다는 일반적 설명이다.
- 다만 답을 추론해서 서술하지 말고, 화면에 `(정답)`이 붙어 있는 위치만 관찰 사실로 기록한다.
- 수치 계산이나 연산 과정이 없으므로, DSL에서는 비교형 이미지 선택 문제로 모델링하면 된다.

[주의 및 TODO]

- 해설 문장은 화면에 인쇄된 그대로 옮겨야 하며, 임의로 다듬지 않는다.
- 가위와 책가방의 정확한 색상, 크기, 질감은 해상도 차이로 달라질 수 있으므로 세부 묘사는 과도하게 확정하지 않는다.
- 좌측 상단의 작은 빈 사각형은 체크박스인지 장식인지 불명확하므로 TODO로 남기는 것이 안전하다.
