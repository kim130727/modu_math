[문제 개요]
- 문제 번호는 `8.`로 보인다.
- 지시문은 `곧은 선을 찾아 기호를 선택하세요.`로 읽힌다.
- 오른쪽에는 4개의 선 그림이 있고, 각 선에 `가, 나, 다, 라`가 하나씩 붙어 있다.
- 하단 왼쪽에는 정답과 해설 문구가 작게 인쇄되어 있다.
- 전체 화면은 `문제 문장 + 그림 선택지 + 하단 정답/해설` 구조이다.

[원본에 보이는 요소]
- 상단 문장: `8. 곧은 선을 찾아 기호를 선택하세요.`
- 선 그림 주변의 표기: `가`, `나`, `다`, `라`
- 그림 아래 괄호 표기: `(가, 나, 다, 라)`
- 하단 왼쪽 인쇄 텍스트:
  - `(정답)다`
  - `(해설)가, 나, 다는 곧은 선입니다.`
- 오른쪽 그림은 왼쪽 점과 오른쪽 점을 연결하는 선 4개로 보인다.
- 각 선은 위에서 아래로 배열되어 있으며, 길이와 굽음이 서로 다르다.

[수식/관계 구조]
- 수식이나 등호 연쇄는 보이지 않는다.
- 이 문제는 계산식이 아니라, 그림의 선 모양을 보고 분류하는 선택형 구조이다.
- 보이는 관계는 다음과 같이 정리할 수 있다.
  - `가, 나, 다, 라` 각각이 하나의 선에 대응한다.
  - 선의 성질을 보고 `곧은 선` 여부를 고르는 형식이다.
- 정답이 인쇄되어 있으나, 이는 관찰 사실로만 기록하며 새로 추론하지 않는다.

[등호별 변환 의미]
- 등호 `=`가 포함된 식은 보이지 않는다.
- 따라서 단계별 변환 의미를 적을 대상이 없다.

[빈칸 정책]
- 이 화면에는 수식 빈칸이나 답 칸이 별도로 보이지 않는다.
- 하단의 `(정답)다`는 인쇄된 해설 정보로 보이며, 빈칸을 채운 결과로 재해석하지 않는다.
- 만약 DSL에서 답 위치를 표현해야 하더라도, 추론값을 추가하지 말고 원문에 보이는 텍스트만 유지한다.

[반복 패턴]
- 왼쪽 시작점에서 오른쪽 끝점으로 이어지는 선이 4개 반복된다.
- 4개의 선은 모두 같은 두 점을 연결하는 형태로 보이며, 가운데 경로만 서로 다르다.
- 위에서 아래로 갈수록 선의 굽음이 달라지며, 하나는 거의 직선에 가깝다.
- `가, 나, 다, 라`는 각 선에 1회씩 대응하는 라벨 패턴이다.
- 아래의 `(가, 나, 다, 라)`는 선택지 전체를 묶어 적은 문자열처럼 배치되어 있다.

[DSL 작성 힌트]
- 상단에는 문제 번호와 지시문을 한 줄 텍스트로 배치한다.
- 오른쪽 큰 영역에 선 4개를 각각 별도 도형으로 그린다.
- 네 선은 모두 같은 좌우 기준점 사이에 놓이도록 정렬한다.
- 각 선 위 또는 근처에 `가`, `나`, `다`, `라` 라벨을 개별 텍스트로 배치한다.
- 선의 위아래 간격을 충분히 두어 서로 겹치지 않게 한다.
- 선 아래에는 `(가, 나, 다, 라)`를 한 줄로 둔다.
- 하단 왼쪽에는 작은 글씨로 `(정답)다`, 그 아래 `(해설)가, 나, 다는 곧은 선입니다.`를 분리해서 배치한다.
- 해설 문구는 본문과 분리된 보조 텍스트 영역으로 취급하는 것이 좋다.
- 도형 렌더링 시 선의 굵기는 비슷하게 맞추되, 곡선과 직선의 차이가 드러나도록 한다.
- 라벨은 선 중앙부 근처에 놓아 어떤 선에 대응하는지 즉시 보이게 한다.

[semantic/solvable 힌트]
- 이 문항의 의미는 `곧은 선`을 시각적으로 식별하는 것이다.
- 보이는 해설 문구에 따르면 `가, 나, 다`는 곧은 선으로 인쇄되어 있다.
- 다만 문제 풀이를 새로 추론해 확정적으로 설명하지는 않는다.
- DSL 관점에서는 각 선의 모양 분류가 핵심이며, 텍스트 해설은 부가 정보로만 다룬다.

[주의 및 TODO]
- 정답 `다`는 이미지에 인쇄되어 보이지만, 새로 계산하거나 추론한 값으로 쓰지 말고 관찰 사실로만 유지한다.
- 선의 정확한 곡률, 끝점 좌표, 선 굵기 차이는 해상도상 정밀 복원이 어렵다.
- `가, 나, 다, 라`의 정확한 배치가 어느 선의 어떤 지점에 붙는지는 이미지 기준으로 최종 확인이 필요하다.
- 하단 해설 문구는 작은 글씨이므로 OCR 오독 가능성이 있다. 최종 DSL 작성 전 원본 이미지 재확인이 필요하다.
- 이 문제는 수식 문제가 아니므로, 등호 변환이나 계산 단계는 추가하지 않는다.
