[문제 개요]
- 문제 번호는 `100.`으로 보인다.
- 상단 지시문은 `계산하여 들이가 더 많은 것의 기호를 선택하세요.`로 보인다.
- 화면 오른쪽 위에 선택지 박스가 있고, 그 아래에 `(정답)`, `(해설)`과 함께 계산 과정이 제시되어 있다.
- 객관식 1문항이며, 보기 `ㄱ`, `ㄴ` 두 개 중 더 큰 값을 고르는 형태로 보인다.

[원본에 보이는 요소]
- 왼쪽 상단에 작은 빈 사각형 기호가 있고 그 옆에 `100.`이 있다.
- 그 오른쪽으로 문제 문장이 한 줄로 이어진다.
- 오른쪽 상단에는 주황색 테두리의 직사각형 선택지 박스가 있다.
- 선택지 박스 안에는 두 줄이 있으며, 각각 `ㄱ`, `ㄴ`이 원형 테두리 안에 들어간 기호처럼 표시된다.
- 보기 1: `ㄱ 4200 mL + 1400 mL`
- 보기 2: `ㄴ 2 L 800 mL + 3 L 300 mL`
- 아래 해설 영역에 `(정답) ㄴ`이 보인다.
- `(해설)` 아래에 계산식 두 줄이 보인다.
- 마지막 줄에는 왼쪽에 회색 화살표가 있고, 오른쪽에 비교식이 있다.
- 비교식은 `5 L 600 mL < 6 L 100 mL`로 보인다.

[수식/관계 구조]
- 보기 ㄱ:
  - `4200 mL + 1400 mL`
- 해설에서 ㄱ의 계산:
  - `4200 mL + 1400 mL = 5600 mL = 5 L 600 mL`
- 보기 ㄴ:
  - `2 L 800 mL + 3 L 300 mL`
- 해설에서 ㄴ의 계산:
  - `2 L 800 mL + 3 L 300 mL = 6 L 100 mL`
- 마지막 비교:
  - `5 L 600 mL < 6 L 100 mL`
- 구조상 두 보기의 값을 각각 계산한 뒤, 더 큰 쪽의 기호를 고르는 관계로 보인다.

[등호별 변환 의미]
- ㄱ의 첫 번째 등호:
  - `4200 mL + 1400 mL`라는 덧셈식을 계산한 결과가 `5600 mL`로 적혀 있다.
- ㄱ의 두 번째 등호:
  - `5600 mL`를 같은 값의 다른 단위 표현인 `5 L 600 mL`로 바꿔 쓴 것이다.
- ㄴ의 등호:
  - `2 L 800 mL + 3 L 300 mL`의 계산 결과가 `6 L 100 mL`로 적혀 있다.
- 마지막 비교식:
  - 앞의 두 계산 결과 `5 L 600 mL`와 `6 L 100 mL`를 크기 비교한 것이다.
- 등호가 이어지는 부분은 `계산 결과 → 단위 바꾸기`의 순서로 읽힌다.

[빈칸 정책]
- 원본 화면에는 답을 적는 빈칸 입력칸이 보이지 않는다.
- `□`처럼 보이는 작은 사각형은 문제 번호 앞의 표시로 보이며, 답칸으로 해석하지 않는다.
- 보기가 아닌 빈칸이 새로 있다고 추론하지 않는다.
- 원본에 인쇄되어 보이는 내용만 기록하고, 빈칸이 있다면 추론값을 넣지 않고 빈칸 상태로 유지해야 한다.
- 정답이 이미 `(정답)` 옆에 보이므로, 이를 새로 계산해 채우는 방식은 사용하지 않는다.

[반복 패턴]
- 선택지는 `기호 + 식`의 형태가 두 개 나란히 제시된다.
- 두 보기 모두 덧셈식이며, 해설에서도 각 보기의 계산을 같은 형식으로 반복한다.
- 반복 구조는 다음과 같다:
  - 보기 제시
  - 각 보기의 계산
  - 단위가 다른 형태로 재표기
  - 두 결과 비교
- 보기 ㄱ은 `mL`만 사용하고, 보기 ㄴ은 `L`와 `mL`가 섞여 있다.
- 해설의 두 줄은 거의 같은 문법 구조를 공유한다.

[DSL 작성 힌트]
- 화면을 크게 3덩어리로 나누면 좋다:
  - 상단 문제문
  - 오른쪽 선택지 박스
  - 하단 정답/해설 영역
- 선택지 박스는 주황색 테두리의 직사각형으로 표현한다.
- 박스 내부는 위아래 2행으로 정렬하고, 각 행은 `원형 기호 + 텍스트` 구조로 둔다.
- `ㄱ`, `ㄴ` 기호는 임의로 바꾸지 말고 원본 순서를 유지한다.
- 해설은 `(정답)` 한 줄, `(해설)` 한 줄, 비교식 한 줄로 분리하면 된다.
- 마지막 줄의 회색 화살표는 별도 아이콘 요소로 두는 것이 좋다.
- 수식 텍스트는 `mL`, `L`, `+`, `=` , `<`를 그대로 유지한다.
- 등호가 여러 개 있는 식은 한 줄 안에서 연속 변환으로 배치한다.
- 분수나 괄호형 슬롯은 보이지 않으므로 별도 수식 슬롯은 필요 없어 보인다.

[semantic/solvable 힌트]
- 문제는 용량을 더한 뒤 값의 크기를 비교하는 유형으로 보인다.
- 보이는 해설만 기준으로 하면, 두 보기의 결과값이 서로 비교되는 구조다.
- 다만 실제 정답을 새로 계산해 확정하는 것은 여기서 하지 않는다.
- `(정답) ㄴ`은 원본에 인쇄되어 보이는 사실로만 취급한다.
- 해설의 비교식은 앞선 계산 결과를 비교하는 근거로 읽힌다.

[주의 및 TODO]
- 상단 지시문 `계산하여 들이가 더 많은 것의 기호를 선택하세요.`는 OCR이 약간 흔들릴 수 있으므로, 원본 이미지 기준으로 최종 판독을 확인할 것.
- `(정답) ㄴ`의 기호 모양은 원형 테두리 안 문자로 보이지만, 세부 모양은 이미지에서 다시 확인할 필요가 있다.
- 마지막 비교식의 화살표는 단순 안내인지, 해설 구분용인지 확정하지 말고 표시만 반영할 것.
- 해설에 보이지 않는 중간 계산을 임의로 추가하지 말 것.
- 원본에 없는 값은 추론하여 적지 말 것.
- `나누기 n은 분모의 ×n으로 반영된다.` 문구가 필요한 유형은 아니며, 본 문제에는 해당 변환이 보이지 않는다.
