[문제 개요]
- 문제 번호는 `□ 30.`으로 보인다.
- 문제 문장은 상단에 2줄로 배치되어 있으며, 다음과 같이 읽힌다.
  - `바둑돌과 같이 무게를 비교하는 데 사용할 수 있는 단위로 알맞은 것을`
  - `모두 선택해 보세요.`
- 가운데에는 비교 대상처럼 보이는 그림 3개가 가로로 배치되어 있다.
- 하단에는 `(정답)` 표시와 함께 그림 2개가 나열되어 있고, 더 아래에 `(해설)` 문장이 있다.

[원본에 보이는 요소]
- 보이는 텍스트
  - `□ 30.`
  - `바둑돌과 같이 무게를 비교하는 데 사용할 수 있는 단위로 알맞은 것을`
  - `모두 선택해 보세요.`
  - `(정답)`
  - `(해설)무게가 일정한 작은 단위여야 무게를 비교하기 쉽습니다.`
- 가운데 그림 3개
  - 왼쪽: 클립 그림
  - 가운데: 수박 그림
  - 오른쪽: `100`이 적힌 동전 그림
- 하단 정답 영역의 그림 2개
  - 클립 그림
  - `100` 동전 그림
- 하단 정답 그림 사이에는 쉼표가 보인다.

[수식/관계 구조]
- 화면에 등호, 계산식, 분수식, 변형식은 보이지 않는다.
- 문제의 관계는 수식이 아니라 “무게 비교에 사용할 수 있는 단위인지 여부를 고르는 선택형” 구조로 보인다.
- 보이는 사실과 추론되는 수학적 의미를 분리하면:
  - 보이는 사실: 클립, 수박, 100원 동전이 비교 대상처럼 제시됨.
  - 추론되는 수학적 의미: 무게를 비교하는 데 적절한 단위인지 판단하는 문제로 보임.
- 수식 단계는 없음.

[등호별 변환 의미]
- 등호가 있는 식 전개가 없다.
- 변환 단계 설명 필요 없음.

[빈칸 정책]
- 빈칸을 채우는 방식의 문제는 아니다.
- 정답을 추론해서 새 값을 쓰지 말고, 원본에 보이는 그대로의 그림과 텍스트만 유지해야 한다.
- 하단 `(정답)` 영역에 보이는 그림도 관찰 사실로만 적고, 그 의미를 추가로 확정하지 않는다.

[반복 패턴]
- 상단 문제 문장 → 가운데 선택 대상 그림들 → 하단 정답 예시 → 해설 문장 순으로 시각적 흐름이 이어진다.
- 가운데의 3개 그림은 같은 행에 넓게 분산 배치되어 있어 선택지 목록처럼 보인다.
- 하단 `(정답)` 영역은 가운데 등장한 그림 중 일부를 다시 보여주는 형태로 보인다.
- 해설 문장은 한 줄로 이어지며, 설명 문장만 단독으로 배치되어 있다.

[DSL 작성 힌트]
- 상단 문제 번호 `□ 30.`과 문제 문장은 하나의 상단 텍스트 블록으로 두는 것이 좋다.
- 문제 문장은 줄바꿈을 반영해 2줄 텍스트로 배치한다.
- 가운데 영역은 그림 3개를 가로로 넓게 배치하는 구성으로 처리한다.
  - 왼쪽: 클립
  - 가운데: 수박
  - 오른쪽: 100원 동전
- 각 그림 사이 간격이 크므로, 균등 간격 또는 넓은 수평 배치로 표현하는 것이 자연스럽다.
- 하단 `(정답)`은 라벨 텍스트로 두고, 그 오른쪽에 클립과 100원 동전을 배치한다.
- 정답 영역의 두 그림 사이에 쉼표가 있으므로, 쉼표를 별도 텍스트로 넣는 구성이 적절하다.
- `(해설)`은 별도 텍스트 블록으로 하단에 배치한다.
- 각 그림에는 텍스트 라벨이 붙어 있지 않으므로, 그림 요소만 독립적으로 배치하면 된다.
- 문제 번호 앞의 `□` 기호는 인쇄된 기호로 보이므로 텍스트에 포함해도 좋다.

[semantic/solvable 힌트]
- 보이는 문장만 기준으로 하면, 이 문제는 “무게를 비교하는 데 사용할 수 있는 단위”를 고르는 선택형이다.
- 하단 해설의 문장은 다음과 같이 읽힌다.
  - `무게가 일정한 작은 단위여야 무게를 비교하기 쉽습니다.`
- 이 해설은 왜 특정 물체가 비교 단위로 적절한지에 대한 설명으로 보인다.
- 다만 정답을 새로 계산하거나 확정할 필요는 없다.
- 원본에 이미 보이는 `(정답)` 영역은 관찰 사실로만 기술하고, 올바름을 판정하지 않는다.
