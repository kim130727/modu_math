[문제 개요]
- 문제 번호는 `39.`로 보인다.
- 문제 문장은 `정사각형을 찾아 기호를 선택하세요.`이다.
- 연한 파란색 격자 배경 위에 도형 4개가 가로로 배치되어 있다.
- 도형에는 각각 `가, 나, 다, 라`가 1개씩 표시되어 있다.
- 아래에는 보기처럼 `( 가, 나, 다, 라 )`가 크게 적혀 있다.
- 하단 왼쪽에 `(정답)나`가 보인다.
- 그 아래에 `(해설)`과 설명 문장이 인쇄되어 있다.

[원본에 보이는 요소]
- 상단 왼쪽: 작은 사각형 표시 뒤에 `39.`
- 상단 문장: `정사각형을 찾아 기호를 선택하세요.`
- 격자 배경: 연한 파란색의 세로·가로 격자
- 도형 4개:
  - `가`가 들어 있는 가로로 긴 직사각형
  - `나`가 들어 있는 정사각형처럼 보이는 도형
  - `다`가 들어 있는 세로로 긴 직사각형
  - `라`가 들어 있는 사다리꼴
- 보기: `( 가, 나, 다, 라 )`
- 정답 표기: `(정답)나`
- 해설 표기: `(해설)정사각형은 네 각이 모두 직각이고 네 변의 길이가 모두 같은 사각형입니다. 따라서 나입니다.`

[수식/관계 구조]
- 수식이나 등호 연쇄는 보이지 않는다.
- 대신 도형 분류 관계가 핵심이다.
- 보이는 분류 대상은 4개이고, 각 대상은 기호 `가/나/다/라`로 대응된다.
- 문제는 “정사각형”의 성질과 도형의 형태를 비교하여 해당 기호를 고르는 구조로 보인다.

[등호별 변환 의미]
- 해당 없음.
- 등호로 이어지는 변형식이나 계산식은 보이지 않는다.

[빈칸 정책]
- 빈칸을 채우는 형식은 보이지 않는다.
- 다만 선택형 답을 고르는 문제이므로, 화면에 이미 인쇄된 `(정답)나`는 관찰 사실로만 기록한다.
- 추론으로 정답을 새로 계산해 빈칸에 넣지 않는다.
- 원본에 보이는 기호, 문구, 배치만 유지하고, 보이지 않는 값은 비워 둔다.

[반복 패턴]
- 도형 4개가 왼쪽에서 오른쪽으로 일정한 간격으로 배열되어 있다.
- 각 도형 내부에 한 글자 라벨이 중앙 배치되어 있다.
- 라벨은 `가 → 나 → 다 → 라` 순서로 대응된다.
- 격자 배경은 전체 도형 영역에 반복적으로 깔려 있어 위치와 길이 비교를 돕는 시각 요소로 보인다.
- 아래 보기 `( 가, 나, 다, 라 )`는 위 도형들의 라벨을 다시 한 번 나열한 형태이다.

[DSL 작성 힌트]
- 상단에 문제 번호와 문제 문장을 한 줄 텍스트로 배치한다.
- 그 아래에 연한 파란색 격자 배경의 큰 사각형 영역을 둔다.
- 격자 영역 위에 도형 4개를 좌에서 우로 배치한다.
- 각 도형 내부에 중앙 정렬된 한 글자 라벨을 넣는다.
- 도형의 기본 형태는 다음과 같이 처리하면 좋다.
  - `가`: 가로 직사각형
  - `나`: 정사각형
  - `다`: 세로 직사각형
  - `라`: 사다리꼴
- 도형 아래 중앙에 보기 `( 가, 나, 다, 라 )`를 둔다.
- 더 아래 왼쪽에 `(정답)나`, `(해설)` 텍스트를 별도 줄로 배치한다.
- 배치상 `나` 도형은 중앙 부근, `라` 도형은 가장 오른쪽에 놓인다.
- 격자와 도형이 겹치는 부분이 있으므로, 도형 외곽선이 격자선보다 우선 보이도록 처리하면 좋다.

[semantic/solvable 힌트]
- 보이는 해설 문장은 정사각형의 정의를 설명한다.
- 해설에는 `정사각형은 네 각이 모두 직각이고 네 변의 길이가 모두 같은 사각형입니다.`라고 적혀 있다.
- 이어서 `따라서 나입니다.`가 보인다.
- 다만 이 문장은 원본에 인쇄된 해설을 옮긴 것이며, 여기서 새로 풀이를 전개하지는 않는다.
- 문제의 의미상 `정사각형`을 찾는 시각적 분류 문제이다.
- 선택지는 도형 기호이며, 실제 정답 판단은 원본에 인쇄된 표기를 따르되 새로 추론하지 않는다.

[주의 및 TODO]
- `가`와 `다` 도형의 정확한 변 길이와 각도는 이미지 해상도상 약간 흐릴 수 있다.
- `다` 도형의 윗변은 격자선과 겹쳐 보이므로 외곽선 판독에 주의가 필요하다.
- `라` 도형은 사다리꼴로 보이지만, 변의 기울기와 길이는 정밀하게 단정하지 말고 필요 시 TODO로 남긴다.
- 원본에 이미 보이는 `(정답)나`와 해설 문구는 관찰 사실로만 기록한다.
- 정답을 다시 계산해서 쓰지 말고, 빈칸이 있다면 빈칸 상태를 유지한다.
- 도형의 세부 비율은 근사치로만 다루고, DSL에서는 상대적 배치 관계를 우선 반영한다.
