[문제 개요]
- 문제 ID: `38`
- 문장: `네 변의 길이가 모두 같은 사각형을 찾아 기호를 선택하세요.`
- 보기 도형 4개가 가로로 배치되어 있으며, 각 도형에는 `가`, `나`, `다`, `라`가 하나씩 들어 있다.
- 하단에는 `(정답) 나`가 인쇄되어 있고, 그 아래 해설 문장이 있다.
- 이 문항은 “도형의 성질을 보고 알맞은 기호를 고르는” 선택형 문제이다.

[원본에 보이는 요소]
- 좌상단에 문제 번호 `38.`이 보인다.
- 문제 문장이 상단에 크게 한 줄로 배치되어 있다.
- 문제 문장 아래에 연한 하늘색 격자 배경이 깔려 있다.
- 격자 위에 네 개의 사각형 계열 도형이 좌→우 순서로 배치되어 있다.
- 각 도형 내부에 한 글자씩 `가`, `나`, `다`, `라`가 들어 있다.
- 도형 아래에는 `( 가, 나, 다, 라 )`가 가운데 정렬로 보인다.
- 화면 하단 왼쪽에 `(정답) 나`가 보인다.
- 하단 해설 문장이 두 줄로 인쇄되어 있다:
  - `네 변의 길이가 모두 같은 사각형은 정사각형입니다 따라서 나입니다.`
  - `정사각형은 네 각이 모두 직각이고 네 변의 길이가 모두 같은 사각형입니다.`

[수식/관계 구조]
- 수식 계산은 없다.
- 관계 구조는 “조건에 맞는 도형을 보기 중에서 1개 선택”하는 형태이다.
- 조건:
  - 대상: 사각형
  - 판정 기준: 네 변의 길이가 모두 같은가
- 보기 대응:
  - `가` : 사각형 1
  - `나` : 사각형 2
  - `다` : 사각형 3
  - `라` : 사각형 4
- 도형 간 수학적 비교는 길이/각도 성질을 시각적으로 판별하는 구조로 보인다.

[등호별 변환 의미]
- 해당 없음.
- 등호 연쇄, 식 변형, 계산 단계가 보이지 않는다.

[빈칸 정책]
- 본 문제에는 빈칸이 직접 보이지 않는다.
- 정답이 하단에 인쇄되어 있으나, DSL 작성용 원천 정보로는 빈칸을 채워 추론하지 말고, 보이는 텍스트만 기록한다.
- 만약 동일 형식의 다른 문항에서 빈칸이 있다면, 추론값을 넣지 않고 빈칸 상태를 유지해야 한다.
- 이번 자료에서는 정답 표기 `나`는 “인쇄된 관찰 사실”로만 취급한다.

[반복 패턴]
- 네 개의 선택 도형이 하나의 가로줄로 반복 배치되어 있다.
- 각 도형 중앙 근처에 기호 글자 1개가 들어가는 패턴이다.
- 격자 배경은 동일한 톤으로 넓게 반복된다.
- 보기 집합 표기 `( 가, 나, 다, 라 )`가 도형 아래에 다시 한 번 제시된다.
- 하단에는 `(정답)` 표시와 해설 문장이 이어지는 반복적인 문제지 레이아웃이다.

[DSL 작성 힌트]
- 상단: 문제 번호와 문제 문장을 한 줄로 배치한다.
- 본문 영역: 연한 하늘색 격자 배경 위에 도형 4개를 좌→우로 정렬한다.
- 각 도형 내부에는 `가`, `나`, `다`, `라`를 각각 넣는다.
- 보기 표기 `( 가, 나, 다, 라 )`는 도형들 아래 중앙 정렬로 배치한다.
- 하단 설명 영역에는 `(정답) 나`와 해설 문장을 별도 텍스트 블록으로 둔다.
- 격자 선은 얇고 옅은 파란색 계열로 처리하는 것이 원본과 유사하다.
- 도형들은 동일 높이의 칸 위에 놓인 것처럼 보이므로, 상대적 위치 정렬을 중시한다.
- 도형의 정확한 좌표값보다 “가로 4개 배치”와 “중앙 기호” 구조를 우선 반영한다.
- 박스/도형 내부의 선 굵기는 비교적 얇게 보인다.
- 해설 영역은 문제 영역과 충분히 떨어져 아래에 배치한다.

[semantic/solvable 힌트]
- 이 문항의 핵심 의미는 “네 변의 길이가 모두 같은 사각형 = 정사각형”을 찾는 것이다.
- 원본 하단 해설에서 `나`가 정답으로 인쇄되어 있다.
- 보이는 해설에 따르면 `나`는 정사각형으로 설명된다.
- 다른 보기 도형들은 정사각형이 아닌 사각형들로 보인다.
- 이 항목은 문제 의미 파악용이며, 새 정답을 추론하기 위한 자리가 아니다.
