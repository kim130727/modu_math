# [전체적인 구성]
- 화면 상단 왼쪽에 문제 번호 `□ 32.`가 있고, 그 오른쪽과 아래로 문제 문장이 2줄로 이어져 있습니다.
- 중앙에는 **표(table)** 가 크게 배치되어 있으며, 표 위에는 제목이 있고 표 아래에는 괄호로 묶인 보기 형태의 텍스트가 있습니다.
- 하단 왼쪽에는 `정답` 표시가 있고, 그 아래에는 `해설` 문장이 여러 줄로 이어집니다.
- 전체적으로 **문제 문장 → 표 → 보기 → 정답/해설** 순서로 구성되어 있습니다.

# [보이는 텍스트]
- `□ 32. 정수네 학교 학생들이 좋아하는 간식을 조사하였습니다. 가장 많은 학생들이 좋아하는 음식을 간식으로 준비하려고 할 때 어느 것을 준비하면 좋을까요?`
- 표 제목: `정수네 학교 학생들이 좋아하는 급식 메뉴`
- 표 머리글: `급식 메뉴 / 떡볶이 / 치킨 / 김밥 / 합계`
- 표 내용:
  - `여학생 수(명) / 38 / 42 / 30 / 110`
  - `남학생 수(명) / 27 / 59 / 40 / 126`
- 표 아래: `( 떡볶이 , 치킨 , 김밥 )`
- 하단 텍스트:
  - `정답 치킨`
  - `해설 각 간식별 좋아하는 학생 수를 구하면 떡볶이: 38 + 27 = 65(명), 치킨: 42 + 59 = 101(명), 김밥: 30 + 40 = 70(명)입니다.`
  - `따라서 가장 많은 학생이 좋아하는 치킨으로 간식을 준비하면 좋을 것 같습니다.`

# [반복되는 시각 패턴]
- 표 안에서 각 간식 종류에 대해 **여학생 수와 남학생 수가 한 줄씩 대응**되는 구조가 반복됩니다.
- 열 방향으로는 `떡볶이 → 치킨 → 김밥 → 합계`처럼 항목이 나열되어 있고, 행 방향으로는 `여학생 수(명)`, `남학생 수(명)`가 반복됩니다.
- 표 아래의 `( 떡볶이 , 치킨 , 김밥 )`는 표의 선택 대상이 되는 **3개 간식 항목을 다시 한 번 묶어 보여주는 형태**입니다.
- 해설에서는 `항목별 수 + 수 = 합`의 형태가 반복되지만, 이는 **풀이 과정에 해당**하며 문제의 표 자체와는 별개로 보입니다.

# [수식 단계]
- 해설에 수식이 3번 제시되어 있습니다.
  1. `38 + 27 = 65(명)`  
  2. `42 + 59 = 101(명)`  
  3. `30 + 40 = 70(명)`
- 각 식은 서로 다른 간식 항목에 대응하는 것으로 보이며, 모두 **여학생 수와 남학생 수를 더하는 형태**입니다.
- 등호가 있는 변형식이라기보다, **각 항목별 합을 계산해 나열한 식**입니다.

# [등호별 변환 의미]
- `38 + 27 = 65(명)`  
  - 보이는 의미: 떡볶이에 대해 여학생 수와 남학생 수를 더한 결과를 적었습니다.
- `42 + 59 = 101(명)`  
  - 보이는 의미: 치킨에 대해 여학생 수와 남학생 수를 더한 결과를 적었습니다.
- `30 + 40 = 70(명)`  
  - 보이는 의미: 김밥에 대해 여학생 수와 남학생 수를 더한 결과를 적었습니다.
- 위 식들은 **서로 변형되는 연결식**이 아니라, **각 항목을 독립적으로 계산한 식**으로 읽힙니다.

# [세부 내용]
- 문제는 “가장 많은 학생들이 좋아하는 음식”을 골라 간식으로 준비하라는 내용입니다.
- 표의 제목은 `정수네 학교 학생들이 좋아하는 급식 메뉴`라고 되어 있지만, 문제 문장과 해설에서는 이를 **간식**으로 활용하는 맥락으로 보입니다.
- 표는 4열 구조로 보이며,
  - 첫 열은 항목명(`급식 메뉴`)
  - 나머지 세 열은 간식 종류(`떡볶이`, `치킨`, `김밥`)
  - 마지막 열은 `합계`
- 각 행의 숫자는 다음처럼 배치되어 있습니다.
  - 여학생 수 행: `38`, `42`, `30`, `110`
  - 남학생 수 행: `27`, `59`, `40`, `126`
- 표의 선은 연한 초록색 계열이며, 셀 경계가 모두 그려져 있습니다.
- 표 제목은 표의 위 중앙에 정렬되어 있습니다.
- 보기처럼 보이는 `( 떡볶이 , 치킨 , 김밥 )`은 표 아래 중앙에 따로 적혀 있습니다.
- 하단의 `정답`과 `해설`은 문제 영역보다 작게 배치되어 있으며, 해설은 문장형 설명과 수식이 섞여 있습니다.

# [Python DSL 작성 힌트]
- 표를 작성할 때는 다음과 같은 구조로 나누어 표현하면 좋습니다.
  - 제목 텍스트
  - 4열 × 3행 정도의 표
  - 첫 행은 머리글
  - 두 번째/세 번째 행은 여학생/남학생 데이터
- `합계` 열은 **결과 칸처럼 보이지만**, 실제 문제 풀이를 대신해 채우지 말고 **보이는 숫자만 그대로 배치**해야 합니다.
- 표 아래의 `( 떡볶이 , 치킨 , 김밥 )`는 별도의 텍스트 객체로 중앙 정렬하여 배치하는 것이 자연스럽습니다.
- 해설 영역은 문제 재현이 목적이라면 별도 텍스트 블록으로 두되, 풀이 결과를 생성하는 용도로 사용하지 않도록 주의하세요.
- 문제 번호 `□ 32.`는 작은 사각형 기호와 번호가 함께 있는 형태로 보이므로, 번호 앞의 작은 빈 사각형도 포함해 구현할 수 있습니다.
- 표의 셀 배경은 연한 초록/연두색 계열로, 머리글 행은 조금 더 강조된 느낌입니다.

# [주의할 점]
- 이 문제는 **풀이형 수치 문제**이므로, 시각 분석에서는 정답을 새로 계산해 적지 마세요.
- 표의 `합계` 열 숫자는 이미 인쇄되어 있으므로, 빈칸으로 오해하지 말고 **보이는 값 그대로** 취급해야 합니다.
- `정답 치킨`은 화면에 보이지만, Python DSL 작성 시에는 문제 본문 재현인지 해설 포함인지에 따라 포함 여부를 구분해야 합니다.
- `급식 메뉴`와 문제 문장 속 `간식` 표현이 섞여 있으므로, 문구를 정확히 옮길 때는 원문 그대로 유지하는 것이 좋습니다.
- 표 아래 괄호 안 항목은 보기처럼 보이지만, 실제로는 선택 대상 목록인지 단정하지 말고 **그렇게 보인다는 수준**으로만 기술하세요.
