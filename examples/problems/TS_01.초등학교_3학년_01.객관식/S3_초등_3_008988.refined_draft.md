# [문제 개요]
- 단일 문항의 **도형 판별 선택형 문제**로 보입니다.
- 문제 번호는 **26번**으로 보입니다.
- 지시문은 **직각삼각형이면 ○, 직각삼각형이 아니면 ×를 선택**하라는 내용입니다.
- 화면 오른쪽에는 **삼각형 그림 1개**가 배치되어 있습니다.
- 화면 왼쪽 아래에는 **(정답)×**, **(해설)** 문장이 보입니다.

# [원본에 보이는 요소]
## 보이는 사실
- 상단 문장:
  - `26. 직각삼각형이면 ○, 직각삼각형이 아니면 ×를 선택하세요.`
- 오른쪽 도형:
  - 삼각형 외곽선 1개
  - 꼭짓점 3개가 있는 일반적인 삼각형 형태
  - 내부에 직각을 나타내는 작은 네모 표시는 보이지 않음
- 도형 아래:
  - 파란색 `×` 모양의 작은 표시가 보임
- 왼쪽 아래:
  - 작은 파란색 원형 표시 1개가 보임
  - `(정답)×`
  - `(해설) 한 각이 직각인 삼각형을 직각삼각형이라고 합니다.`

## 추론되는 수학적 의미
- 이 문항은 그림을 보고 **직각삼각형 여부를 판별**한 뒤 기호를 고르는 형태로 해석됩니다.
- 다만 실제 판정 결과를 새로 계산해 답을 확정하면 안 됩니다.

# [수식/관계 구조]
- 등호를 포함한 수식은 보이지 않습니다.
- 수식 변형 연쇄도 보이지 않습니다.
- 구조는 다음과 같은 **판정 관계**로 읽힙니다.
  - 조건: 직각삼각형 여부
  - 선택: `○` 또는 `×`

# [등호별 변환 의미]
- 해당 없음.
- 등호가 없으므로 단계별 변환 설명 대상이 없습니다.

# [빈칸 정책]
- 화면에 **빈칸 입력란은 보이지 않습니다.**
- 선택형 기호 `○`, `×`는 **빈칸으로 추론해서 채우지 말고**, 원본에 보이는 그대로 유지해야 합니다.
- 특히 `(정답)×`가 보이더라도, 이것을 답안 칸에 자동 반영하지 말고 **보이는 사실로만 기록**합니다.

# [반복 패턴]
- `○ / ×` 두 기호를 제시하고 하나를 선택하게 하는 **판정형 문항 패턴**입니다.
- 도형을 보고 성질을 판단하는 **도형 + 조건 확인** 구조입니다.
- 하단의 `(정답)`, `(해설)` 표시는 다른 문항에서도 반복될 수 있는 **정답 안내/설명 영역 패턴**으로 보입니다.
- 파란색 원형 표시는 **선택 마커 또는 편집용 표식**처럼 보이나, 용도는 확정할 수 없습니다.

# [DSL 작성 힌트]
## 렌더링/레이아웃 힌트
- 상단에 문제 문장을 **한 줄 텍스트**로 배치합니다.
- 오른쪽 상단 영역에 **삼각형 도형 1개**를 배치합니다.
- 도형 아래 중앙에 **파란색 `×` 표식**이 작게 보이도록 배치합니다.
- 왼쪽 아래에는 **작은 원형 표식**, 그 아래에 `(정답)×`, `(해설)` 순으로 배치합니다.
- 전체 레이아웃은 대략:
  - 상단: 문제 문장
  - 우측: 도형
  - 좌하단: 정답/해설 텍스트
  로 나누어 읽는 것이 자연스럽습니다.

## DSL component hints
- 문제 번호 텍스트
- 지시문 텍스트
- 삼각형 윤곽선 1개
- 도형 아래의 작은 `×` 표식
- 좌하단의 원형 마커 1개
- `(정답)×` 텍스트
- `(해설) ...` 텍스트

## 표현 주의
- 삼각형은 **외곽선만 있는 단순 도형**으로 처리합니다.
- 내부 직각 표시가 없다는 점을 반영합니다.
- 원형 마커의 정확한 의미는 불명확하므로, 기능을 단정하지 말고 **시각 요소**로만 옮깁니다.
- 답을 계산해서 채우지 말고, 원본에 보이는 `×`와 설명만 옮깁니다.

# [semantic/solvable 힌트]
- 문제의 의미는 **직각이 있는 삼각형인지 아닌지를 판별**하는 것입니다.
- 해설 문장에는 직각삼각형의 정의가 직접 제시됩니다:
  - `한 각이 직각인 삼각형을 직각삼각형이라고 합니다.`
- 다만 이 해설을 근거로 삼각형의 정답을 새로 판정하지는 마세요.
- 원본에서 이미 `(정답)×`가 보이므로, 이는 **관찰 사실**로만 기록 가능합니다.
