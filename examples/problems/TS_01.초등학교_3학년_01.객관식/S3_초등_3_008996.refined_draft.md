[문제 개요]
- 문제 번호는 **34.**
- 지시문은 **“정사각형이면 ○, 정사각형이 아니면 ×를 선택하세요.”**
- 화면 오른쪽 위에는 **기울어진 사각형 도형 1개**가 제시됨.
- 화면 아래쪽에는 **(정답) ×**가 보임.
- 해설 문장이 하단에 표시되어 있음.

[원본에 보이는 요소]
- **보이는 사실**
  - `34.`
  - `정사각형이면 ○, 정사각형이 아니면 ×를 선택하세요.`
  - 오른쪽 상단 쪽에 **마름모처럼 기울어진 사각형 윤곽선** 1개
  - 도형 아래에 **파란색 원형 표식**과 **× 표식**
  - 왼쪽 아래쪽에 `(정답)×`
  - `(해설)네 각이 모두 직각이고 네 변의 길이가 모두 같은 사각형을 정사각형이라고 합니다.`
- **추론되는 수학적 의미**
  - 이 문항은 도형의 분류를 묻는 **선택형 문제**로 보임.
  - 해설은 정사각형의 정의를 설명하는 문장임.
- **렌더링/레이아웃 힌트**
  - 상단에 문제 번호와 지시문을 한 줄로 배치.
  - 우측 상단에 도형을 크게 두고, 도형 바로 아래에 작은 **×** 표식을 배치.
  - 좌측 하단에 `(정답)×`를 별도 텍스트로 배치.
  - 하단 해설은 `(해설)` 라벨 뒤에 왼쪽 정렬로 길게 배치.
  - 도형은 정사각형보다 **회전된 사각형/마름모형 외곽선**처럼 보이도록 렌더링.

[수식/관계 구조]
- **보이는 사실**
  - 등호를 포함한 계산식이나 변형식은 없음.
  - 도형 판별을 위한 문자 선택 구조만 있음.
- **추론되는 수학적 의미**
  - `정사각형이면 ○`, `정사각형이 아니면 ×`의 **이분 선택 관계**.
- **렌더링/레이아웃 힌트**
  - 수식 블록 대신 일반 문장형 지시문으로 처리.
  - 선택 결과 표시는 기호 하나만 강조해 배치.

[등호별 변환 의미]
- 해당 없음.
- 등호 연쇄나 식 변형이 보이지 않음.

[빈칸 정책]
- 빈칸은 없음.
- **빈칸이 보이더라도 추론값을 넣지 말고, 원문에 보이는 형태 그대로 빈칸으로 유지**해야 함.
- 이 문항에서는 빈칸 채우기보다 **표시된 선택 결과를 관찰 사실로 기록**하는 방식이 적절함.

[반복 패턴]
- **보이는 사실**
  - `지시문 + 도형 제시 + 정답 표시 + 해설` 구조.
- **추론되는 수학적 의미**
  - 도형 판별 문제에서 자주 쓰이는 전형적인 구성.
- **렌더링/레이아웃 힌트**
  - 동일한 스타일의 선택 기호(○, ×)를 지시문과 도형 아래 표식에 함께 사용.
  - 해설은 본문보다 작은 설명 블록처럼 분리.

[DSL 작성 힌트]
- 문제 번호 `34.`와 지시문을 상단 좌측에 배치.
