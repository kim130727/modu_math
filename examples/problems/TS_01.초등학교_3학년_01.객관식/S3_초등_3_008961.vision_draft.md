# [전체적인 구성]
- 상단 왼쪽에 문제 번호와 본문 설명이 한 줄 문장 형태로 배치되어 있다.
- 중앙에는 연한 살구색 계열의 표가 있고, 표 위쪽에 별도의 제목 상자가 붙어 있다.
- 표 아래에는 범례처럼 보이는 아이콘 설명이 있으며, 그 아래쪽에는 보기(선택지)가 ①~⑤ 형태로 가로 방향으로 여러 줄에 걸쳐 배치되어 있다.
- 더 아래에는 `(정답)` 표시와 `(해설)` 문장이 이어서 제시되어 있다.
- 전체적으로 “그림그래프를 보고 순서를 고르는 문제” 구조이며, 표와 보기, 해설이 한 화면에 함께 있다.

# [보이는 텍스트]
- 문제 번호: `31.`
- 본문:
  - `정호네 학교 학생 1250명의 혈액형을 조사하여 그림그래프로 나타내었습니다.`
  - `학생 수가 많은 혈액형부터 차례대로 나타낸 것을 고르세요.`
- 표 제목: `학생들의 혈액형`
- 표의 열 제목:
  - `혈액형`
  - `학생 수`
- 행 레이블:
  - `A`
  - `B`
  - `AB`
  - `O`
- 범례:
  - 아이콘 `🙂` 옆에 `100명`
  - 작은 아이콘 `🙂` 옆에 `10명`
- 선택지:
  - `① A, B, O, AB`
  - `② AB, O, B, A`
  - `③ A, O, B, AB`
  - `④ O, A, B, AB`
  - `⑤ B, O, A, AB`
- 정답 표기: `(정답)③`
- 해설 첫머리:
  - `(해설) ...`
  - 이어지는 문장들에는 아이콘 개수를 비교하여 어떤 혈액형이 더 많고 적은지 설명하는 내용이 보인다.

# [반복되는 시각 패턴]
- 표 안의 각 혈액형 행마다, `🙂` 아이콘이 여러 개 반복되어 학생 수를 나타낸다.
- 아이콘의 크기가 두 종류로 보인다.
  - 큰 `🙂`는 `100명`
  - 작은 `🙂`는 `10명`
- 각 행마다 큰 아이콘 몇 개와 작은 아이콘 몇 개가 나열되어 있어, 같은 형식이 네 줄(A, B, AB, O)에 반복된다.
- 선택지는 모두 혈액형 기호 4개를 쉼표로 나열한 순서형 보기이며, 순서만 다르게 반복된다.
- 해설 문장에서도 `아이콘의 수를 비교하면`이라는 식의 비교 패턴이 반복되어 나온다.

# [수식 단계]
- 이 문제 화면에는 등호 `=`를 사용한 계산식 변형 단계는 보이지 않는다.
- 따라서 수식 전개나 식 변형 단계는 별도로 표시되지 않는다.

# [등호별 변환 의미]
- 해당 없음.
- 화면에 등호를 포함한 식이 없어서, 단계별 변환 의미를 설명할 수 없다.

# [세부 내용]
- 표는 2열 구조로 보인다.
  - 왼쪽 열: 혈액형 이름
  - 오른쪽 열: 해당 혈액형의 학생 수를 그림으로 나타낸 영역
- 표 제목 `학생들의 혈액형`은 표 바로 위 중앙에 독립된 작은 제목 상자처럼 붙어 있다.
- 표의 테두리와 내부 구분선은 연한 주황색으로 보이며, 각 행은 수평선으로 분리되어 있다.
- 각 행의 오른쪽 칸에는 아이콘이 왼쪽에서 오른쪽으로 한 줄로 정렬되어 있고, 행별로 아이콘 개수가 다르다.
- 범례는 표의 오른쪽 아래쪽에 따로 있으며, 큰 웃는 얼굴과 작은 웃는 얼굴이 각각 어떤 명수를 뜻하는지 알려 준다.
- 보기 번호는 파란색 원 안의 숫자처럼 표시되어 있고, 각 보기 뒤에 혈액형 4개가 나열된다.
- `(정답)③` 표시는 보기 목록 아래 왼쪽에 따로 적혀 있다.
- 해설 문단은 문제의 정답 순서를 설명하는 문장으로 이어지며, 실제로는 아이콘 개수를 비교하는 설명이 포함되어 있다.

# [Python DSL 작성 힌트]
- 먼저 상단 문제 문장을 텍스트 블록으로 배치한다.
- 가운데에 제목 상자 `학생들의 혈액형`을 놓고, 그 아래 2열 표를 만든다.
- 표의 왼쪽 열에는 `A`, `B`, `AB`, `O`를 행별로 넣는다.
- 오른쪽 열에는 각 행에 아이콘들을 가로로 반복 배치한다.
- 범례는 표 아래 오른쪽에 `큰 아이콘 = 100명`, `작은 아이콘 = 10명`처럼 두 줄 또는 한 줄로 배치한다.
- 선택지는 하단에 ①~⑤ 번호와 함께 4개 혈액형 문자열을 나열하는 형태로 구성한다.
- 해설 텍스트는 선택지 아래에 문단 형태로 배치하면 된다.
- 표 셀 내부의 아이콘은 행마다 개수가 다르므로, 각 행별 반복 횟수를 따로 지정해야 한다.
- 아이콘의 크기 차이를 표현할 수 있도록 큰 아이콘과 작은 아이콘을 서로 다른 심볼 또는 다른 크기 이미지로 구분하면 좋다.

# [주의할 점]
- 표 안의 아이콘 개수는 실제 시각 요소를 기준으로 정확히 세어야 하며, 추정해서 같은 수로 맞추면 안 된다.
- 정답으로 표시된 `③`은 화면에 보이는 정답 표기일 뿐, 빈칸을 추론해 채운 결과가 아니다.
- 문제를 풀어 순서를 재구성하는 방식으로 답안을 만들지 말고, 화면에 있는 그대로만 옮겨 적어야 한다.
- 해설 문장은 일부가 화면 하단에서 줄바꿈되어 보이므로, 줄 단위가 아니라 문장 단위로 옮기는 것이 더 적절할 수 있다.
- 표 제목과 표 본문 사이의 간격, 범례의 위치, 보기의 줄바꿈 위치가 중요하다.

# [불확실한 부분]
- 표 안 각 행의 아이콘 개수는 이미지 해상도상 정확한 수세기가 다소 헷갈릴 수 있다.
- 해설 문장의 일부는 화면 하단에서 잘려 보이거나 줄바꿈되어 있어, 문장 경계가 완전히 명확하지 않을 수 있다.
- 각 보기의 정확한 수직 정렬과 행 간 간격은 이미지에서 완전히 동일하게 재현하기 어려울 수 있다.
