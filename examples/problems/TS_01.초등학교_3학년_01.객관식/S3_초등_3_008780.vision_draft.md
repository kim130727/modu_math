# [전체적인 구성]
- 화면 상단 왼쪽에 문제 번호와 지시문이 있고, 오른쪽에는 주황색 테두리의 선택지 박스가 있습니다.
- 아래쪽에는 `(정답)`, `(해설)` 표시와 함께 두 선택지를 계산한 식이 순서대로 적혀 있으며, 마지막에 화살표와 비교식이 이어집니다.
- 전체적으로 **객관식 1문항 + 선택지 박스 + 풀이/정답 안내** 형태입니다.

# [보이는 텍스트]
- `□ 100. 계산하여 더이가 더 많은 것의 기호를 선택하세요.`
  - 첫 부분에 작은 빈 사각형 기호가 있고, 그 뒤에 문제 번호 `100.`이 보입니다.
  - 문장은 “계산하여 더가 더 많은 것의 기호를 선택하세요.”처럼 보이지만, 화면상 글자는 `더이가`로 읽히는 부분이 있습니다. 정확한 판독은 아래 불확실한 부분에 따로 적습니다.
- 선택지 박스 안:
  - `ㄱ 4200 mL + 1400 mL`
  - `ㄴ 2 L 800 mL + 3 L 300 mL`
- 아래 설명 부분:
  - `(정답) ㄴ`
  - `(해설) ㄱ 4200 mL + 1400 mL = 5600 mL = 5 L 600 mL`
  - `ㄴ 2 L 800 mL + 3 L 300 mL = 6 L 100 mL`
  - 화살표 기호 `→`
  - `5 L 600 mL < 6 L 100 mL`

# [반복되는 시각 패턴]
- 선택지는 **원형 기호 + 한글 자모 번호 + 식**의 형식으로 2개가 나란히 제시됩니다.
- 두 선택지 모두 **같은 덧셈 형태**이지만, 첫 번째는 `mL` 단위만 있고 두 번째는 `L`과 `mL`가 함께 섞여 있습니다.
- 해설에서는 각 선택지를 **계산식 → 같은 값을 다른 단위 표기 → 최종 비교** 순서로 정리합니다.
- 마지막 비교 결과는 화살표 뒤에 제시되어, 앞선 두 계산 결과를 비교하는 구조입니다.

# [수식 단계]
- `ㄱ 4200 mL + 1400 mL = 5600 mL = 5 L 600 mL`
  - 첫 등호 앞에서는 두 용량의 덧셈식이 보입니다.
  - 첫 등호 뒤에서는 덧셈 결과가 `5600 mL`로 적혀 있습니다.
  - 두 번째 등호 뒤에서는 같은 값을 `5 L 600 mL`로 바꿔 적었습니다.
- `ㄴ 2 L 800 mL + 3 L 300 mL = 6 L 100 mL`
  - 첫 등호 앞에서는 `L`와 `mL`가 섞인 두 수의 덧셈식이 보입니다.
  - 등호 뒤에는 덧셈의 결과가 혼합 단위 표기로 적혀 있습니다.
- `5 L 600 mL < 6 L 100 mL`
  - 앞의 두 계산 결과를 비교한 식으로 보입니다.
  - 왼쪽 값이 작고 오른쪽 값이 큰 형태의 비교식입니다.

# [등호별 변환 의미]
- 첫 번째 식에서는:
  - `4200 mL + 1400 mL`라는 **원래 덧셈식**이
  - `5600 mL`라는 **계산 결과**로 바뀌고,
  - 다시 `5 L 600 mL`라는 **다른 단위 표현**으로 바뀝니다.
- 두 번째 식에서는:
  - `2 L 800 mL + 3 L 300 mL`라는 **원래 덧셈식**이
  - `6 L 100 mL`라는 **계산 결과 표기**로 정리됩니다.
- 마지막 비교식은:
  - 앞선 두 결과를 나란히 두고 **크기 비교**를 나타내는 것으로 보입니다.
  - 다만 실제로 어떤 수를 기준으로 비교하는지까지는 문제를 풀지 않고는 단정하지 않습니다.

# [세부 내용]
- 문제 번호 `100.`은 왼쪽 상단에 있고, 번호 앞에 작은 빈 사각형이 있습니다.
- 선택지 박스는 화면 오른쪽 상단에 크게 배치되어 있으며, 얇은 주황색 외곽선으로 둘러싸여 있습니다.
- 박스 안의 두 선택지는 위아래로 세로 배치되어 있고, 각 항목은 원 안에 들어간 기호(`ㄱ`, `ㄴ`)가 먼저 나오고 그 뒤에 식이 이어집니다.
- 아래 해설 영역은 왼쪽 정렬로 배치되어 있으며, `(정답)`과 `(해설)` 표기가 먼저 나온 뒤 각 선택지의 계산이 이어집니다.
- 마지막 줄의 왼쪽에는 회색 화살표 아이콘이 있고, 그 오른쪽에 비교식이 배치되어 있습니다.
- 텍스트 크기는 상단 지시문이 가장 크고, 해설은 그보다 약간 작아 보입니다.
- 식은 모두 일반 텍스트처럼 쓰였고, 분수선이나 괄호형 슬롯은 보이지 않습니다.
- 도형 요소로는 선택지 박스, 작은 빈 사각형, 화살표 아이콘이 확인됩니다.

# [Python DSL 작성 힌트]
- 문제 본문, 선택지 박스, 해설 영역을 **세 개의 독립된 블록**으로 나누어 배치하면 좋습니다.
- 선택지 박스는 주황색 테두리의 직사각형으로 표현하고, 내부 항목은 위아래 2줄로 정렬하세요.
- 각 선택지는 `원형 기호 + 텍스트` 구조로 배치하세요.
- 해설 부분은 `(정답)` 라인, `(해설)` 라인, 마지막 비교식 라인으로 구분하면 읽기 쉽습니다.
- 마지막 비교식 앞의 화살표는 별도 아이콘 또는 문자 요소로 처리하세요.
- 수식은 수학 기호와 단위 `mL`, `L`을 그대로 텍스트로 유지하고, 계산 결과를 임의로 보정하지 마세요.

# [주의할 점]
- 문제를 풀어서 답을 새로 만들지 마세요.
- 선택지의 계산 결과를 추론해서 추가로 적지 마세요.
- `ㄱ`, `ㄴ`의 순서와 기호 모양을 임의로 바꾸지 마세요.
- 해설에 이미 보이는 식만 그대로 옮기고, 중간 과정을 새로 생성하지 마세요.
- 마지막 비교식의 의미를 넘어선 추가 설명은 피하세요.
- 상단 지시문의 문구는 판독이 다소 애매하므로, 원문 그대로 복기할 때 주의가 필요합니다.

# [불확실한 부분]
- 상단 문제 지시문이 화면상 `계산하여 들이가 더 많은 것의 기호를 선택하세요.`처럼 보이는데, 일부 글자가 흐려 정확한 판독이 어렵습니다.
- `정답` 표기의 뒤에 붙은 원형 기호가 `ㄴ`인지, 혹은 다른 기호가 함께 있는지 세부 모양은 해상도상 약간 흐립니다.
- 마지막 비교식 앞의 화살표는 단순 안내용인지, 해설 단계 구분용인지 화면만으로는 확정하기 어렵습니다.
