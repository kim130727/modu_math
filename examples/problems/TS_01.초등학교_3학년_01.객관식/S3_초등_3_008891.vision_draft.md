[전체적인 구성]
- 전체는 **초등 수학 문제 1개**가 한 장에 배치된 형태입니다.
- 상단 왼쪽에 **문제 번호 55.**가 있고, 그 오른쪽으로 문제 지문이 한 줄~두 줄로 이어집니다.
- 중앙에는 **일주일 동안 팔린 음식의 수**를 나타내는 **그림그래프**가 큰 박스 안에 들어 있습니다.
- 그림그래프 아래에는 **선택지 4개**가 두 줄 배치되어 있습니다.
- 더 아래 왼쪽에는 **(정답) ②**가 표시되어 있고, 그 아래에는 **(해설)**과 함께 숫자 비교식 및 화살표가 있습니다.

[보이는 텍스트]
- 문제 번호: **55.**
- 지문: **어느 음식점에서 일주일 동안 팔린 음식의 수를 그림그래프로 나타내었습니다. 일주일 동안 가장 많이 팔린 음식과 가장 적게 팔린 음식을 순서대로 나열한 것을 고르세요.**
- 그래프 제목: **일주일 동안 팔린 음식의 수**
- 그래프 아래 음식 이름:
  - **김치찌개**
  - **비빔밥**
  - **만둣국**
  - **된장찌개**  
  - 마지막 항목 **된장찌개**는 줄바꿈되어 **된장찌 / 개**처럼 보입니다.
- 범례:
  - 큰 그릇 그림 옆에 **100그릇**
  - 작은 그릇 그림 옆에 **10그릇**
- 선택지:
  - **① 된장찌개, 비빔밥**
  - **② 김치찌개, 된장찌개**
  - **③ 된장찌개, 김치찌개**
  - **④ 김치찌개, 비빔밥**
- 정답 표시:
  - **(정답) ②**
- 해설 표시:
  - **(해설) 320>230>210>150**
  - 아래에 화살표 기호 **→**
  - **김치찌개>만둣국>비빔밥>된장찌개**

[반복되는 시각 패턴]
- 그림그래프 각 칸마다 **그릇 모양 아이콘**이 세로로 반복 배치되어 있습니다.
- 아이콘은 두 종류가 반복됩니다.
  - **큰 그릇**: 범례에서 **100그릇**을 뜻함
  - **작은 그릇**: 범례에서 **10그릇**을 뜻함
- 각 음식별로 그릇 아이콘의 **개수와 크기 조합**이 다르게 반복되어 있어 비교를 유도합니다.
- 네 항목 모두 **세로 열 구조**로 정렬되어 있고, 아래쪽에 각 음식 이름이 한 칸씩 대응됩니다.

[수식 단계] (해당될 때)
- 이 문제 자체는 계산식 변형이 중심은 아니지만, 아래쪽 해설에 **비교식**이 보입니다.
- 보이는 비교식:
  - **320>230>210>150**
- 이어서 화살표 뒤에 음식 이름 순서가 제시됩니다:
  - **김치찌개>만둣국>비빔밥>된장찌개**
- 등호는 없고, **부등호(>)**로 크기 관계를 나타내는 형태입니다.

[등호별 변환 의미] (해당될 때)
- 등호 변형은 보이지 않습니다.
- 대신 **부등호 체인**이 있어, 각 수의 크기 순서를 비교한 것으로 보입니다.
- 화살표 **→** 뒤의 음식 이름 나열은, 위의 숫자 비교 결과에 대응시키는 **정렬 결과 표시**로 보입니다.
- 다만 이 부분은 해설에 제시된 결과를 설명하는 것이므로, 실제 풀이 과정 전체를 추정해 확정하면 안 됩니다.

[세부 내용]
- 그림그래프는 큰 테두리 박스 안에 있고, 상단 제목이 박스 위쪽 중앙에 있습니다.
- 박스는 **세로 4칸**으로 나뉘어 있으며, 각 칸 아래에 음식명이 적혀 있습니다.
- 각 칸 내부에는 그릇 그림이 위에서 아래로 쌓여 있는 듯한 형태로 배치되어 있습니다.
- 네 항목의 그릇 개수는 눈으로 비교 가능하도록 서로 다릅니다.
- 오른쪽 끝 항목인 **된장찌개** 칸은 아래 텍스트 줄바꿈 때문에 이름이 두 줄로 보입니다.
- 범례는 그래프 박스 아래쪽 오른편에 위치하며, 큰 그릇과 작은 그릇이 **나란히** 놓여 있습니다.
- 선택지는 그래프 아래 왼쪽과 오른쪽에 나눠져 있으며, 각 선택지 앞에 **동그라미 숫자**가 붙어 있습니다.
- 정답 표시와 해설은 선택지보다 더 아래에 배치되어 있어, 문제 본문과 구분됩니다.

[Python DSL 작성 힌트]
- 그림그래프를 만들 때는 다음 구조를 분리해서 표현하면 좋습니다.
  - 문제 지문 텍스트 영역
  - 중앙의 **표 형태 그래프 박스**
  - 범례 영역
  - 선택지 영역
  - 정답/해설 영역
- 그래프 박스는 **4개의 세로 열**로 나누고, 각 열의 하단에 음식명을 넣는 방식이 적절합니다.
- 각 열 내부에는 **그릇 아이콘을 세로 정렬**해서 반복 배치합니다.
- 범례의 두 아이콘은 크기가 다르므로, 아이콘 종류를 **대형/소형**으로 분리해 사용할 수 있습니다.
- 선택지는 2행 2열 배치처럼 보이므로, 좌우 분할과 줄바꿈을 고려해 작성하면 좋습니다.
- 해설 부분의 숫자열과 화살표는 별도 텍스트 블록으로 처리하면 됩니다.
- 음식명 줄바꿈(특히 **된장찌개**)도 레이아웃에서 그대로 반영하는 것이 좋습니다.

[주의할 점]
- **답을 추론해서 채우지 마세요.**
- 그림그래프에서 각 음식의 실제 수를 분석해도, 여기서는 최종 답을 새로 계산해 적지 않는 것이 원칙입니다.
- 선택지의 정답 표시가 보이더라도, Python DSL 작성 시에는 **정답만 강조하는 방식**이 아니라 전체 레이아웃을 먼저 충실히 옮기는 것이 중요합니다.
- **100그릇, 10그릇** 범례는 실제 수량 단위이므로, 그릇 그림의 개수와 크기를 섞어 잘못 해석하지 않도록 주의해야 합니다.
- 텍스트의 줄바꿈, 특히 **된장찌개**의 분할 표기를 그대로 반영해야 합니다.

[불확실한 부분]
- 각 음식 칸 안의 그릇 아이콘 개수는 이미지상 비교 가능하지만, 정확한 배치 간격이나 픽셀 단위 위치는 확정하기 어렵습니다.
- 해설의 숫자 **320>230>210>150**이 각 음식과 정확히 어떻게 1:1 대응되는지는, 이미지에 보이는 결과만으로는 내부 계산 과정을 완전히 단정하기 어렵습니다.
- 선택지의 시각적 간격과 줄 간격은 스크린샷 해상도에 따라 약간 달라질 수 있습니다.
- 범례의 아이콘 크기 차이는 분명하지만, 실제 SVG/그림 객체의 정확한 비율은 불확실합니다.
