[문제 개요]
- 문제 번호는 `90.`으로 보인다.
- 지시문은 다음처럼 보인다: `대성이네 반의 모둠별 찬장 도장 수를 조사하여 그림그래프로 나타내었습니다. 가장 많은 찬장 도장을 받은 모둠을 선택하세요.`
- 중앙에는 `모둠별 찬장 도장 수`라는 제목의 표가 있다.
- 표는 모둠별 도장 수를 그림으로 나타낸 그림그래프 형태이다.
- 화면 하단에는 `(정답) 다 모둠`, `(해설) 이 가장 많은 모둠을 찾으면 다 모둠입니다.`가 인쇄되어 보인다.
- 다만 정답/해설 표기는 원본에 보이는 관찰 사실이며, 본문 문제 데이터와 분리해서 다루는 것이 안전하다.

[원본에 보이는 요소]
- 상단 문제 문장
  - `90.`
  - `대성이네 반의 모둠별 찬장 도장 수를 조사하여 그림그래프로 나타내었습니다.`
  - `가장 많은 찬장 도장을 받은 모둠을 선택하세요.`
- 표 제목
  - `모둠별 찬장 도장 수`
- 표 머리글
  - 왼쪽 열: `모둠`
  - 오른쪽 열: `찬장 도장 수`
- 행 이름
  - `가`
  - `나`
  - `다`
- 범례
  - 큰 빨간 도장 그림 옆: `10개`
  - 작은 빨간 도장 그림 옆: `1개`
- 선택지처럼 보이는 괄호 텍스트
  - `( 가 모둠, 나 모둠, 다 모둠 )`
- 하단 표기
  - `(정답) 다 모둠`
  - `(해설) 이 가장 많은 모둠을 찾으면 다 모둠입니다.`
- 도장 내부 문구
  - 각 빨간 원형 도장 안에 `참 잘했어요`가 들어가 있다.

[수식/관계 구조]
- 이 문항은 등호를 이용한 수식 전개가 아니라, 그림그래프에서 수량을 읽는 구조이다.
- 표의 오른쪽 칸에는 빨간 원형 도장들이 가로로 배치되어 있고, 범례에 의해 각 도장의 크기가 의미를 가진다.
- 보이는 관계는 다음과 같이 해석된다.
  - 큰 도장 1개 = 10개
  - 작은 도장 1개 = 1개
- 각 모둠의 도장 수는 `큰 도장 개수`와 `작은 도장 개수`를 함께 읽는 방식으로 표현되어 있다.
- 다만 실제 총합 계산은 이 초안에서 확정하지 않는다.

[등호별 변환 의미]
- 해당 없음.
- 원본에는 `=`를 포함한 계산식 또는 변형 연쇄가 보이지 않는다.
- 따라서 등호별 단계 설명은 필요하지 않다.

[빈칸 정책]
- 빈칸을 채워야 하는 형태의 문제로 보이지 않는다.
- 선택형 구조처럼 보이는 괄호 `( 가 모둠, 나 모둠, 다 모둠 )`가 있으나, 여기서 임의로 답을 채워 넣지 않는다.
- 정답이 화면 하단에 인쇄되어 보이더라도, DSL 작성용 본문에서는 추론값을 넣지 않고 원문 관찰만 기록한다.
- 빈칸이 있다면 추론하지 말고 빈칸 상태로 유지한다는 정책을 적용한다.

[반복 패턴]
- 빨간색 원형 도장 그림이 여러 번 반복된다.
- 각 행마다 동일한 스타일의 도장 아이콘이 가로 방향으로 반복 배열된다.
- 같은 도장 아이콘이 크기만 달라져 범례와 본문에 반복 사용된다.
- 표의 가로줄/세로줄은 일정한 간격으로 반복되는 테두리 구조를 이룬다.
- 도장 안의 `참 잘했어요` 문구도 모든 도장에서 반복된다.

[DSL 작성 힌트]
- 화면을 크게 다음 덩어리로 나누면 좋다.
  - 상단 문제 문장 영역
  - 가운데 표 영역
  - 표 오른쪽 아래의 범례 영역
  - 그 아래의 선택지 괄호 영역
  - 하단 정답/해설 영역
- 표는 2열 3행 구조로 보는 것이 자연스럽다.
  - 왼쪽 열: 모둠 이름
  - 오른쪽 열: 도장 그림 나열
- 렌더링 힌트
  - 표 제목은 표 위 중앙 정렬에 가깝게 배치
  - 표는 연한 갈색/베이지색 테두리로 보임
  - 오른쪽 셀은 넓고, 도장들이 좌→우로 일정 간격 배치됨
  - 도장들은 모두 원형이며 빨간색 계열
  - 큰 도장과 작은 도장은 크기 차이가 분명함
  - 범례는 표 아래 오른쪽에 따로 두고, 큰 도장 아이콘과 작은 도장 아이콘을 나란히 배치
  - 괄호 텍스트 `( 가 모둠, 나 모둠, 다 모둠 )`는 표 아래 우측에 한 줄로 둠
- 텍스트 처리 힌트
  - 도장 내부의 `참 잘했어요`는 아이콘 내부 텍스트로 보존
  - 표 제목, 머리글, 행 이름은 별도 텍스트 요소로 분리
- 정답/해설은 본문 문제와 시각적으로 분리된 보조 텍스트로 취급하는 것이 안전하다.

[semantic/solvable 힌트]
- 이 문제의 핵심 의미는 “그림그래프에서 가장 많은 모둠을 찾기”이다.
- 큰 도장과 작은 도장의 의미가 범례로 주어져 있으므로, 각 행의 수량은 그림 아이콘을 읽어서 비교하는 구조이다.
- 그러나 이 초안에서는 각 행의 총량을 계산하거나 최댓값을 단정하지 않는다.
- 원본 하단에 정답 표기가 보이지만, 문제 해결 결과로 확정해서 서술하지 않는다.
