[문제 개요]
- 문제 번호는 `1.`로 시작하며, 지시문은 `바르게 계산한 것을 선택하세요.`이다.
- 화면 상단에는 좌우로 2개의 주황색 테두리 박스가 배치되어 있고, 각 박스 안에는 세로셈 형태의 덧셈식이 들어 있다.
- 화면 중간 왼쪽에는 `(정답)` 라벨 아래 세로셈 1개가 따로 제시되어 있다.
- 화면 하단 왼쪽에는 `(해설)` 라벨 아래 세로셈 2개가 나란히 제시되어 있다.
- 전체 구성은 여러 보기 중 바르게 계산한 것을 고르는 판단형 문제로 보인다.

[원본에 보이는 요소]
- 상단 문구: `1. 바르게 계산한 것을 선택하세요.`
- 왼쪽 주황색 박스 안:
  - `188`
  - `+ 456`
  - `534`
- 오른쪽 주황색 박스 안:
  - `265`
  - `+ 347`
  - `612`
- 중간 왼쪽 `(정답)` 아래:
  - `265`
  - `+ 347`
  - `612`
- 하단 왼쪽 `(해설)` 아래:
  - 노란색 강조 숫자 `11`
  - `188`
  - `+ 456`
  - `644`
  - 노란색 강조 숫자 `11`
  - `265`
  - `+ 347`
  - `612`
- 이미지 하단의 해설 영역에서는 두 세로셈이 좌우로 배치되어 있으며, 각각 위쪽에 노란색 강조 숫자가 있다.

[수식/관계 구조]
- 선택지 형태의 세로셈 2개가 상단에 있고, 각각은 다음처럼 읽히는 구조이다.
  - `188 + 456 = 534`처럼 보이는 왼쪽 항목
  - `265 + 347 = 612`처럼 보이는 오른쪽 항목
- 중간 `(정답)` 영역에는 오른쪽과 같은 숫자 배열이 다시 나타난다.
- 하단 `(해설)` 영역에는 두 개의 세로셈이 비교용으로 나란히 제시된다.
- 해설의 왼쪽 세로셈은 결과가 `644`로 보이고, 오른쪽 세로셈은 결과가 `612`로 보인다.

[등호별 변환 의미]
- 화면에는 `=`를 사용한 연속된 변형식이 직접적으로 보이지 않는다.
- 따라서 등호를 따라 단계별로 바뀌는 식의 의미는 별도로 해석할 수 없다.
- 세로셈 구조에서는 입력된 두 수와 결과 수가 한 묶음으로 제시되어 있을 뿐, 등호 연쇄는 표시되지 않는다.

[빈칸 정책]
- 본 이미지에는 빈칸 입력란이 보이지 않는다.
- 따라서 빈칸을 추론해서 채우지 않는다.
- 만약 DSL 변환 과정에서 빈칸 요소를 따로 다루게 되더라도, 보이는 값이 없으면 공란으로 유지해야 한다.

[반복 패턴]
- 세로셈 형식이 여러 번 반복된다.
- 각 세로셈은 세 줄 구조로 보인다.
  - 첫 줄: 첫 번째 수
  - 둘째 줄: `+` 기호와 두 번째 수
  - 셋째 줄: 가로선 아래 결과
- 각 자릿수는 세로로 정렬되어 있으며, 고정폭 글꼴처럼 일정한 간격으로 배치되어 있다.
- 하단 해설에서는 같은 형태의 세로셈 2개가 좌우로 반복된다.
- 각 해설 세로셈의 위쪽에는 노란색 강조 숫자 `11`이 붙어 있다.
- 노란색 강조는 받아올림이나 자리 관계를 설명하는 시각적 장치로 보이지만, 그 정확한 의미는 단정하지 않는다.

[DSL 작성 힌트]
- 상단에 문제 문구 `1. 바르게 계산한 것을 선택하세요.`를 배치한다.
- 상단의 두 주황색 박스는 같은 크기의 사각형으로 좌우 정렬해 배치한다.
- 각 박스 내부에는 세로셈 3줄을 넣는다.
  - 첫 줄은 첫 번째 수
  - 둘째 줄은 `+`와 두 번째 수
  - 셋째 줄은 가로선 아래 결과
- 세로셈은 각 자리 숫자가 위아래로 정확히 맞도록 정렬한다.
- `(정답)` 라벨은 중간 왼쪽에 별도로 텍스트로 둔다.
- `(해설)` 라벨은 하단 왼쪽에 별도로 텍스트로 둔다.
- 하단 해설 영역에서는 두 세로셈을 좌우로 나란히 배치하고, 각 세로셈 상단에 노란색 강조 숫자 `11`을 작게 추가한다.
- 가로선은 결과 줄 바로 위에 짧은 수평선으로 넣는다.
- 박스 테두리는 얇은 주황색 선으로 표현한다.
- 숫자와 기호는 중앙 정렬에 가깝게 배치하되, 자릿수 정렬이 우선되도록 한다.

[semantic/solvable 힌트]
- 이 문제는 보이는 구성만 보면 “어느 계산이 바르게 되었는지 고르는 문제”로 해석된다.
- 상단의 두 보기 중 하나가 선택 대상으로 제시되고, 중간의 `(정답)` 표시는 해설/정답 제시용으로 보인다.
- 하단 `(해설)`은 두 세로셈을 비교하면서 계산 결과를 보여 주는 형태로 보인다.
- 다만 화면만으로는 어떤 선택지가 실제 정답인지, 또는 노란색 숫자의 정확한 의미가 무엇인지는 단정하지 않는다.

- 빈칸이 없으므로 빈칸 채우기 추론은 하지 않는다.
- 하단 해설의 노란색 `11` 표시는 정확한 규칙이 불명확하므로 TODO로 남긴다.
- `(정답)` 아래의 세로셈이 선택지와 어떤 대응 관계인지, 화면만으로는 완전히 확정하지 않는다.
- 주황색 박스가 클릭 가능한 선택지인지, 단순 강조 박스인지도 TODO로 둔다.
- `188 + 456 = 534`, `188 + 456 = 644`, `265 + 347 = 612`와 같은 식은 화면에 보이는 배열을 그대로 옮긴 관찰값이며, 계산 진위는 판단하지 않는다.
