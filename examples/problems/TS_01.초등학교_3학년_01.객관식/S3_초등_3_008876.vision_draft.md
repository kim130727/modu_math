[전체적인 구성]
- 상단 왼쪽에 문제 번호와 지시문이 한 줄로 배치되어 있습니다.
- 가운데 영역에는 비교 대상 3개가 가로로 나란히 그림과 함께 배치되어 있습니다.
  - 왼쪽: 야구공 그림
  - 가운데: 지우개 그림
  - 오른쪽: 의자 그림
- 각 그림 아래에는 물건 이름이 각각 텍스트로 표시되어 있습니다.
- 하단 왼쪽에는 `(정답)` 표시와 함께 답이 제시되어 있고, 그 아래에는 `(해설)` 문장으로 비교 순서를 설명합니다.
- 전체적으로 “가벼운 것부터 무거운 것까지”의 순서를 그림으로 보여 주는 구성입니다.

[보이는 텍스트]
- `□ 31. 무게가 두 번째로 가벼운 것의 이름을 선택해 보세요.`
- 그림 아래 라벨:
  - `야구공`
  - `지우개`
  - `의자`
- 하단 텍스트:
  - `(정답) 야구공`
  - `(해설) 무게를 비교해 보면 지우개<야구공<의자이므로 두 번째로 가벼운 것은 야구공입니다.`

[반복되는 시각 패턴]
- 세 개의 대상이 “그림 + 이름” 형태로 반복 배치되어 있습니다.
- 각 대상은 비슷한 세로 간격을 두고, 이름은 그림 바로 아래 중앙 정렬에 가깝게 놓여 있습니다.
- 비교 대상은 모두 실물 모양의 아이콘/삽화로 표현되어 있고, 텍스트 라벨은 단어 하나씩입니다.
- 하단 해설에서는 `지우개<야구공<의자`처럼 `<` 기호를 사용해 크기/무게 순서를 한 줄로 나타냅니다.

[수식 단계]
- 별도의 계산식이나 등호가 이어지는 변형식은 보이지 않습니다.
- 해설에 비교식 `지우개<야구공<의자`가 한 번 제시됩니다.
- 이 비교식은 단계별 변형이라기보다, 세 물건의 무게 순서를 한 번에 나타낸 형태입니다.

[등호별 변환 의미]
- 해당 없음.
- 화면에는 등호 `=`를 포함한 수식 변형 과정이 보이지 않습니다.

[세부 내용]
- 문제 문장은 “무게가 두 번째로 가벼운 것의 이름을 선택해 보세요.”입니다.
- 비교 대상은 3개이며, 각 대상의 위치는 왼쪽에서 오른쪽으로:
  1. 야구공
  2. 지우개
  3. 의자
  로 보입니다.
- 가운데의 지우개는 연한 색의 직육면체 모양, 오른쪽 의자는 분홍색 좌석과 등받이가 있는 의자 그림입니다.
- 아래 해설은 무게 비교 결과를 문장으로 설명하고 있습니다.
- `(정답)` 뒤에 답이 따로 표기되어 있어, 문제 선택형/객관식 성격을 암시합니다.

[Python DSL 작성 힌트]
- 상단 문제 문장은 좌상단 텍스트 박스로 배치하면 됩니다.
- 중앙에는 3개의 그림 요소를 가로 정렬로 배치하고, 각 그림 아래에 대응 텍스트를 넣으면 됩니다.
- 그림과 텍스트 사이 간격은 비교적 좁고, 세 대상 사이에는 충분한 가로 여백이 있습니다.
- 하단에는 `(정답)`과 `(해설)`을 별도 텍스트 블록으로 분리해 배치합니다.
- 해설 문장 안의 `<` 기호는 일반 텍스트로 유지해야 합니다.
- 실제 문제 풀이 결과를 코드에 하드코딩하지 말고, 시각 요소만 재현하는 방향이 좋습니다.

[주의할 점]
- 답을 추론해서 빈칸처럼 채우는 방식은 피해야 합니다.
- 그림의 실제 크기 차이보다 화면 배치상 크기가 다르게 보일 수 있으므로, “무게”와 “크기”를 혼동하지 않도록 주의해야 합니다.
- 정답과 해설은 화면에 이미 인쇄된 텍스트이므로, 별도 계산 결과처럼 다루지 않도록 합니다.
- 문제의 핵심은 “두 번째로 가벼운 것”을 고르는 것이지만, 이 초안에서는 정답을 풀이하지 않습니다.

[불확실한 부분]
- 각 그림의 실제 외곽선 색이나 세부 음영은 해상도상 완전히 정확히 판별하기 어렵습니다.
- 야구공, 지우개, 의자 그림의 세부 비율은 화면 캡처 특성상 약간 다르게 보일 수 있습니다.
- 좌상단의 작은 사각형 기호는 체크박스처럼 보이지만, 실제 인쇄 의도는 단순 문제 번호 표식일 수 있습니다.
