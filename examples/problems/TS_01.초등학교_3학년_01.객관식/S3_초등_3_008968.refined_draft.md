[문제 개요]
- 문제 번호 `2.`가 좌측 상단에 보인다.
- 본문은 2줄 정도로 줄바꿈되어 있으며, 나무 높이를 조사한 뒤 `100 cm`에 가장 가까운 높이 차를 찾는 문제이다.
- 가운데에는 연한 초록색 계열의 표가 있고, 표 아래 오른쪽에는 선택지처럼 보이는 괄호 텍스트가 있다.
- 그 아래에는 `정답`, `풀이`, 계산식 3개, 결론 문장이 세로로 배치되어 있다.

[원본에 보이는 요소]
- 본문 문장:
  - `현주는 집 주변 나무들의 높이를 조사했습니다. 두 나무의 높이 차가 100 cm에 가장 가까운 나무는 어떤 나무와 어떤 나무인지 알맞은 것을 선택하세요.`
- 표의 열 제목:
  - `나무 종류`
  - `소나무`
  - `전나무`
  - `벚나무`
- 표의 행 제목:
  - `높이(cm)`
- 표의 수치:
  - `247`
  - `356`
  - `115`
- 표 오른쪽 아래 괄호 텍스트:
  - `( 소나무, 전나무, 벚나무 )`
- 풀이 부분의 인쇄 텍스트:
  - `(정답) 소나무, 전나무`
  - `(풀이)`
  - `(전나무) - (소나무) = 356 - 247 = 109(cm)`
  - `(소나무) - (벚나무) = 247 - 115 = 132(cm)`
  - `(전나무) - (벚나무) = 356 - 115 = 241(cm)`
  - `두 나무의 높이의 차가 100 cm에 가장 가까운 나무는 소나무와 전나무입니다.`

[수식/관계 구조]
- 표는 `나무 종류`와 각 나무의 `높이(cm)`를 대응시키는 3열 구조로 보인다.
- 높이 대응 관계:
  - 소나무 → `247`
  - 전나무 → `356`
  - 벚나무 → `115`
- 풀이의 계산식은 3개이며, 모두 두 나무의 높이 차를 구하는 형태이다.
  - 전나무와 소나무의 차: `356 - 247 = 109`
  - 소나무와 벚나무의 차: `247 - 115 = 132`
  - 전나무와 벚나무의 차: `356 - 115 = 241`
- 구조적으로는 각 쌍에 대해 큰 수에서 작은 수를 빼는 비교를 반복한다.
- `100 cm`와의 가까움을 판단하는 문제지만, 계산 결과를 새로 추론하지 않고 원본에 적힌 풀이만 관찰해야 한다.

[등호별 변환 의미]
- `(전나무) - (소나무) = 356 - 247 = 109(cm)`
  - 나무 이름으로 적은 비교식을 숫자로 바꾸어 계산한 뒤, 결과 `109`를 적은 형태이다.
  - 등호 왼쪽의 나무 이름 비교와 오른쪽의 실제 수치 계산이 대응된다.
- `(소나무) - (벚나무) = 247 - 115 = 132(cm)`
  - 소나무와 벚나무의 높이 차를 숫자로 계산한 식이다.
- `(전나무) - (벚나무) = 356 - 115 = 241(cm)`
  - 전나무와 벚나무의 높이 차를 숫자로 계산한 식이다.
- 이 문제에는 분수형 변환이나 나눗셈 변환은 보이지 않는다.

[빈칸 정책]
- 보이는 원본에는 빈칸 입력칸이 없다.
- 따라서 추론값을 채워 넣지 말고, 표와 풀이에 인쇄된 값만 그대로 유지해야 한다.
- 선택지처럼 보이는 `( 소나무, 전나무, 벚나무 )`는 빈칸이 아니라 별도 텍스트 요소로 취급한다.
- 정답이 화면에 인쇄되어 보이지만, DSL 작성에서는 새로 계산해 넣지 말고 관찰 사실로만 다룬다.

[반복 패턴]
- 표에서는 3개의 나무가 가로로 나란히 놓이고, 각 나무 아래에 해당 높이가 정렬된다.
- 풀이에서는 두 나무를 비교하는 식이 3번 반복된다.
  - `A - B = 수치`
  - `A - C = 수치`
  - `B - C = 수치`
- 각 계산식은 `나무 이름` → `숫자 대입` → `결과`의 순서로 반복된다.
- 마지막 결론 문장은 정답을 자연어로 다시 한 번 반복하는 요약형 문장이다.

[DSL 작성 힌트]
- 상단 문제 문장은 2줄 텍스트 블록으로 배치하는 것이 자연스럽다.
- 표는 연한 초록색 테두리/격자선이 있는 2행 4열 형태로 보인다.
- 왼쪽 첫 칸에는 항목명(`나무 종류`, `높이(cm)`)을 넣고, 나머지 3칸에는 나무 이름과 수치를 각각 대응시키면 된다.
- 열 정렬은 중앙 정렬에 가깝고, 숫자들은 각 열 아래에 정확히 맞춰 놓는 구성이 적합하다.
- 표 아래 오른쪽의 `( 소나무, 전나무, 벚나무 )`는 별도 텍스트 요소로 분리하는 것이 좋다.
- 풀이 블록은 정답 표시와 계산식 3줄, 결론 문장을 세로로 이어 배치하면 원본 구조를 잘 반영한다.
- 계산식은 등호가 여러 개 있는 한 줄식으로 보이므로, 각 식을 한 줄 단위로 유지하는 편이 좋다.
- 괄호로 나무 이름을 먼저 표시한 뒤 수식으로 이어지는 형태를 그대로 반영하면 된다.

[semantic/solvable 힌트]
- 이 문제는 표에 제시된 세 나무의 높이를 바탕으로, 높이 차가 `100 cm`에 가장 가까운 두 나무를 찾는 선택형 문제로 보인다.
- 원본 풀이에는 세 쌍의 높이 차 계산이 모두 제시되어 있다.
- 정답 표시와 결론 문장이 화면에 인쇄되어 보인다.
- 다만 DSL 작성 단계에서는 정답을 새로 판단하거나 재서술하지 말고, 보이는 텍스트만 반영해야 한다.
