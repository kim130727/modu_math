[문제 개요]
- 문제 번호는 `□ 32.`로 보인다.
- 문제 문장은 상단에 2~3줄로 줄바꿈되어 배치되어 있으며, 내용은 다음과 같이 읽힌다.
  - `정수네 학교 학생들이 좋아하는 간식을 조사하였습니다.`
  - `가장 많은 학생들이 좋아하는 음식을 간식으로 준비하려고 할 때 어느 것을 준비하면 좋을까요?`
- 중앙에는 제목이 있는 표가 있고, 표 아래에는 괄호로 묶인 선택 항목 텍스트가 있다.
- 하단에는 `(정답) 치킨`, `(해설) ...` 형태의 정답/해설이 보인다.
- 원본 이미지 기준으로, 이 문항은 표를 보고 가장 많은 학생이 좋아하는 항목을 고르는 형식으로 보인다.

[원본에 보이는 요소]
- 상단 왼쪽:
  - 작은 네모 기호 `□`
  - 문제 번호 `32.`
- 상단 본문 문장:
  - `정수네 학교 학생들이 좋아하는 간식을 조사하였습니다. 가장 많은 학생들이 좋아하는 음식을 간식으로 준비하려고 할 때 어느 것을 준비하면 좋을까요?`
- 표 제목:
  - `정수네 학교 학생들이 좋아하는 급식 메뉴`
- 표 머리글:
  - `급식 메뉴 / 떡볶이 / 치킨 / 김밥 / 합계`
- 표의 행:
  - `여학생 수(명) / 38 / 42 / 30 / 110`
  - `남학생 수(명) / 27 / 59 / 40 / 126`
- 표 아래 괄호 텍스트:
  - `( 떡볶이 , 치킨 , 김밥 )`
- 하단 텍스트:
  - `(정답)치킨`
  - `(해설)각 간식별 좋아하는 학생 수를 구하면 떡볶이: 38 + 27 = 65(명), 치킨: 42 + 59 = 101(명), 김밥: 30 + 40 = 70(명)입니다.`
  - `따라서 가장 많은 학생이 좋아하는 치킨으로 간식을 준비하면 좋을 것 같습니다.`

[수식/관계 구조]
- 표는 2개의 수집단(여학생 수, 남학생 수)과 3개의 메뉴 항목(떡볶이, 치킨, 김밥), 그리고 합계 열로 구성된다.
- 수치 관계는 다음처럼 읽힌다.
  - 떡볶이: `38`과 `27`이 대응
  - 치킨: `42`와 `59`가 대응
  - 김밥: `30`과 `40`이 대응
- 합계 열에는 이미 인쇄된 값이 보인다.
  - 여학생 수 합계: `110`
  - 남학생 수 합계: `126`
- 해설에는 각 메뉴별로 두 수를 더하는 계산식이 3개 제시된다.
  - `38 + 27 = 65(명)`
  - `42 + 59 = 101(명)`
  - `30 + 40 = 70(명)`
- 이 식들은 서로 이어지는 변형식이 아니라, 각 항목의 합을 따로 계산한 독립 식으로 보인다.

[등호별 변환 의미]
- `38 + 27 = 65(명)`
  - 보이는 의미: 떡볶이에 대해 여학생 수와 남학생 수를 더한 결과를 적었다.
- `42 + 59 = 101(명)`
  - 보이는 의미: 치킨에 대해 여학생 수와 남학생 수를 더한 결과를 적었다.
- `30 + 40 = 70(명)`
  - 보이는 의미: 김밥에 대해 여학생 수와 남학생 수를 더한 결과를 적었다.
- 등호 연쇄 변형이 아니라, 각각의 계산 결과를 나열한 형태다.
- 각 식의 단계 변화는 `두 수를 더함 → 합을 적음`으로 이해할 수 있다.

[빈칸 정책]
- 이 원본에서 표의 값들은 빈칸이 아니라 인쇄된 숫자로 보인다.
- 따라서 숫자 칸을 추론으로 채우지 말고, 보이는 값만 그대로 유지해야 한다.
- 정답도 원본에 `(정답)치킨`으로 보이지만, 이는 관찰 사실로만 기록하고 새로 계산해 채우는 방식은 사용하지 않는다.
- 문제 재현용 DSL에서는 빈칸이 있는 경우에도 추론값을 넣지 말고 빈칸 상태를 유지해야 한다.
- 본 문항은 실제로 비어 보이는 칸이 확인되지 않으므로, “빈칸”을 새로 설정할 필요는 없다.

[반복 패턴]
- 표 안에서 `여학생 수(명)`와 `남학생 수(명)`이 각 메뉴 열에 대해 반복적으로 대응된다.
- 열 방향 반복:
  - `떡볶이 → 치킨 → 김밥 → 합계`
- 행 방향 반복:
  - `여학생 수(명)`
  - `남학생 수(명)`
- 해설에서는 `메뉴별 두 수를 더해 합을 구하는 패턴`이 3번 반복된다.
- 표 아래의 `( 떡볶이 , 치킨 , 김밥 )`는 3개의 메뉴를 한 번 더 묶어 보여주는 반복/요약형 텍스트로 보인다.

[DSL 작성 힌트]
- 레이아웃은 다음 순서로 잡는 것이 자연스럽다.
  1. 상단 문제 번호와 본문 문장
  2. 가운데 정렬된 표 제목
  3. 4열 표
  4. 표 아래 중앙 정렬된 괄호 텍스트
  5. 하단의 정답/해설 텍스트
- 표는 4열 3행 구성으로 보인다.
  - 첫 행: 머리글
  - 둘째 행: 여학생 수
  - 셋째 행: 남학생 수
- 셀 경계선은 모두 표시되고, 전체 표는 연두색 계열의 선과 연한 배경으로 보인다.
- 머리글 행과 첫 열은 시각적으로 구분되는 셀 스타일이 적용된 듯하다.
- 표 제목은 표의 위쪽 중앙에 단독 텍스트로 배치한다.
- 표 아래 괄호 텍스트는 별도 텍스트 박스로 중앙 정렬하는 것이 좋다.
- 하단 `(정답)`과 `(해설)`은 본문과 분리된 작은 텍스트 블록으로 처리할 수 있다.
- 문제 번호 앞의 작은 네모 기호 `□`도 별도 문자로 포함하는 편이 안전하다.

[semantic/solvable 힌트]
- 이 문항의 의미 구조는 “각 메뉴를 좋아하는 학생 수의 합을 비교하여 가장 큰 항목을 고르기”이다.
- 표의 각 열은 하나의 메뉴를 뜻하고, 여학생 수와 남학생 수를 더해 전체 선호 인원을 구하는 구조로 읽힌다.
- 해설에는 각 합이 이미 인쇄되어 있으므로, 문제를 푸는 방식과 정답 사실이 동시에 보인다.
- 다만 DSL 작성 시에는 정답을 새로 계산해 입력하기보다, 보이는 텍스트와 수치만 재현하는 것이 우선이다.
- 표의 `합계` 열은 문제 풀이 결과와 직접 연결되지만, 원본에서는 여학생/남학생 행의 합계 숫자도 함께 인쇄되어 있다.
- `나누기 n은 분모의 ×n으로 반영된다.`는 이 문항에는 해당하지 않는다.

[주의 및 TODO]
- 표 제목 `정수네 학교 학생들이 좋아하는 급식 메뉴`는 해상도 때문에 일부 획이 흐릴 수 있으므로, 이미지 기준으로 글자 하나씩 재확인하는 것이 좋다.
- 문제 본문에서는 `간식`을 사용하고, 표 제목에서는 `급식 메뉴`를 사용한다. 두 표현이 혼재하므로 원문 그대로 유지하되 의미를 임의로 통일하지 않는다.
- `(정답)치킨`은 화면에 실제로 보이지만, 이를 풀이로부터 추론한 값처럼 다루지 말고 관찰된 텍스트로만 취급한다.
- 표 아래 괄호 텍스트의 띄어쓰기 간격은 정확한 규칙이 다소 애매하므로, 이미지에 보이는 간격을 우선한다.
- 해설 첫 줄의 줄바꿈 위치와 문장 분리는 원본 이미지 기준으로 다시 확인할 필요가 있다.
- 표의 `합계` 열 숫자 `110`, `126`은 빈칸이 아니므로 누락 처리하지 않도록 주의한다.
