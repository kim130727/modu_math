[전체적인 구성]
- 화면은 흰 배경의 수학 문제 지면 일부로 보입니다.
- 상단 왼쪽에 문제 번호와 지시문이 있고, 그 오른쪽 아래쪽에 주황색 테두리의 비교 박스가 있습니다.
- 하단에는 괄호로 시작하는 해설/정답 표시 문장이 2줄 정도 배치되어 있습니다.
- 전체적으로 “두 식을 비교하여 더 작은 결과를 고르는” 형식의 객관식/선택형 설명 구조처럼 보입니다.

[보이는 텍스트]
- `3. 계산 결과가 700보다 작은 것을 선택하세요.`
- 주황색 박스 안:
  - `376 + 380`
  - `351 + 290`
- 아래 설명:
  - `(정답)351 + 290`
  - `(해설)376 + 380 = 756, 351 + 290 = 641`
  - `641<700이므로 700보다 작은 것은 351 + 290입니다.`
- 보이는 텍스트만 기준으로 보면, 두 식이 제시되고 그중 하나가 선택된 형태입니다.

[반복되는 시각 패턴]
- 두 개의 덧셈 식이 나란히 배치되어 있습니다.
- 같은 형식의 수식 `숫자 + 숫자`가 반복됩니다.
- 아래 해설에서는 두 덧셈 결과를 각각 `= 결과` 형태로 적은 뒤, 마지막에 부등호 비교 `641<700`가 등장합니다.
- 숫자와 식이 좌우로 나열되는 구조가 반복되어, 비교 대상들을 한눈에 보게 합니다.

[수식 단계]
- 해설 부분에 등식/부등식이 포함된 수식 흐름이 있습니다.
- `376 + 380 = 756`
- `351 + 290 = 641`
- `641<700`
- 마지막 문장은 위 계산 결과를 이용해 선택을 설명하는 문장으로 이어집니다.

[등호별 변환 의미]
- `376 + 380 = 756`
  - 보이는 형태상, 덧셈식과 그 계산 결과를 연결한 것입니다.
  - 식의 좌변은 원래 덧셈이고, 우변은 그 결과값입니다.
- `351 + 290 = 641`
  - 위와 같은 방식으로 두 번째 덧셈식의 결과를 적은 것입니다.
- `641<700`
  - 계산 결과와 기준값 700을 비교하는 부등호 표현입니다.
  - 이 비교는 식의 크기가 700보다 작은지 확인하는 용도로 보입니다.
- 마지막 문장 `641<700이므로 700보다 작은 것은 351 + 290입니다.`는
  - 앞의 비교 결과를 근거로 선택 항목을 말로 정리한 문장입니다.
  - 다만 여기서는 문제 풀이 자체를 재진술한 형태로만 보아야 합니다.

[세부 내용]
- 문제 번호 `3.`이 문장 앞에 붙어 있습니다.
- 지시문은 “계산 결과가 700보다 작은 것을 선택하세요.”로, 두 식 중 조건에 맞는 것을 고르는 과제입니다.
- 주황색 테두리 박스 안에 두 식이 가로 방향으로 배치되어 있습니다.
  - 왼쪽: `376 + 380`
  - 오른쪽: `351 + 290`
- 박스는 선택지 영역처럼 보이며, 각 식은 별도의 칸처럼 간격을 두고 놓여 있습니다.
- 아래쪽 해설에서는 각 식의 계산 결과를 직접 적어 비교하는 방식입니다.
- `(정답)` 표시는 특정 식이 선택되었음을 나타내고, `(해설)`은 계산 과정을 텍스트로 설명하는 역할입니다.
- 마지막 줄은 부등호 비교 후 조건 충족 여부를 자연어로 마무리합니다.

[Python DSL 작성 힌트]
- 상단 문제 문장은 텍스트 객체로 배치합니다.
- 주황색 테두리의 직사각형 박스를 하나 두고, 그 안에 두 개의 수식을 좌우로 정렬합니다.
- 각 수식은 동일한 스타일로 표시하되, 별도의 선택지처럼 충분한 간격을 둡니다.
- 아래 해설 영역은 텍스트 줄 단위로 배치하고, `(정답)`, `(해설)` 같은 라벨을 포함합니다.
- 수식 `=`, `<`는 일반 텍스트가 아니라 수식 표시 요소로 분리해도 좋습니다.
- 박스 내부의 수식은 가운데 정렬 또는 균등 간격 배치를 고려합니다.
- 해설 줄에서는 등식이 이어지는 형태가 아니라, 각 덧셈식과 결과를 각각 독립적으로 제시하는 구조로 작성하면 됩니다.

[주의할 점]
- 계산 결과를 추론해서 별도로 채워 넣지 마세요. 이 초안은 시각 요소 설명용입니다.
- 주황색 박스가 선택지 영역인지, 강조 표시인지 단정할 때는 실제 레이아웃만 근거로 서술하세요.
- `(정답)` 표시는 원문에 보이는 텍스트이므로 그대로 반영하되, 그것을 추가로 해석해 확장하지 마세요.
- 마지막 문장의 의미를 문제 풀이로 확정해 설명하기보다, 화면에 적힌 해설 문장으로만 다루는 것이 안전합니다.
- 화면이 일부 잘려 보이는 경우는 없지만, 아주 세밀한 글자 크기나 정확한 폰트는 단정하지 마세요.

[불확실한 부분]
- 주황색 테두리 박스의 정확한 용도는 선택지 박스처럼 보이지만, 원문 의도까지는 확정할 수 없습니다.
- `(정답)351 + 290` 문구가 별도 채점 표시인지, 예시인지, 편집용 표기인지 화면만으로는 명확하지 않습니다.
- 해설 문장의 줄바꿈 위치는 보이는 대로 추정해 적었으며, 실제 원본의 편집 행간은 다를 수 있습니다.
