[전체적인 구성]
- 한 장의 객관식 문제 화면이다.
- 상단 왼쪽에 문제 번호와 지시문이 있고, 그 아래 중앙 영역에 그림 3개가 가로로 배치되어 있다.
- 각 그림 아래에는 이름표가 붙어 있으며, 하단에는 보기 5개가 두 열 형태로 나뉘어 배치되어 있다.
- 가장 아래쪽에는 정답 표기와 해설 문장이 작게 들어 있다.

[보이는 텍스트]
- `□ 32. 무게가 무거운 것부터 순서대로 바르게 나열한 것을 고르세요.`
- 그림 아래 이름:
  - `수박`
  - `풍선`
  - `참외`
- 보기:
  - `① 풍선, 수박, 참외`
  - `② 수박, 풍선, 참외`
  - `③ 수박, 참외, 풍선`
  - `④ 참외, 풍선, 수박`
  - `⑤ 참외, 수박, 풍선`
- 하단 표시:
  - `(정답)③`
  - `(해설)순으로 들어왔을 때 힘이 많이 드는 것부터 순서대로 쓰면 수박, 참외, 풍선입니다.`
- 주의: 마지막 해설은 문제 해결 결과를 포함하지만, 여기서는 보이는 텍스트로만 기록한다.

[반복되는 시각 패턴]
- 사물 그림 3개가 동일한 높이 선상에 가로로 배열되어 있다.
- 각 그림 아래에 한글 이름이 한 번씩 반복되어 표시된다.
- 보기는 번호가 원형/동그라미 느낌의 파란 숫자 아이콘과 함께 나열된다.
- 보기 문장 형식은 모두 `이름, 이름, 이름`처럼 쉼표로 구분된 3개 항목 구조이다.
- 왼쪽 열에 ①, ③, ⑤가 세로로 정렬되고, 오른쪽 열에 ②, ④가 세로로 정렬되어 있다.

[수식 단계]
- 해당 없음. 숫자 계산식이나 등호를 포함한 변형 과정은 보이지 않는다.

[등호별 변환 의미]
- 해당 없음.

[세부 내용]
- 상단 문장 `무게가 무거운 것부터 순서대로 바르게 나열한 것을 고르세요.`는 기준을 제시하는 안내문이다.
- 중앙의 3개 그림은 각각 실제 사물의 아이콘처럼 보이며, 왼쪽부터 수박, 가운데 풍선, 오른쪽 참외 순으로 놓여 있다.
- 각 그림은 이름 아래에 정중앙 정렬에 가깝게 배치되어 있고, 그림과 글자 사이에 여백이 있다.
- 보기 문항들은 같은 길이의 문장처럼 보이도록 배치되어 있으며, 각 보기마다 3개의 대상이 쉼표로 구분된다.
- 하단의 `(정답)③`은 정답 표시로 보이며, 문제 본문과는 분리된 위치에 작게 표시되어 있다.
- 해설 문장은 괄호를 포함한 한 줄 문장으로, 문제의 기준과 결과를 설명하는 형태이다.

[Python DSL 작성 힌트]
- 전체 레이아웃은 `상단 문제문장 → 중앙 그림 3개 → 하단 보기 5개 → 맨 아래 정답/해설` 순서로 구성하면 좋다.
- 그림 3개는 가로 정렬 그룹으로 처리하고, 각 그림 아래에 텍스트 라벨을 붙이는 구조가 적합하다.
- 보기들은 2열 배치로 만들되, 왼쪽 열에 3개, 오른쪽 열에 2개를 두는 방식이 화면과 유사하다.
- 각 보기 앞의 번호는 파란색 원형 번호 스타일을 반영할 수 있다.
- 정답과 해설은 별도 작은 텍스트 영역으로 분리하는 것이 좋다.
- 문제 해결은 DSL 단계에서 하지 말고, 텍스트와 배치만 재현한다.

[주의할 점]
- 그림의 실제 상대적 무게를 추론해 서술하지 말 것.
- 보기의 정답을 새로 계산하거나 빈칸을 채우지 말 것.
- 하단의 `(정답)③`은 화면에 보이는 표시일 뿐, 분석 과정에서 정답을 확정하는 근거로 추가 해석하지 말 것.
- 해설 문장은 포함하되, 그 내용을 바탕으로 새 결론을 덧붙이지 말 것.
- 사물 그림은 모두 아이콘형이며, 실제 사진은 아니다.

[불확실한 부분]
- 각 그림의 세부 윤곽선 스타일은 해상도 때문에 아주 미세한 장식까지는 단정하기 어렵다.
- 정답 표시와 해설의 글꼴 크기, 색상은 화면상 작게 보이지만 정확한 폰트 정보는 알 수 없다.
- 상단 왼쪽의 작은 사각형 기호 `□`는 체크박스인지 여백 표기인지 화면만으로는 확정하기 어렵다.
