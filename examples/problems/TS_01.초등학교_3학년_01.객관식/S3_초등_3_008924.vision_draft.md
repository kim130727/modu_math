# [전체적인 구성]
- 화면 상단에는 문제 번호와 지시문이 한 줄 또는 두 줄에 걸쳐 배치되어 있습니다.
- 중앙에는 **표 형태의 그림 자료**가 크게 배치되어 있고, 표 위쪽에 제목이 있습니다.
- 표 아래 오른쪽에는 **그림 1개 = 10개, 그림 1개 = 1개**를 나타내는 범례가 있습니다.
- 범례 아래에는 괄호로 묶인 항목 **(가 모둠, 나 모둠, 다 모둠)** 이 보입니다.
- 화면 하단 왼쪽에는 **(정답) 다 모둠**, 그 아래에 **(해설)** 문장이 인쇄되어 있습니다.
- 전체적으로는 “모둠별 찬장 도장 수를 그림그래프로 나타낸 표”를 보고 가장 많은 모둠을 찾는 구성입니다.  
  - 이는 보이는 구조를 바탕으로 한 의미 설명이며, 실제 답은 여기서 확정하지 않습니다.

# [보이는 텍스트]
- `90.`
- `대성이네 반의 모둠별 찬장 도장 수를 조사하여 그림그래프로 나타내었습니다.`
- `가장 많은 찬장 도장을 받은 모둠을 선택하세요.`
- 표 제목: `모둠별 찬장 도장 수`
- 표 머리글: `모둠`, `찬장 도장 수`
- 행 이름: `가`, `나`, `다`
- 범례 옆 텍스트:
  - 빨간 원 그림 옆에 `10개`
  - 작은 빨간 원 그림 옆에 `1개`
- 괄호 텍스트: `( 가 모둠, 나 모둠, 다 모둠 )`
- 하단 표기:
  - `(정답) 다 모둠`
  - `(해설) 이 가장 많은 모둠을 찾으면 다 모둠입니다.`
- 각 빨간 원 내부에는 공통적으로 `참 잘했어요`라는 문구가 들어 있습니다.

# [반복되는 시각 패턴]
- 빨간색 원형 도장 그림이 여러 개 반복됩니다.
- 각 행(가, 나, 다)에 동일한 스타일의 도장 그림이 가로로 나열되어 있습니다.
- 도장 그림은 **큰 도장**과 **작은 도장** 두 종류로 구분되어 보입니다.
- 같은 행 안에서는 도장들이 일정한 간격으로 좌우로 배열되어 있습니다.
- 표의 테두리와 칸 경계선도 일정한 간격과 색으로 반복됩니다.
- 범례에서도 표 본문과 같은 도장 그림이 다시 사용되어, 그림 하나가 어떤 수량을 뜻하는지 안내합니다.

# [세부 내용]
- 표는 2열 구조로 보입니다.
  - 왼쪽 열: 모둠 이름
  - 오른쪽 넓은 열: 각 모둠의 찬장 도장 수를 그림으로 표시
- 각 행의 오른쪽 칸에 도장 그림이 가로로 배치되어 있으며,  
  큰 도장과 작은 도장이 섞여 있습니다.
- 범례의 큰 도장은 `10개`, 작은 도장은 `1개`를 뜻하는 것으로 보입니다.
- 따라서 각 모둠의 도장 수는 **도장 그림의 개수와 종류를 함께 읽는 방식**으로 표시되어 있습니다.
- 표 오른쪽 아래의 괄호 안에는 모둠 이름이 나열되어 있어, 선택지 또는 비교 대상임을 알 수 있습니다.
- 하단의 `(정답) 다 모둠` 및 `(해설)`은 편집용 예시 또는 정답 표시로 보입니다.
  - 다만 사용 목적상, 본문 문제만 추출할 때는 이 부분을 제외해야 할 수 있습니다.

# [Python DSL 작성 힌트]
- 표 영역과 범례 영역을 분리해서 작성하면 좋습니다.
- 표는 다음과 같이 구조화하면 좋습니다.
  - 제목 1개
  - 2열 표 1개
  - 행 3개: 가, 나, 다
  - 각 행 오른쪽 칸에 도장 이미지들이 가로로 여러 개 배치
- 범례는 표 아래쪽 오른편에 별도 그룹으로 둡니다.
  - 큰 도장 아이콘 + `10개`
  - 작은 도장 아이콘 + `1개`
- 각 행의 도장 개수는 **정답 계산용으로 추론하지 말고**, 보이는 도장 개수만 그대로 기록하는 방식이 안전합니다.
- 도장 아이콘은 동일한 이미지 자산을 크기만 다르게 써도 됩니다.
- 문제 문장은 상단에 2줄로 배치하는 것이 자연스럽습니다.
- 하단의 정답/해설 표기는 문제 본문과 분리하여 주석 또는 제외 대상로 처리하는 것이 좋습니다.

# [주의할 점]
- 실제 수량을 합산하거나 비교하여 답을 확정하지 마세요.
- “가장 많은”이라는 문장은 문제 의도이지만, 여기서는 시각 요소만 분석해야 합니다.
- 큰 도장과 작은 도장의 숫자 의미를 혼동하지 않도록 주의해야 합니다.
- 표 안의 빨간 원 안에는 모두 같은 문구가 들어가 있으므로, 이를 서로 다른 글자로 해석하지 마세요.
- 하단의 `(정답)`과 `(해설)`은 분석 대상 이미지에 포함되지만, 새 문제 생성 시에는 보통 제외할 수 있습니다.
- 괄호 안의 `가 모둠, 나 모둠, 다 모둠`은 선택지처럼 보이지만, 별도의 버튼 형태는 아닙니다.

# [불확실한 부분]
- 각 행의 도장 개수를 정확히 세는 작업은 여기서 수행하지 않았습니다.
- 도장 그림의 크기 차이가 실제로 정확히 `10개`와 `1개`를 의미하는지, 범례 해석은 이미지상 그렇게 보이지만 세부 인쇄 의도는 원본 편집 설정을 확인하는 것이 안전합니다.
- 표의 테두리 색과 간격은 이미지 해상도 때문에 약간 다르게 보일 수 있습니다.
- 하단의 `(정답) 다 모둠`은 원문 문제에 포함된 요소인지, 별도의 해설 삽입인지 불분명할 수 있습니다.
