[문제 개요]
- 문제 번호는 `12.`로 보인다.
- 상단에 긴 문제 문장이 한 줄로 배치되어 있다.
- 가운데에는 `일주일 동안 팔린 음식`이라는 제목의 그림그래프 표가 있다.
- 표 아래에는 범례가 있고, 오른쪽 아래에는 괄호로 묶인 텍스트와 `(정답)`, `(해설)` 문구가 보인다.
- 원본 이미지 기준으로 이 문제는 그림그래프를 보고 `된장찌개보다 더 많이 팔린 음식`을 고르는 형태로 보인다.

[원본에 보이는 요소]
- 상단 문제 문장:
  - `12. 어느 음식점에서 일주일 동안 팔린 음식의 수를 조사하여 그림그래프로 나타내었습니다. 된장찌개보다 더 많이 팔린 음식을 모두 선택해 보세요.`
- 표 제목:
  - `일주일 동안 팔린 음식`
- 표의 음식 이름:
  - `김치찌개`
  - `된장찌개`
  - `비빔밥`
  - `냉면`
- 범례 텍스트:
  - `100그릇`
  - `10그릇`
- 괄호 텍스트:
  - `( 김치찌개 , 비빔밥 , 냉면 )`
- 하단 출력 텍스트:
  - `(정답) 김치찌개, 냉면`
  - `(해설) 된장찌개보다 큰 그림 수가 더 많은 음식은 김치찌개와 냉면입니다.`

[수식/관계 구조]
- 이 화면에는 등호를 포함한 계산식이나 변형식이 따로 보이지 않는다.
- 관계 구조는 수식보다 `그림 개수 ↔ 음식의 양`을 대응시키는 그림그래프 구조이다.
- 각 음식 칸의 그릇 그림 개수와 범례의 의미를 연결해 해석하는 문제로 보인다.
- 표 내부의 세로 칸들은 서로 같은 폭으로 나뉘어 있으며, 각 칸 위쪽에 그릇 아이콘이 세로로 반복 배치되어 있다.

[등호별 변환 의미]
- 해당 없음.
- 화면에 `=` 형태의 변환 단계가 보이지 않는다.

[빈칸 정책]
- 화면에 명시적인 빈칸은 보이지 않는다.
- 따라서 추론으로 값을 채우지 않는다.
- 만약 DSL 작성 과정에서 빈 슬롯/빈칸 형태로 재현해야 하더라도, 보이는 텍스트나 그림 수를 임의로 완성하지 말고 원본 그대로 유지한다.
- `(정답)`으로 인쇄된 텍스트는 보이지만, 이것을 근거로 추가 빈칸을 채우지 않는다.

[반복 패턴]
- 표는 4개의 세로 칸이 같은 폭으로 반복된다.
- 각 칸의 위쪽에는 동일한 스타일의 그릇 아이콘이 여러 개 세로로 반복 배치되어 있다.
- 아래쪽 범례에도 같은 계열의 그릇 아이콘이 2종류 반복되어 사용된다.
- 반복 구조는 다음처럼 읽힌다.
  - 1칸 = 음식 이름 1개
  - 각 칸 위 = 그릇 아이콘의 반복 개수
  - 아래 = 범례 2개
- 음식 이름 중 `비빔밥`은 좁은 칸 때문에 두 줄로 줄바꿈되어 보인다.

[DSL 작성 힌트]
- 중심 도형은 연녹색 테두리의 큰 직사각형 표 1개로 잡는다.
- 표 내부는 세로 4칸으로 분할하고, 아래쪽에는 라벨 영역을 두는 구조가 적절하다.
- 각 칸의 구성:
  - 위쪽: 그릇 아이콘 여러 개를 세로로 정렬
  - 아래쪽: 음식 이름 텍스트를 중앙 정렬
- `비빔밥`은 실제 화면처럼 두 줄 텍스트로 배치하는 것이 좋다.
- 범례는 표 아래 중앙 부근에 두 개의 아이콘과 텍스트를 나란히 배치한다.
  - 큰 그릇 아이콘 + `100그릇`
  - 작은 그릇 아이콘 + `10그릇`
- 하단의 `( 김치찌개 , 비빔밥 , 냉면 )`은 별도 텍스트 객체로 분리하는 것이 좋다.
- `(정답)`과 `(해설)`도 별도 텍스트로 처리한다.
- 상단 문제문장은 번호 `12.`를 포함한 한 줄 텍스트로 배치한다.
- 표는 가운데 정렬에 가깝고, 좌우 여백이 비교적 넓다.
- 각 칸의 구분선과 바깥 테두리는 연녹색 계열로 보인다.

[semantic/solvable 힌트]
- 문제의 의미는 `된장찌개`와 비교하여 더 큰 그림 수를 가진 음식 이름을 고르는 것이다.
- 원본 하단에는 `(정답) 김치찌개, 냉면`이 인쇄되어 보인다.
- 다만 DSL 작성에서는 이를 계산 결과로 새로 추론하기보다, 보이는 출력 사실로만 기록하는 편이 안전하다.
- 해설 문구도 함께 보이므로, 원문 재현 시에는 해설 텍스트를 별도 요소로 넣을 수 있다.
- 그림그래프의 실제 수량 해석은 범례와 아이콘 개수 대응을 통해 이루어지지만, 정답을 새로 계산해 빈칸에 채우지는 않는다.

[주의 및 TODO]
- `된장찌개보다 더 많이 팔린 음식`이라는 질문 문장은 정확히 유지한다.
- `비빔밥`의 줄바꿈은 원본처럼 두 줄 표현으로 반영한다.
- 표 안의 그릇 아이콘은 완전히 동일한 도형이라기보다 약간 다른 크기/비율로 보일 수 있다.
- 하단 `( 김치찌개 , 비빔밥 , 냉면 )`이 보기 목록인지 별도 메모인지 문맥상 애매하므로, 텍스트 위치만 보존하고 기능은 단정하지 않는다.
- 그림 수는 해상도상 세부 판독이 헷갈릴 수 있으므로, 각 칸의 개수를 단정적으로 다시 계산한 설명은 피한다.
- 정답이 하단에 인쇄되어 보이지만, 이를 바탕으로 문제 해결 과정을 길게 쓰지 않는다.
- 추가로, 이 화면에는 등호 변환식이 없으므로 변환 단계 설명을 억지로 만들지 않는다.
