[문제 개요]
- 문제 번호는 **11.**로 보인다.
- 지시문은 **“직선을 선택하세요.”**이다.
- 화면에는 여러 개의 선 그림이 제시되어 있고, 그중 직선과 곡선을 구분하는 도형 판별 문제로 보인다.
- 하단에는 **“(정답)”** 표기와 **“(해설) 양쪽으로 끝없이 늘이게 곧은 선을 찾습니다.”**라는 문장이 함께 보인다.
- 원본 PNG 기준으로, 정답 표시 옆에 별도의 선 그림이 배치되어 있다.

[원본에 보이는 요소]
- 보이는 텍스트:
  - `11.`
  - `직선을 선택하세요.`
  - `(정답)`
  - `(해설) 양쪽으로 끝없이 늘이게 곧은 선을 찾습니다.`
- 그림 요소:
  - 위쪽 왼쪽: 수평에 가까운 곧은 선
  - 위쪽 가운데: 아래로 휘는 곡선
  - 위쪽 오른쪽: 오른쪽 위로 기울어진 곧은 선
  - 아래쪽 왼쪽: 왼쪽 아래에서 오른쪽 위로 올라가는 곧은 선
- 각 선의 양끝 근처에 작은 점 표시가 보인다.
- 전체 배치는 상단 텍스트 영역과 하단 도형 영역이 분리된 형태다.

[수식/관계 구조]
- 이 문제에는 수식, 등호 연쇄, 계산식이 보이지 않는다.
- 보이는 관계는 다음의 분류 관계이다.
  - 직선 후보: 위쪽 왼쪽, 위쪽 오른쪽, 아래쪽 왼쪽
  - 곡선 후보: 위쪽 가운데
- 수학적 변형 구조보다는 도형의 성질을 판단하는 구조이다.

[등호별 변환 의미]
- 해당 없음.
- 등호(`=`)가 포함된 식이나 변환 단계가 보이지 않는다.

[빈칸 정책]
- 빈칸 문제는 아니다.
- 따라서 추론값을 채워 넣지 말고, 보이는 요소만 그대로 유지한다.
- 만약 DSL에서 선택 결과를 표현하더라도, 이미지에 직접 인쇄된 표식만 반영하고 임의로 답을 추가하지 않는다.

[반복 패턴]
- 선 도형이 여러 개 반복 제시되어 있어, 서로 비교하며 고르는 구조이다.
- 반복되는 시각적 패턴:
  - 선 한가닥
  - 양끝의 작은 점 표시
  - 직선과 곡선의 대비
- 직선은 길이와 기울기만 다르고, 곡선은 중앙에서 휘어지는 형태로 구분된다.
- 하단의 `(정답)` 표시는 별도의 텍스트 블록으로 반복적인 문제-해설 레이아웃을 형성한다.

[DSL 작성 힌트]
- 상단 텍스트 영역과 하단 도형 영역을 분리해서 배치하는 구성이 적절하다.
- 문제 번호 `11.`과 지시문 `직선을 선택하세요.`는 좌상단에 한 줄 또는 두 줄 텍스트로 처리하면 좋다.
- 선 도형들은 각각 독립 객체로 다루고, 위치를 실제 화면처럼 흩어 배치한다.
- 각 선의 양끝에 있는 점은 작은 마커 또는 끝점 표시로 반영할 수 있다.
- 위쪽 가운데의 곡선은 다른 직선들과 확실히 구분되도록 곡률을 살려야 한다.
- `(정답)`과 `(해설)`은 하단의 별도 텍스트 블록으로 두고, 해설 문장은 한 줄로 길게 배치하는 것이 자연스럽다.
- 도형 사이 간격을 충분히 두어 선택지처럼 보이게 구성한다.

[semantic/solvable 힌트]
- 이 문항의 핵심 의미는 **직선과 곡선을 구별하는 것**이다.
- 해설 문장 **“양쪽으로 끝없이 늘이게 곧은 선을 찾습니다.”**는 직선의 정의를 설명하는 문장으로 읽힌다.
- 보이는 도형 중 곡선 1개와 직선 여러 개가 함께 제시되어, 시각적으로 분류하게 만든다.
- 다만 정답 선택을 추론해서 새로 쓰지는 말고, 화면에 있는 구성만 반영한다.

[주의 및 TODO]
- 원본 이미지와 초안이 일치하므로, 현재 서술은 큰 수정 없이 사용 가능해 보인다.
- `정답` 표기 옆의 아래쪽 왼쪽 선이 실제로 정답 예시인지, 단순 배치 요소인지에 대해서는 화면상 그렇게 보이지만 완전 단정은 피한다.
- 선 끝의 점이 실제 인쇄 요소인지, 편집 마커인지 해상도상 애매하다.
- 선의 정확한 길이, 각도, 곡률은 수치화하지 말고 시각적 인상 수준으로만 반영한다.
- 이 문제에는 수식 변형이 없으므로 등호별 설명이나 분수 변환 문장은 추가하지 않는다.
