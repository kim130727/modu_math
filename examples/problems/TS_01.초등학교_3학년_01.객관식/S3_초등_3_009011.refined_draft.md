[문제 개요]
- 문제 ID는 `50.`으로 보이며, 지시문은 `알맞은 말을 선택하세요.`이다.
- 화면은 좌측의 그림, 우측의 선택 문장 박스, 하단의 해설/정답 텍스트로 구성되어 있다.
- 정답이 하단에 **인쇄된 상태로 보이지만**, 새로 추론하지 말고 보이는 문구만 기록해야 한다.

[원본에 보이는 요소]
- 좌측 상단: `50. 알맞은 말을 선택하세요.`
- 좌측 그림:
  - 좌우에 점처럼 보이는 검은 표시 2개
  - 두 점 사이를 잇는 굽은 선 1개
  - 각 점 근처에 짧은 보조선/표시가 있음
- 우측 큰 박스:
  - 연보라색 테두리의 직사각형
  - 내부 문구:
    - `굽은 선이므로`
    - `( 반직선입니다. 반직선이 아닙니다 ).`
- 하단 설명:
  - `(해설)반직선은 한 점에서 시작해 한 쪽으로 끝없이 늘인 굵은 선입니다.`
- 하단 정답:
  - `(정답)반직선이 아닙니다`

[수식/관계 구조]
- 수식이나 등호 연쇄는 보이지 않는다.
- 계산식, 분수, 변형 관계는 없다.
- 이 문제는 개념 판별형 문장 선택 구조이다.
- 선택지 구조:
  - `반직선입니다.`
  - `반직선이 아닙니다.`
- 우측 박스의 문장은 원인-판단 형태로 읽힌다:
  - `굽은 선이므로` → 뒤의 판단 문장 선택
- 그림의 곡선은 개념 설명을 위한 시각 자료로 보인다.

[등호별 변환 의미]
- 해당 없음.
- 등호(`=`)가 있는 식이 나타나지 않는다.
- 변환 단계도 없다.

[빈칸 정책]
- 실제 빈칸 컴포넌트는 보이지 않는다.
- 따라서 빈칸을 추론해서 채우지 말고, 원문에 보이는 선택 문장 그대로 유지한다.
- 정답이 하단에 인쇄되어 보이더라도, 편집/입력용 빈칸 답으로 자동 대체하지 않는다.
- 선택지 두 문장은 모두 **보이는 텍스트**로만 취급하고, 하나를 단정적으로 대입하지 않는다.

[반복 패턴]
- `반직선입니다.` / `반직선이 아닙니다.`처럼 두 문장 중 하나를 고르는 패턴이 보인다.
- `(해설)`, `(정답)` 라벨이 하단에 반복적으로 쓰여 있다.
- 그림 요소도 좌측 점 2개와 곡선 1개라는 단순 반복 구조를 가진다.

[DSL 작성 힌트]
- 전체 레이아웃은 가로 분할로 보는 것이 적절하다:
  - 좌측: 그림
  - 우측: 큰 테두리 박스
  - 하단: 해설과 정답 텍스트
- 좌측 그림 렌더링 힌트:
  - 검은 점 2개를 좌우에 배치
  - 점 사이에 위로 볼록한 곡선을 1개 배치
  - 각 점 근처의 짧은 보조 표시가 있으므로 세부 선분을 추가할 수 있음
- 우측 박스 렌더링 힌트:
  - 연보라색 테두리의 직사각형
  - 내부 텍스트는 2줄
  - 첫 줄 `굽은 선이므로`
  - 둘째 줄은 괄호 포함 문장 `(... )`
  - 문장 간 세로 간격을 약간 넉넉하게 둠
- 하단 텍스트 렌더링 힌트:
  - `(해설)`와 설명 문장을 한 줄로
  - `(정답)`와 답 문장을 한 줄로
  - 두 줄은 아래쪽에 세로로 배치
- 표, 분수 슬롯, 수식 박스는 필요 없다.

[semantic/solvable 힌트]
- 이 문제는 “굽은 선”이 무엇인지에 대한 개념 판별 문제로 보인다.
- 보이는 해설 문장은 반직선의 정의를 설명하고 있다:
  - `한 점에서 시작해`
  - `한 쪽으로 끝없이 늘인 굵은 선`
- 다만 이 설명을 바탕으로 정답을 새로 유도하지 말고, 이미 보이는 정답 문구만 기록 대상으로 삼는다.
- 화면에 정답 문구가 인쇄되어 있으므로, DSL에서는 해당 텍스트를 관찰 사실로만 다루면 된다.
