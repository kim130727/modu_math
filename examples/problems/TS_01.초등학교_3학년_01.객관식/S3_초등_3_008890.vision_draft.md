# [전체적인 구성]
- 문제 번호 **54.**가 왼쪽 상단에 있고, 그 오른쪽으로 문제 문장이 한 줄~두 줄로 이어져 있습니다.
- 중앙에는 **표 형태의 그림그래프**가 크게 배치되어 있습니다.
- 표 오른쪽에는 **그림의 크기 기준을 설명하는 범례**가 세로로 놓여 있습니다.
- 표 아래에는 괄호로 묶은 선택지 문구가 있고, 그 아래에 **(정답)**, **(해설)** 문장이 이어집니다.
- 전체적으로 **문제 설명 → 그림그래프 → 범례 → 선택지/해설** 순서의 구조입니다.

# [보이는 텍스트]
- `54.`
- `진아네 마을의 농장에서 하루 동안 생신힌 토마토의 양을 조사하여 그림그래프로 나타내었습니다. 토마토 생산량이 다 붕장보다 적은 농장을 모두 선택해 보세요.`
  - ※ 문장에 보이는 철자는 그대로 적으면 `생신힌`, `붕장`처럼 보입니다. 다만 의도는 다를 수 있습니다.
- 표 제목: `농장별 토마토 생산량`
- 표 머리글:
  - `농장`
  - `토마토 생산량`
- 농장 이름: `가`, `나`, `다`, `라`, `마`
- 범례:
  - 큰 토마토 그림 옆: `10상자`
  - 작은 토마토 그림 옆: `1상자`
- 표 아래 괄호: `( 가 농장, 나 농장, 다 농장, 라 농장, 마 농장 )`
- 하단 문구:
  - `(정답)나 농장, 마 농장`
  - `(해설)큰 그림 수가 2개보다 적은 농장을 찾고, 큰 그림 수가 2개인 경우 작은 그림의 수가 3개보다 적은 농장을 찾으면 나 농장과 마 농장입니다.`

# [반복되는 시각 패턴]
- 각 농장 행마다 **토마토 그림이 여러 개 반복**되어 있습니다.
- 그림에는 **큰 토마토**와 **작은 토마토** 두 종류가 있으며, 크기가 수량 단위를 구분하는 데 사용됩니다.
- 각 행의 그림은 왼쪽에서 오른쪽으로 나란히 배열되어 있고, 큰 그림과 작은 그림이 함께 섞여 나타납니다.
- 범례에서도 큰 그림 1개와 작은 그림 1개가 각각 따로 제시되어, 표 안의 반복 패턴과 대응됩니다.
- 표의 각 행은 가로줄로 분리되어 있고, 농장 이름은 왼쪽 칸에 세로로 정렬되어 있습니다.

# [세부 내용]
- 표는 연한 초록색 배경과 테두리로 구성되어 있으며, 왼쪽 좁은 열은 농장 이름, 오른쪽 넓은 열은 토마토 그림이 들어가는 영역입니다.
- 표의 행은 위에서 아래로 `가`, `나`, `다`, `라`, `마` 순서입니다.
- 각 행의 토마토 그림은 모두 같은 기준으로 비교되지만, **큰 그림과 작은 그림의 조합 개수**가 다르게 보입니다.
- 범례는 표 오른쪽 바깥에 따로 배치되어 있어, 그림의 크기 차이를 해석하는 역할을 합니다.
- 표 아래의 괄호 안에는 선택 대상이 될 농장 이름들이 쉼표로 구분되어 나열되어 있습니다.
- 하단의 정답/해설 문장은 문제 풀이에 해당하므로, DSL 작성 시에는 보조 정보로만 참고할 수 있습니다.

# [Python DSL 작성 힌트]
- 표를 먼저 큰 박스로 만들고, 그 안에 **제목 영역**과 **2열×5행 구조**를 배치하면 좋습니다.
- 왼쪽 열에는 `가, 나, 다, 라, 마`를 세로로 정렬하고, 오른쪽 열에는 각 행마다 토마토 아이콘을 가로로 반복 배치합니다.
- 토마토 아이콘은 **두 가지 크기**가 있으므로, 큰 아이콘과 작은 아이콘을 서로 다른 이미지/도형으로 정의해야 합니다.
- 범례는 표 오른쪽에 별도 그룹으로 두고, 큰 토마토 그림 옆에 `10상자`, 작은 토마토 그림 옆에 `1상자`를 붙여야 합니다.
- 표 제목은 표 상단 중앙 정렬로 두는 것이 자연스럽습니다.
- 표 아래의 괄호 문구는 텍스트 박스로 별도 배치하면 좋습니다.
- 해설 문장은 문제 화면에 포함되어 있지만, 문제 본문 재현용 DSL과 해설용 DSL은 분리하는 편이 좋습니다.

# [주의할 점]
- **문제를 풀어 답을 새로 추론하지 마세요.**
- 토마토의 **크기 차이**를 수량 단위와 연결하는 것은 범례를 근거로만 표현해야 합니다.
- 표 안의 각 행에 있는 토마토 개수는 정확히 보이도록 배치해야 하지만, 이를 바탕으로 선택 결과를 새로 계산해 쓰지는 마세요.
- 문제 문장에 오타처럼 보이는 부분이 있어도, 화면에 보이는 대로 기록하되 필요하면 별도 주석으로만 처리하세요.
- `(정답)`과 `(해설)`은 화면에 존재하는 텍스트이지만, 시각적 요소라기보다 보조 문구입니다.

# [불확실한 부분]
- 문제 문장 중 `생신힌`, `붕장`은 원문 오타인지, 이미지 판독의 한계인지 확실하지 않습니다.
- 표 안의 토마토 그림 개수는 보이지만, 각 그림이 정확히 어떤 수량을 뜻하는지 계산하는 단계는 여기서 단정하지 않습니다.
- 하단의 `(해설)` 문장은 글자 크기와 줄바꿈이 화면에서 약간 겹쳐 보일 수 있어, DSL에서 줄 수를 조정할 필요가 있습니다.
