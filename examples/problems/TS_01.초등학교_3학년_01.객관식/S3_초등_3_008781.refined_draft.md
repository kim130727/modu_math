[문제 개요]
- 초등 수학/비교 문제로 보이며, 고구마·옥수수·감자 3개의 **한 개 무게를 비교**하여 **무거운 것부터 순서대로** 고르는 형식이다.
- 화면 상단에 문제 문장이 2줄로 배치되어 있다.
- 가운데에는 저울 그림 2개가 좌우로 나란히 배치되어 있고, 아래에는 선택지 4개와 정답/해설 텍스트가 있다.
- 원본 이미지에는 `(정답) ④`와 `(해설)` 문장이 **실제로 인쇄되어 보인다**.

[원본에 보이는 요소]
- 상단 문제 문장:
  - `1. 고구마, 옥수수, 감자 한 개의 무게를 비교하려고 합니다. 무거운 것부터 순서대로 쓴 것을 고르세요.`
- 중앙 그림:
  - 왼쪽 저울 1개
  - 오른쪽 저울 1개
- 선택지:
  - `① 고구마, 감자, 옥수수`
  - `② 감자, 고구마, 옥수수`
  - `③ 옥수수, 감자, 고구마`
  - `④ 옥수수, 고구마, 감자`
- 하단 표기:
  - `(정답) ④`
  - `(해설) 고구마와 옥수수 중 옥수수가 더 무겁고 고구마와 감자 중 고구마가 더 무겁습니다. 따라서 무거운 순서대로 쓰면 옥수수, 고구마, 감자입니다.`

[수식/관계 구조]
- 직접적인 숫자식, 등호 연쇄, 계산식은 보이지 않는다.
- 보이는 관계는 **비교 관계 2개**이다.
  - 왼쪽 저울: 고구마와 옥수수의 무게 비교
  - 오른쪽 저울: 고구마와 감자의 무게 비교
- 이 문제는 두 비교 결과를 종합하여 3개의 대상의 상대적 순서를 고르는 구조로 보인다.
- 보기 4개는 3개 물체의 순서 배열을 서로 다르게 제시하는 선택지 구조이다.

[등호별 변환 의미]
- 해당 없음.
- 등호가 포함된 변형 과정이나 식의 단계 변화는 보이지 않는다.

[빈칸 정책]
- 빈칸 문제는 아니다.
- 따라서 빈칸을 추론값으로 채우지 않는다.
- 원본에 이미 보이는 선택지, 정답 표시, 해설 텍스트는 **그대로 관찰 사실로만 기록**한다.
- 만약 DSL에서 빈칸 슬롯이 필요하더라도, 원본에 없는 값은 임의로 생성하지 않는다.

[반복 패턴]
- 동그라미 숫자 선택지 `①~④`가 반복된다.
- 저울 그림이 2개 반복되며, 구조가 매우 유사하다.
  - 받침대
  - 중심 지지대
  - 좌우 접시
  - 접시 위 물체 그림
- 보기 문장들은 모두 **물체 이름 3개를 쉼표로 나열**하는 패턴이다.
- 해설 문장은 비교 결과 2개를 먼저 말한 뒤, 마지막에 전체 순서를 한 번 더 정리하는 구조다.

[DSL 작성 힌트]
- 상단 문제 문장은 좌측 정렬된 텍스트 블록으로 처리하는 것이 자연스럽다.
- 중앙에는 저울 그림 2개를 **가로로 나란히 배치**한다.
- 각 저울은 이미지 한 덩어리로 둘 수도 있고, 필요하면 다음 요소로 분해해 표현할 수 있다.
  - 받침대
  - 중앙 지지대
  - 좌/우 접시
  - 접시 위 물체
- 왼쪽 저울에는:
  - 왼쪽 접시: 고구마
  - 오른쪽 접시: 옥수수
- 오른쪽 저울에는:
  - 왼쪽 접시: 고구마
  - 오른쪽 접시: 감자
- 선택지 영역은 **2열 2행** 배치가 적절하다.
  - 왼쪽 열: ①, ③
  - 오른쪽 열: ②, ④
- 하단의 `(정답) ④`와 `(해설)`은 본문과 분리된 설명 영역으로 둔다.
- 보기의 문장은 텍스트만 정확히 옮기고, 정답 판정 로직은 별도로 두지 않는다.
- 저울의 기울기 방향은 그림 의미를 해석하는 데 도움이 되지만, 정확한 수치로 환산하지는 않는다.

[semantic/solvable 힌트]
- 보이는 비교를 의미적으로 해석하면:
  - 옥수수는 고구마보다 무겁다.
  - 고구마는 감자보다 무겁다.
- 따라서 전체 순서는 무거운 것부터 가벼운 것 순으로 정렬하는 문제이다.
- 다만 이는 **문제 풀이 방향의 의미 해석**이며, DSL 작성 시에는 원본에 보이는 관계만 우선 기록하는 것이 안전하다.
- 원본 화면에는 정답 표시 `④`가 보이므로, 이를 시각적 사실로 기록할 수 있다.
