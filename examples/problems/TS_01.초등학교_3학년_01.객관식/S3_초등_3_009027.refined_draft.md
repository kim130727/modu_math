[문제 개요]
- 문제 번호는 `64.`로 보인다.
- 지시문은 `삼각형을 보고 알맞은 기호를 모두 선택하세요.`로 읽힌다.
- 화면 상단에는 삼각형 4개가 가로로 배치되어 있고, 각 삼각형 근처에 한글 기호 `가`, `나`, `다`, `라`가 각각 붙어 있다.
- 아래에는 `(3)`으로 시작하는 문장이 2~3줄 보이며, 그중 일부는 선택 결과처럼 보인다.
- 전체 형식은 도형을 보고 해당 기호를 고르는 선택형 문제로 보인다.

[원본에 보이는 요소]
- 상단 좌측에 문제 번호 `64.`가 있다.
- 제목/지시문: `삼각형을 보고 알맞은 기호를 모두 선택하세요.`
- 삼각형 도형 4개가 가로 방향으로 나열되어 있다.
- 각 도형 위쪽 근처에 라벨이 하나씩 보인다:
  - `가`
  - `나`
  - `다`
  - `라`
- 하단 문장들:
  - `(3) 직각삼각형은 ( 가, 나, 다, 라 )입니다.`
  - `(3) 직각삼각형은 나, 라입니다.`
  - `(3) 나, 라`
- 원본 이미지에서도 위 텍스트들이 순서대로 보이며, 마지막 줄은 축약된 형태처럼 보인다.

[수식/관계 구조]
- 등호(`=`)가 있는 수식은 보이지 않는다.
- 수식 변형 연쇄도 보이지 않는다.
- 관찰 가능한 구조는 다음과 같다:
  - `직각삼각형은 ( 가, 나, 다, 라 )입니다.` 형태의 목록 제시
  - 이어서 `직각삼각형은 나, 라입니다.` 형태의 결과처럼 보이는 문장
- 도형-라벨 대응 구조:
  - 도형 4개 ↔ 라벨 4개가 1:1로 배치되어 있다.
- 괄호 안에 여러 기호를 쉼표로 나열한 형태가 보인다.

[등호별 변환 의미]
- 해당 없음.
- 등호를 이용한 변형 단계가 보이지 않으므로 단계별 변환 의미를 설명할 대상이 없다.

[빈칸 정책]
- 빈칸이나 답안으로 보이는 부분이 있더라도 추론값을 채우지 않는다.
- 원본에 정답이 명확히 인쇄되어 보이는 경우에만, 그 사실을 관찰 사실로 기록한다.
- 현재 화면에서 `나, 라`가 보이지만, 이것을 최종 정답으로 단정해서 채우는 방식은 피한다.
- DSL 작성 시에도 빈칸은 빈칸 상태를 유지하는 방향이 적절하다.

[반복 패턴]
- 삼각형 도형이 4개 반복되어 가로로 배치되어 있다.
- 각 도형마다 위쪽 근처에 기호 1개가 반복적으로 대응된다.
- 아래쪽 문장에서도 `(3)`이 반복되어 나타난다.
- 괄호 `(` `)`와 쉼표 `,`가 포함된 목록형 표현이 반복된다.
- 마지막 줄은 짧은 답안 후보처럼 보이는 축약 표현이다.

[DSL 작성 힌트]
- 상단 좌측에 `64.`와 지시문을 한 줄 또는 두 줄로 배치하는 구성이 자연스럽다.
- 도형 영역은 가로 4열 구조로 잡는 것이 좋다.
- 각 삼각형은 별도의 선분 3개로 구성된 폴리곤으로 처리하면 된다.
- 삼각형 위쪽 근처에 `가`, `나`, `다`, `라` 텍스트를 각각 독립적으로 배치한다.
- 도형과 라벨의 상대적 위치를 유지하는 것이 중요하다.
- 아래 문장은 일반 텍스트 줄로 배치한다.
- 괄호 문장은 실제 보이는 기호 순서와 구두점 위치를 유지한다.
- 마지막 줄 `(3) 나, 라`는 축약 표기처럼 보이므로, 원문 문자열 자체를 보존하는 편이 좋다.
- 렌더링 시 도형 간 간격은 비슷하게 유지하고, 각 라벨이 해당 도형 가까이에 오도록 정렬한다.

[semantic/solvable 힌트]
- 문제의 의도는 삼각형들 중 어떤 것이 직각삼각형인지 고르는 것으로 보인다.
- 다만 여기서는 정답을 추론하지 않는다.
- 화면에 보이는 텍스트만 기준으로 보면, 아래에 `나, 라`가 노출되어 있어 선택 결과처럼 읽힐 수 있다.
- 그러나 이것이 최종 정답인지, 예시인지, OCR 산출물인지 단정하지 않는다.
- 도형의 모양은 시각적으로 직각처럼 보이는 것과 아닌 것이 섞여 있다.

[주의 및 TODO]
- 정답을 추론해서 넣지 말 것.
- 빈칸은 임의로 채우지 말 것.
- `직각삼각형은 나, 라입니다.`는 보이는 텍스트일 뿐, 정답 확정 문장으로 단정하지 말 것.
- 마지막 줄 `(3) 나, 라`는 일부가 잘린 표시일 가능성이 있으므로 TODO로 남겨 둘 것.
- 두 번째, 네 번째 삼각형의 직각 여부는 화면상 그렇게 보이지만 해상도 때문에 100% 단정하지 말 것.
- 각 기호 `가/나/다/라`의 정확한 도형 대응은 배치상 관찰 가능하나, 약간의 위치 어긋남 가능성은 TODO로 남겨 둔다.
- 등호 변환식은 없으므로 관련 DSL 설명은 불필요하다.
