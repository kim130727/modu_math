[전체적인 구성]
- 화면 상단 왼쪽에 문제 번호와 문장이 있고, 그 아래 중앙에 분홍색 제목 상자가 있습니다.
- 중앙에는 큰 표가 있으며, 왼쪽과 오른쪽이 같은 형식으로 나뉘어 있습니다.
- 표 아래 오른쪽에는 그림 범례가 있고, 더 아래 왼쪽에는 정답과 해설 문장이 작게 배치되어 있습니다.
- 전체적으로 “가게별 일주일 동안 팔린 막대사탕 수”를 그림으로 나타낸 표를 읽고, 조건에 맞는 가게를 고르는 형태입니다.

[보이는 텍스트]
- 상단 문제 문장:
  - `□ 27. 가게별 일주일 동안 팔린 막대사탕 수를 조사하여 그림그래프로 나타내었습니다. 나 가게보다 막대사탕을 더 많이 판 가게를 모두 선택하세요.`
- 표 제목:
  - `일주일 동안 팔린 막대사탕 수`
- 표 머리글:
  - 왼쪽 묶음: `가게`, `막대사탕 수`
  - 오른쪽 묶음: `가게`, `막대사탕 수`
- 가게 이름:
  - 왼쪽: `가`, `다`, `마`
  - 오른쪽: `나`, `라`, `바`
- 범례:
  - 그림 아이콘 옆에 `100개`
  - 작은 그림 아이콘 옆에 `10개`
  - 아래에 `( 가, 나, 다, 라 ) 가게`
- 하단 텍스트:
  - `(정답)다, 라`
  - `(해설)나 가게에서 팔린 막대사탕 수는 240개이므로 240개보다 많은 가게를 찾습니다. 따라서 나 가게보다 막대사탕을 더 많이 판 가게는 다 가게와 라 가게입니다.`

[반복되는 시각 패턴]
- 표가 좌우로 동일한 구조를 반복합니다.
  - 왼쪽 3행, 오른쪽 3행의 가게 목록이 같은 형식으로 배치됨.
  - 각 행마다 가게 이름 1개와 그림 아이콘들이 나란히 놓임.
- 그림 아이콘이 두 종류 반복됩니다.
  - 큰 막대사탕 모양 1개 = 100개
  - 작은 막대사탕 모양 1개 = 10개
- 같은 종류의 아이콘이 여러 개 연속으로 배치되어 수량을 나타냅니다.
- 표의 셀 안에서 아이콘들은 왼쪽에서 오른쪽으로 가로로 정렬되어 있습니다.

[수식 단계]
- 표 자체에는 등호로 이어지는 계산식은 보이지 않습니다.
- 다만 하단 해설 문장에 수량 비교의 결과로 `240개`가 언급됩니다.
- 이 문제 이미지에서는 구체적인 계산식 전개는 나타나지 않고, 비교 결과만 문장으로 제시되어 있습니다.

[등호별 변환 의미]
- 해당 없음
- 화면에 `=` 형태의 수식 변형 단계가 보이지 않습니다.

[세부 내용]
- 표는 연한 분홍색 테두리의 큰 직사각형 형태입니다.
- 가운데 세로선으로 좌우 두 묶음이 나뉘어 있고, 각 묶음은 다시 `가게 / 막대사탕 수` 두 열로 구성됩니다.
- 각 가게 행에는:
  - 왼쪽 열에 가게 이름
  - 오른쪽 열에 해당 수량을 나타내는 그림 그래프가 들어 있습니다.
- 그림 그래프는 두 가지 크기의 사탕 그림으로 구성되어 있습니다.
  - 큰 사탕 그림은 원형 막대사탕처럼 보이며, 범례상 100개를 뜻합니다.
  - 작은 사탕 그림은 더 작은 막대사탕 모양이며, 범례상 10개를 뜻합니다.
- 오른쪽 아래의 범례는 큰 그림과 작은 그림이 서로 아래로 약간 어긋나게 배치되어 있고, 각 그림 옆에 수량 텍스트가 붙어 있습니다.
- 범례 아래에는 `( 가, 나, 다, 라 ) 가게`라는 문구가 있어, 선택 대상이 4개의 가게임을 보여 줍니다.
- 하단 정답/해설 영역은 본문보다 작은 글씨이며, 본문과 분리되어 있습니다.

[Python DSL 작성 힌트]
- 표 구조를 먼저 잡는 것이 좋습니다.
  - 큰 바깥 테두리 1개
  - 상단 제목 박스 1개
  - 내부를 좌우 2개 구획으로 나누기
  - 각 구획 안에 `가게`, `막대사탕 수` 2열 배치
- 그림그래프는 셀 안에 아이콘 반복으로 표현할 수 있습니다.
  - 큰 아이콘과 작은 아이콘을 서로 다른 종류의 도형/이미지로 처리
  - 각 행마다 아이콘 개수를 다르게 배치
- 범례는 표 오른쪽 아래에 별도 텍스트와 아이콘으로 구성
- 정답 표시와 해설은 문제 본문과 분리된 하단 텍스트 박스로 처리
- 좌우 대칭형 표이므로, 같은 셀 스타일을 반복 재사용하면 좋습니다.

[주의할 점]
- `나` 가게의 실제 수량이나 다른 가게의 총개수를 이 이미지 정보만으로 임의로 계산해 채우지 마세요.
- 그림 아이콘의 개수를 잘못 세지 않도록 주의해야 합니다.
- 정답 `(다, 라)`는 이미지에 보이지만, DSL용 시각 분석에서는 정답을 새로 계산해서 확정하는 식으로 쓰지 말고, 보이는 텍스트로만 다루는 것이 안전합니다.
- 표의 오른쪽 영역과 왼쪽 영역을 혼동하지 않도록 주의하세요.
- 범례의 큰 아이콘과 작은 아이콘 크기 차이를 명확히 구분해야 합니다.

[불확실한 부분]
- 각 그림 아이콘이 실제로 몇 개인지 일부는 이미지 해상도 때문에 세밀하게 세기 어려울 수 있습니다.
- 큰 아이콘과 작은 아이콘의 정확한 상대 크기나 간격은 이미지에서 약간 흐릿하게 보일 수 있습니다.
- 표의 선 색은 연분홍색으로 보이나, 실제 코드 구현 시 정확한 색상값은 불확실합니다.
