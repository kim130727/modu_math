[문제 개요]
- 흰색 배경의 초등 수학/도형 문제로 보인다.
- 같은 문제 문장이 위아래로 2번 반복되어 있다.
- 각 문제 문장 아래에 삼각형 그림 4개가 가로로 배치되어 있고, 각 그림 위에 기호 `가, 나, 다, 라`가 붙어 있다.
- 아래쪽 블록에는 설명 문장과 `(2)`에 대한 답처럼 보이는 텍스트가 추가로 보인다.
- 전체 구성은 “그림을 보고 알맞은 기호를 모두 고르기” 유형으로 해석된다.

[원본에 보이는 요소]
- 상단 문제 문장: `□ 64. 삼각형을 보고 알맞은 기호를 모두 선택하세요.`
- 하단에 같은 문장이 한 번 더 반복됨: `□ 64. 삼각형을 보고 알맞은 기호를 모두 선택하세요.`
- 각 블록의 그림 위 라벨:
  - 첫 번째: `가`
  - 두 번째: `나`
  - 세 번째: `다`
  - 네 번째: `라`
- 아래쪽 설명 문장: `(2) 한 각이 직각인 삼각형은 나, 라입니다.`
- 맨 아래 텍스트: `(2) 나, 라`

[수식/관계 구조]
- 등호, 분수, 계산식은 보이지 않는다.
- 수학적 관계는 “삼각형의 성질을 판별해서 해당 기호를 고르는” 분류 구조이다.
- 아래쪽 문장 `(2) 한 각이 직각인 삼각형은 나, 라입니다.` 는 기호 `나`, `라`를 조건에 맞는 삼각형으로 연결하는 언어적 관계를 나타낸다.

[등호별 변환 의미]
- 등호 연쇄나 식 변형은 없다.
- 변환 단계 설명 불필요.
- 문장형 답안만 보이며, 수식적인 변환은 관찰되지 않는다.

[빈칸 정책]
- 빈칸을 추론으로 채우지 않는다.
- 원본에 인쇄되어 보이는 텍스트만 그대로 반영한다.
- `□` 기호는 체크박스일 수도 있고 편집 표식일 수도 있으므로, 빈칸/체크 영역으로만 취급하고 추가 의미를 단정하지 않는다.
- `(2) 나, 라`는 인쇄된 답안인지 학생 작성 예시인지 불명확하므로, 추정값을 넣지 않고 보이는 그대로 유지한다.

[반복 패턴]
- 동일한 문제 문장이 상하로 2회 반복된다.
- 각 반복마다 4개의 삼각형이 같은 순서로 왼쪽에서 오른쪽으로 배열된다.
- 각 삼각형 위의 라벨도 동일한 순서 `가, 나, 다, 라`를 따른다.
- 두 번째 블록은 첫 번째 블록과 거의 같은 레이아웃이지만, 아래에 설명 문장과 답 텍스트가 추가되어 있다.
- 두 블록의 삼각형은 매우 비슷하지만, 선의 기울기와 꼭짓점 위치가 약간 달라 보일 수 있다.

[DSL 작성 힌트]
- 상단과 하단에 동일한 문제 문장을 각각 별도 텍스트 요소로 배치하면 좋다.
- 각 블록은 다음처럼 구성하면 적절하다:
  - 문제 번호/지시문 텍스트 1줄
  - 그 아래 가로 4열의 삼각형 도형 배열
  - 각 도형 상단에 라벨 `가, 나, 다, 라`
- 삼각형은 내부 채움 없이 외곽선만 있는 단순 선분 도형으로 표현한다.
- 도형들 사이 간격을 일정하게 두고, 각 삼각형의 기준선 높이를 맞춰 정렬한다.
- 아래쪽 블록에는 추가로 문장 `(2) 한 각이 직각인 삼각형은 나, 라입니다.` 를 별도 텍스트 줄로 넣는다.
- 맨 아래 `(2) 나, 라`는 답안 표기인지 확실하지 않으므로, 별도 텍스트 요소로 두되 정답 처리용으로 단정하지 않는 것이 안전하다.
- 체크박스처럼 보이는 `□`는 문제 번호 앞의 표시로 함께 렌더링하되, 실제 체크 상태 요소로 해석하지 않는 편이 안전하다.
- 삼각형은 모두 회전/기울기가 있는 외곽선 도형이므로, 사각형이나 채움 도형으로 대체하지 않는다.

[semantic/solvable 힌트]
- 보이는 문장만 놓고 보면, 사용자는 “직각이 있는 삼각형”에 해당하는 기호를 고르는 문제를 풀고 있다.
- 다만 이 단계에서는 정답을 새로 추론하지 않는다.
- 원본에 이미 `(2) 한 각이 직각인 삼각형은 나, 라입니다.` 와 `(2) 나, 라` 가 보이므로, 이는 관찰 가능한 텍스트 사실로만 다룬다.
- 삼각형 도형의 정확한 직각 여부는 이미지 해상도와 선 기울기 때문에 완전 단정이 어렵다.
