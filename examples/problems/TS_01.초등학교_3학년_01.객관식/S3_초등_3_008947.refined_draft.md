[문제 개요]
- 초등 수학/자료 해석 문제로 보임.
- 화면 상단에 문제 번호와 질문 문장이 있고, 중앙에는 `학생들이 좋아하는 운동`이라는 제목의 표가 있음.
- 표 아래에는 운동 아이콘 범례가 있으며, 하단에는 `(정답)`, `(해설)`이 인쇄되어 있음.
- 이 화면은 **자료표를 보고 해당 학생의 좋아하는 운동을 찾는 유형**으로 읽힘.

[원본에 보이는 요소]
- 문제 문장:
  - `□ 17. 서진이는 반 학생들이 좋아하는 운동을 조사하였습니다. 좋아하는 운동은 무엇일까요?`
- 제목 상자:
  - `학생들이 좋아하는 운동`
- 표 안에 보이는 학생 이름:
  - `서진`, `희재`, `서연`, `인철`
  - `성민`, `은수`, `주영`, `정훈`
  - `재영`, `수호`, `희재`, `해주`
  - `영은`, `승우`, `윤아`, `준서`
- 범례 텍스트:
  - `축구`
  - `야구`
  - `농구`
  - `배구`
- 범례 괄호 표기:
  - `( 축구 , 야구 , 농구 , 배구 )`
- 하단 인쇄 텍스트:
  - `(정답)축구`
  - `(해설)조사한 자료에서 재영이가 좋아하는 운동을 찾아보면 축구입니다.`

[수식/관계 구조]
- 등호나 계산식은 보이지 않음.
- 대신 표의 각 칸은 **아이콘 ↔ 학생 이름**의 대응 관계를 나타내는 자료 구조로 읽힘.
- 화면에서 보이는 대응은 다음과 같이 해석될 수 있음.
  - 각 열에서 위 칸의 운동 아이콘이 아래 이름의 학생이 좋아하는 운동을 표시하는 형태.
- 아이콘 종류는 4개:
  - 축구공 모양
  - 야구 글러브/공 모양
  - 농구공 모양
  - 배구공 모양

[등호별 변환 의미]
- 해당 없음.
- 등호 연결식, 계산식, 식의 변형 단계는 보이지 않음.

[빈칸 정책]
- 이 화면에는 별도의 빈칸 입력란이 보이지 않음.
- 따라서 빈칸을 추론값으로 채우지 말고, **보이는 그대로의 텍스트와 배치만** 반영해야 함.
- 만약 DSL에서 빈칸/placeholder를 쓰는 구조라면, 원문에 없는 값은 넣지 말고 공란으로 유지해야 함.

[반복 패턴]
- 표는 `4열`로 보이며, 각 열마다
  - 위쪽: 운동 아이콘
  - 아래쪽: 학생 이름
  의 2단 구성 패턴이 반복됨.
- 전체적으로 `아이콘 행 + 이름 행`이 번갈아 나타나는 구조가 4번 반복되는 형태로 읽힘.
- 표 아래 범례는 `아이콘 + 운동 이름`이 좌에서 우로 4개 나열된 반복 패턴임.

[DSL 작성 힌트]
- 상단 문제 문장은 좌측 정렬로 배치.
- 그 아래 중앙에 연한 파란색 배경의 제목 박스 배치.
- 제목 박스 내부에 `학생들이 좋아하는 운동` 텍스트를 넣음.
- 제목 아래에 파란색 테두리의 표를 배치.
- 표는 4열 구성으로, 각 열에서
  - 첫 행: 운동 아이콘
  - 둘째 행: 학생 이름
  이 정렬되도록 구성.
- 표의 각 칸은 단순 나열이 아니라 **격자 형태의 정렬**이 핵심.
- 표 아래에는 범례를 가로 방향으로 배치:
  - 아이콘
  - 운동명 텍스트
  를 한 세트로 4개 나란히 둠.
- 하단의 `(정답)`, `(해설)`은 본문과 분리된 작은 설명 영역처럼 배치.
- 제목 상자, 표 테두리, 범례 간격은 전체적으로 정돈된 학습지 스타일로 맞추는 것이 자연스러움.

[semantic/solvable 힌트]
- 이 문제는 표에서 특정 학생이 좋아하는 운동을 찾는 자료 해석 문제로 보임.
- 화면 하단의 정답/해설 문구는 원본에 인쇄된 정보로 관찰 가능함.
- 다만 일반적인 DSL 작성에서는 정답을 새로 추론해 입력하지 말고, **이미 보이는 텍스트만 반영**하는 것이 원칙.
- 표와 범례의 대응을 통해 학생-운동 연결을 읽는 구조임.

[주의 및 TODO]
- 하단에 인쇄된 `(정답)축구`, `(해설)...`은 원본에 **보이는 사실**로만 기록하고, 별도 추론 결과처럼 다루지 말 것.
- 표 안의 아이콘 종류는 비슷해 보일 수 있으므로, 축구/야구/농구/배구를 혼동하지 않도록 주의.
- 범례의 괄호 안 공백 배치는 이미지에서 다소 넓어 보이므로, DSL에서는 시각적 유사성을 우선하되 글자 간격은 약간 달라도 됨.
- 표의 정확한 선 굵기와 색상은 해상도에 따라 차이가 있을 수 있으므로 TODO로 남겨도 됨.
- 학생 이름 `희재`가 표 안에 두 번 보이는 점은 원본 그대로 유지해야 하며, 오타로 단정하지 말 것.
- 이미지와 초안이 다를 경우, 최종 구성은 **이미지 기준**으로 우선 교정해야 함.
