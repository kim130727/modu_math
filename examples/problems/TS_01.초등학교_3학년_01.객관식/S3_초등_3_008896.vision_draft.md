# 시각 분석 초안

[전체적인 구성]
- 화면 상단에는 문제 번호 `61.`과 함께 문장형 지시문이 한 줄 이상으로 배치되어 있습니다.
- 중앙에는 제목이 있는 **표(그림그래프 형태)**가 크게 배치되어 있습니다.
- 표 오른쪽에는 책의 크기를 나타내는 **범례 그림 2개**가 세로로 배치되어 있습니다.
- 표 아래에는 **객관식 보기 4개**가 2열 형태로 배치되어 있고, 왼쪽 아래에 정답 표시와 해설 문장이 보입니다.
- 전체적으로 “문제 문장 → 그림그래프 → 보기 → 정답/해설”의 순서로 구성되어 있습니다.

[보이는 텍스트]
- 상단 문제 문장:
  - `61. 승우네 모둠 학생들이 1년 동안 읽은 책의 수를 조사하여 그림그래프로 나타내었습니다. 책을 가장 많이 읽은 사람과 둘째로 많이 읽은 사람을 바르게 나열한 것을 고르세요.`
- 표 제목:
  - `학생들이 1년 동안 읽은 책의 수`
- 표 머리글:
  - `이름`
  - `책의 수`
- 이름 항목:
  - `승우`
  - `정원`
  - `다경`
  - `준혁`
- 범례:
  - 큰 책 그림 옆 `10권`
  - 작은 책 그림 옆 `1권`
- 보기:
  - `① 정원, 준혁`
  - `② 준혁, 정원`
  - `③ 준혁, 다경`
  - `④ 승우, 다경`
- 정답 표기:
  - `(정답)③`
- 해설 문장:
  - `(해설)52>50>45>30이므로 책을 가장 많이 읽은 사람부터 순서대로 이름을 쓰면 준혁, 다경, 정원, 승우입니다.`  
  - 단, 이 해설은 문제 풀이 결과를 포함하므로 시각적으로는 보이지만 분석 초안에서는 사실로만 참고합니다.

[반복되는 시각 패턴]
- 각 사람 이름 왼쪽에 해당 행이 있고, 오른쪽 칸에는 **책 아이콘이 반복적으로 배치**되어 있습니다.
- 책 아이콘은 두 종류가 반복됩니다.
  - 큰 보라색 책 그림: 범례상 `10권`
  - 작은 보라색 책 그림: 범례상 `1권`
- 같은 행 안에서 큰 책과 작은 책이 **가로 방향으로 연속 나열**되어 있어 수량을 그림으로 표시한 구조입니다.
- 표 전체는 주황색 선으로 칸이 구분된 **격자형 표**입니다.
- 보기 번호는 파란색 동그라미 숫자(①~④)로 반복 표시됩니다.

[수식 단계]
- 해당 없음.  
- 표 안에 등호로 연결된 수식 변형은 보이지 않습니다.

[등호별 변환 의미]
- 해당 없음.  
- 수식을 단계적으로 변형한 장면이 없습니다.

[세부 내용]
- 표는 2열 구조입니다.
  - 왼쪽 열: 이름
  - 오른쪽 열: 책의 수를 그림으로 표현
- 이름은 위에서 아래로 `승우, 정원, 다경, 준혁` 순으로 배열되어 있습니다.
- 각 행의 오른쪽에는 책 그림이 서로 다른 개수로 놓여 있습니다.
  - 승우 행: 큰 책 그림만 몇 개 연속 배치
  - 정원 행: 큰 책 그림 여러 개 뒤에 작은 책 그림 여러 개가 이어짐
  - 다경 행: 큰 책 그림 여러 개가 연속 배치
  - 준혁 행: 큰 책 그림 여러 개 뒤에 작은 책 그림 몇 개가 이어짐
- 오른쪽 범례는 책의 크기와 권수를 대응시키는 역할입니다.
  - 큰 책 1개가 `10권`
  - 작은 책 1개가 `1권`
- 보기 4개는 이름 2개씩을 콤마로 나열한 형태이며, “가장 많이 읽은 사람”과 “둘째로 많이 읽은 사람”의 순서를 고르게 되어 있습니다.
- 문제의 핵심은 **그림그래프를 해석하여 순서를 고르는 선택형 문항**이라는 점입니다.

[Python DSL 작성 힌트]
- 표 영역은 하나의 큰 사각형 컨테이너로 묶고, 내부를 2열 5행 정도의 격자로 나누면 좋습니다.
- 좌측 열에는 텍스트 라벨, 우측 열에는 책 아이콘 반복 배열을 넣습니다.
- 범례는 표 오른쪽에 세로 정렬된 두 개의 아이콘+텍스트 조합으로 배치합니다.
- 보기 영역은 표 아래쪽에 2열 레이아웃으로 배치하고, 각 보기 앞에는 파란색 원형 번호를 둡니다.
- 상단 문제 문장은 좌측 정렬로 두 줄 정도로 줄바꿈 처리하는 것이 자연스럽습니다.
- 책 아이콘은 크기가 두 종류이므로, DSL에서는 `big_book`, `small_book` 같은 요소를 분리해서 반복 배치하는 방식이 편합니다.
- 표 내부 아이콘은 정확한 개수를 맞추되, 분석 초안 단계에서는 개수를 추정해 쓰지 말고 “여러 개”, “몇 개”처럼 기록해 두는 것이 안전합니다.

[주의할 점]
- 책 아이콘의 정확한 개수는 최종 DSL 작성 시 반드시 이미지 기준으로 다시 세어야 합니다.
- 해설에 적힌 수치나 순서를 그대로 문제 본문 요소로 오해하지 않도록 주의해야 합니다.
- 보기의 순서와 정답 표시는 화면에 보이지만, 초안에서는 답을 재작성하거나 강조하지 않는 것이 좋습니다.
- 표의 선 색상은 주황색 계열이며, 배경은 연한 베이지/살구색 느낌입니다.
- 책 그림은 행마다 간격이 다소 다를 수 있으므로, 일정한 등간격 정렬을 따로 맞추는 편이 좋습니다.

[불확실한 부분]
- 각 행의 책 아이콘 정확한 수는 이미지 확대 없이 확정하기 어렵습니다.
- 큰 책과 작은 책의 실제 픽셀 크기 차이, 그리고 행 내 간격은 화면 해상도에 따라 약간 달라 보일 수 있습니다.
- 표 제목의 정확한 중앙 정렬 위치와 글자 크기는 이미지 기준으로만 대략 파악됩니다.
- 해설 문장의 숫자 관계는 보이지만, 본 초안에서는 문제 풀이가 아니므로 구조 참고용으로만 남깁니다.
