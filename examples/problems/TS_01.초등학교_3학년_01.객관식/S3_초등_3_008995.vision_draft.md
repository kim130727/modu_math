- 해설 문장과 예시 도형이 함께 반복적으로 “정사각형의 조건”을 시각적으로 강조한다.

# [세부 내용]
- 상단 문제 문장은 왼쪽에서 오른쪽으로 한 줄에 배치되어 있다.
- 문제 번호 `33.` 다음에 지시문이 이어진다.
- 지시문 문장 중 `○`, `×`는 선택해야 할 기호로 보이며, 글자 크기와 색이 본문과 비슷하게 표시된다.
- 중앙의 도형은 정사각형/직사각형처럼 보일 수 있는 사각형 1개이며, 테두리는 연한 회색이고 내부는 비어 있다.
- 도형 아래에는 파란색 `×` 기호가 따로 표시되어 있어, 이 도형이 정사각형이 아님을 선택하는 형식의 화면처럼 보인다.
- 왼쪽 중간에는 파란색 원형 표시가 하나 있고, 그 아래에 `(정답)` 텍스트와 함께 빈 원이 보인다.  
  - 이 영역은 정답 표기 또는 선택 결과를 나타내는 UI처럼 보이지만, 정확한 의미는 화면만으로 단정하기 어렵다.
- `(해설)` 문장은 왼쪽 아래에 큰 글씨로 두 줄에 걸쳐 배치되어 있다.
- 해설 아래 예시 도형은 아래쪽 왼쪽에 위치하며, 사각형의 네 꼭짓점 부근에 작은 파란 표시가 붙어 있어 직각임을 암시하는 시각 요소처럼 보인다.
- 예시 도형은 위쪽 문제 도형보다 아래로 충분히 떨어져 있어, 문제 그림과 해설 예시가 명확히 구분된다.

# [Python DSL 작성 힌트]
- 문제 텍스트를 먼저 배치하고, 그 아래에 판별 대상 도형 1개를 중앙 정렬에 가깝게 놓는 구성이 적절하다.
- 선택 기호 `○`, `×`는 텍스트 요소로 따로 표시하고, 정답/오답 영역처럼 보이는 원형 마커가 있으면 별도 객체로 분리하는 것이 좋다.
- 해설은 문제 본문과 시각적으로 분리된 하단 영역에 배치한다.
- 해설 문장 아래의 예시 도형은 별도 도형 객체로 만들고, 꼭짓점 근처의 파란 표시도 작은 장식 요소로 분리해 표현하면 좋다.
- 상단 도형은 단순 사각형 윤곽선 1개로 표현하고, 내부 채움은 없는 상태로 설정한다.
- 전체 레이아웃은 세로 흐름형이며, 상단 문제 영역 / 중간 선택 영역 / 하단 해설 영역으로 나누면 구현하기 쉽다.

# [주의할 점]
- 정답이 무엇인지 추론하여 채우지 말아야 한다.
- `○`와 `×`의 실제 선택 여부를 확정적으로 해석하지 말아야 한다.
- 위쪽 도형이 정사각형인지 아닌지를 시각 분석에서 단정하지 말고, 보이는 형태만 묘사해야 한다.
- 해설 문장의 의미를 요약할 수는 있지만, 문제 풀이 결과를 대신 제시하면 안 된다.
- 아래쪽 예시 도형의 파란 표시는 직각 표시로 추정되지만, 화면만 보고 정확한 기호 의미를 단정하지 않는 것이 좋다.

# [불확실한 부분]
- 왼쪽 중간의 파란 원형 표시와 `(정답)` 옆 빈 원의 관계는 화면만으로는 정확히 확정하기 어렵다.
- 위쪽 도형이 실제로는 정사각형인지, 직사각형인지, 혹은 정사각형이 아닌 다른 사각형인지 화면 해상도만으로는 단정하기 어렵다.
- 아래쪽 예시 도형의 파란 표시가 정확히 어떤 편집 기호인지(직각 표시, 선택 표시, 강조 표시)는 명확하지 않다.
- 문제 화면 전체가 일부 잘려 보이므로, 좌우/상하 여백의 정확한 범위는 알 수 없다.
