[문제 개요]
- 문제 번호: 30
- 지시문: `원의 중심의 개수가 가장 많은 모양을 고르세요.`
- 선택지: ①, ②, ③, ④, ⑤의 5개 도형 보기
- 화면 중간에 `(정답) ①`이 인쇄되어 있음
- 아래쪽에 `(해설)`과 함께 각 보기의 중심 개수를 비교하는 그림 설명이 있음
- 맨 아래 결론 문장: `따라서 원의 중심의 개수가 가장 많은 모양은 ①입니다.`

[원본에 보이는 요소]
- 상단 왼쪽에 문제 번호 `30.`이 있고, 그 오른쪽에 문제 문장이 한 줄로 배치됨
- 보기 ①~⑤는 모두 도형 그림과 파란색 동그라미 숫자 표식으로 구성됨
- 해설 영역에는 각 도형 위/옆에 빨간 점이 찍혀 있어 중심 위치를 표시하는 듯함
- 각 해설 그림 옆에는 `: 4개`, `: 1개`, `: 3개`, `: 3개`, `: 2개`라는 텍스트가 붙어 있음
- 아래쪽 결론 문장은 가로로 길게 한 줄 표시됨

[수식/관계 구조]
- 등호 `=`를 사용하는 수식 전개는 보이지 않음
- 대신 도형별 중심의 개수를 비교하는 구조임
- 보이는 개수 표시는 다음과 같음:
  - ①: `4개`
  - ②: `1개`
  - ③: `3개`
  - ④: `3개`
  - ⑤: `2개`
- 이 문제는 계산식보다 시각적 비교가 핵심인 도형 관찰 문제로 보임

[등호별 변환 의미]
- 해당 없음
- 등호 연쇄나 식 변형이 나타나지 않음

[빈칸 정책]
- 빈칸은 없음
- 만약 DSL에서 빈 입력 칸이나 미기재 슬롯이 필요하더라도, 추론값으로 채우지 말고 비워 둬야 함
- 원본에 인쇄된 내용만 사실로 반영하고, 보이지 않는 값은 채우지 않음

[반복 패턴]
- 5개의 보기 모두 같은 형식의 번호 표식이 반복됨
- 해설에서는 각 도형마다 빨간 점으로 중심 위치를 표시하는 패턴이 반복됨
- `: n개` 형태의 개수 표기가 각 보기 설명 옆에 반복적으로 배치됨
- 원 또는 원의 일부가 겹치는 도형이 여러 보기에서 반복되어 나타남
- 파란색 원형 번호 표식은 보기 번호를 강조하는 시각적 반복 요소임

[DSL 작성 힌트]
- 전체 구성은 위에서 아래로 `문제문 → 5개 보기 → 정답 표시 → 해설 그림들 → 결론 문장` 순서로 잡으면 좋음
- 보기 ①~⑤는 각각 독립 도형으로 배치하되, 원본처럼 공간을 넓게 써서 분산 배치하는 것이 자연스러움
- 번호 표식은 파란색 동그라미 안 숫자처럼 처리하면 원본 느낌을 살리기 좋음
- 해설 영역의 빨간 점은 도형 위에 오버레이된 강조 표시로 표현하는 것이 적절함
- `: 4개`, `: 1개` 같은 문구는 각 해설 그림 오른쪽에 붙여 배치
- 결론 문장은 하단 전체 폭을 차지하는 한 줄 텍스트로 배치
- 도형의 선 굵기와 색은 대체로 얇은 검은 선, 해설 점은 빨간색, 번호 표식은 파란색으로 구분하는 것이 좋음
- ① 도형은 사각형 외곽선 안쪽에 곡선이 들어간 구조이므로 직선 도형으로 단순화하지 말 것
- ③, ④, ⑤는 원들이 서로 겹치는 구조라서 위치 관계를 유지하는 배치가 중요함

[semantic/solvable 힌트]
- 보이는 사실 기준으로는 각 보기 옆에 중심 개수가 이미 적혀 있음
- `(정답) ①`이 화면에 인쇄되어 있어 정답 표시는 관찰 사실로만 반영 가능함
- 문제의 의미는 “각 도형에서 원의 중심이 몇 개인가”를 비교해 가장 많은 보기를 찾는 것임
- 해설의 빨간 점은 중심 후보를 시각적으로 표시한 것으로 해석됨
- 다만 실제 중심 개수를 새로 계산해 채우거나, 빈칸을 추론해서 보완하지는 말아야 함
