- 전체적으로는 **문제 문장 + 도형 시각 자료**로 구성된 단순한 한 문제 화면이다.

# [세부 내용]
- 도형은 굵은 검은 선으로 그려진 정사각형들로 구성됨.
- 전체 배열을 보면 다음처럼 보인다:
  - **윗줄**: 가운데 부분에 정사각형 3개가 나란히 있음.
  - **가운데줄**: 좌우로 더 길게 이어진 정사각형 5개가 나란히 있음.
  - **아랫줄**: 가운데 부분에 정사각형 3개가 나란히 있음.
- 윗줄과 아랫줄은 가운데보다 좁은 폭으로 배치되어 있고, 가운데줄이 좌우로 더 넓다.
- 정사각형들의 외곽선은 서로 정확히 맞닿아 있어, 겹쳐진 큰 도형 안에서 작은 정사각형과 더 큰 정사각형을 찾아 세는 문제로 보인다.
- 도형 전체는 가운데를 중심으로 좌우 대칭처럼 보인다.
- 배경은 흰색이고, 다른 장식이나 추가 설명은 보이지 않음.

# [Python DSL 작성 힌트]
- 문제 문장은 왼쪽 상단에 2줄 텍스트로 배치하는 형태로 작성하면 좋음.
- 도형은 개별 정사각형들을 **정확한 좌표 격자**로 배치해 표현해야 함.
- 가운데 가로줄이 가장 긴 구조이므로, 상단/하단의 3칸 줄과 가운데의 5칸 줄을 서로 어긋나게 정렬하는 방식이 필요함.
- 각 정사각형의 테두리는 얇은 검은색 선으로 표현하는 것이 적절해 보임.
- 전체 도형은 하나의 큰 직사각형이 아니라, **계단식으로 돌출된 정사각형 묶음**으로 구성하는 방식이 맞음.

# [주의할 점]
- 이 이미지는 풀이 결과를 직접 보여주지 않으므로, 정답 숫자를 추론해 적지 말아야 함.
- 칸의 개수와 전체 구조를 설명할 때, 실제로 보이는 배치만 기준으로 써야 함.
- 도형을 임의로 더 큰 격자나 정사각형 테두리로 바꾸어 해석하지 말아야 함.
- 중앙 부분의 정사각형들이 정확히 몇 칸인지, 외곽선이 어디까지 공통선인지 등은 그림을 보고 신중하게 처리해야 함.

# [불확실한 부분]
- 각 정사각형의 정확한 한 변 길이와 간격은 이미지 해상도상 수치로 확정하기 어렵다.
- 선 두께 때문에 내부 경계선이 일부 겹쳐 보이는 구간이 있어, 세부 좌표는 약간의 오차가 있을 수 있다.
- 도형이 “큰 직사각형 안의 일부 칸”인지, 아니면 “정사각형들이 이어진 독립 도형”인지 시각적으로는 후자로 보이지만, 구현 시에는 격자 기반으로 재구성해야 할 가능성이 있다.
